## 目标

定义关系系统的运行时框架：领袖关系存储与演算、组织间接传导、人口损失→关系折算（双轨）、贸易与委托、关系行动（脱离/效忠）的结算管线。

## 范围

- **包含**：直接领袖关系（R）与间接传导（×1/4）的存储与演算；关系档位（敌对/中立/友善）判定与玩法挂钩；人口损失的双轨累计与折算；领袖委托与商队贸易的流程框架；关系行动（脱离/效忠）的结算时点与状态变更；关系下降的对话触发。
- **不包含**：具体领袖名单与初始值（sy-27）；贸易基准价表（sy-28）；占领经营细则（sy-30）；势力消亡清理（sy-31）；领袖变更影响（sy-32）。

## 详细说明

### 系统定位

关系系统是玩家与外部势力互动的主框架，覆盖交互链的 **[④ 关系经营](../../01-草稿/交互链-循光之城.md#④-关系经营)** 块。核心循环：

```
未效忠（资源封存、不可贸易）→ 完成委托提升关系 → 友善（可贸易、可招募）→ 招募（未效忠）→ 继续委托 → 效忠（R≥100，与关系解耦）
            ↓ 造成人口损失下降关系 → R≤−50 → 脱离 ∨ 敌对
```

### 模块结构

关系系统划分为五个子模块：

| 子模块 | 职责 | 结算阶段 |
|--------|------|----------|
| **关系存储** | 领袖关系表读写、档位判定、历史记录 | 各阶段均可写入 |
| **组织传导** | 组织归属图、直接→间接 ×1/4 演算 | 直接关系变化时触发 |
| **人口损失追踪** | 双轨累计（单位轨 + 城区轨）、折算出关系下降量 | 交战/灾害等造成损失时 |
| **委托与贸易** | 领袖委托刷新/接取/完成、商队履约、贸易结算 | 玩家指挥 / 玩家行动阶段 |
| **关系行动** | 脱离（R≤−50）、效忠（R≥100）的结算与状态变更 | 关系行动阶段 |

### 关系存储

#### 核心概念

| 概念 | 定义 |
|------|------|
| **城市领袖** | 每座城市一名的关系主体；玩家面对的是**对方城市领袖** |
| **直接领袖关系（R）** | 玩家方城市领袖与对方城市领袖之间的直接关系值 |
| **间接领袖关系** | 同组织内其他城市领袖，因直接关系变化而连带变化（×1/4） |
| **关系档位** | 敌对（R≤−50）、中立（−50<R<50）、友善（R≥50） |

#### 关系档案

每位城市领袖（含玩家方）持有一套关系档案，包含与其它每个已知城市领袖的直接关系记录：

| 字段 | 类型 | 说明 |
|------|------|------|
| `selfLeaderId` | string | 本档案所属城市领袖 ID |
| `records` | RelationRecord[] | 对每位其他领袖的直接关系记录 |
| `organizationId` | string? | 所属组织 ID |

单条 `RelationRecord`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `targetLeaderId` | string | 对方城市领袖 ID |
| `rValue` | int | 直接领袖关系 R |
| `previousTier` | RelationTier | 上一回合结束时档位（用于跨越检测） |

#### 档位判定与玩法挂钩（已定）

| 档位 | R 范围 | 贸易 | 常规委托 | 招募委托 | 敌对判定 |
|------|--------|------|----------|----------|----------|
| **敌对** | R ≤ −50 | 不可 | 可 | 不可 | 互为敌人 |
| **中立** | −50 < R < 50 | 可（领袖页面） | 可 | 不可 | 中立 |
| **友善** | R ≥ 50 | 可（领袖页面、八折） | 可 | 可（补位刷新） | 中立 |

关系档位以 −50 和 50 为界；**±50 不算中立**。

### 组织传导

#### 组织归属图

外部城市可归属同一组织（势力）——如 **城邦同盟**、**无敌骄阳会**、**渊光教团**、**诸王国土** 等（见 [设定 · 角色与势力](../../04-设定/02-角色与势力/)）。

关系系统维护一张**组织归属图**（`OrganizationGraph`）：

```
OrganizationGraph：
  - 组织 → 城市领袖列表
  - 每个组织内，各领袖间互为间接关系主体
```

#### 传导规则（已定）

当玩家与组织内某城市领袖的**直接领袖关系 R** 发生变化（ΔR）时：

- 同组织内其他每名城市领袖的间接关系 **同时** 变化 **ΔR × 1/4**（向下取整）
- 间接传导**仅组织内**，**不**向组织外扩散（已定，2026-07-16 裁定）
- 连锁不传导（B 的间接变化不再传回 A）
- **NPC 触发同规则（已定）**：NPC 领袖间、NPC 对玩家造成人口损失时，折算与间接传导与玩家侵害他方**同一规则**
- **组织定位与嵌套（已定）**：同组织城市的相处机制与**玩家招募**相似——**相互友善，但各自独立**（每位领袖仍是独立关系主体）；**不设组织嵌套**，组织为并列分组，每座城市至多归属一个组织（见 [势力系统 · 组织](./势力系统.md#组织)）

**示例**：组织内有领袖 A、B、C。玩家对 A 直接关系 +20 → A 直接 +20，B 间接 +5，C 间接 +5。

### 人口损失追踪

#### 双轨累计（已定）

人口损失由**单位轨**与**城区轨**分别统计、分别累计、分别折算：

| 轨道 | 统计载体 | 计入人口 | 折算 | 结算目标 |
|------|----------|----------|------|----------|
| **单位** | 受害队伍、设施守军等可交战单位实例 | 编制人数减员 | 每 **2** 人→直接关系 −1 | 该单位的 `home_city_ref` 对应的城市领袖 |
| **城区** | 受害城区实例 | 城区居民减员 | 同上 | 城区所属城市的城市领袖 |

- 各轨维护**独立的待结算人口损失累计**；不足 2 人的余数**仅在本轨内**跨事件保留
- 两轨**不跨轨合并**后再折算（单位轨 1 人 + 城区轨 1 人 → 均不触发）
- 达阈值后 **当场** 更新受害城城市领袖对行为方的直接领袖关系，并触发组织间接传导

#### 己方归属人口损失（招募后，已定）

玩家已招募某城市领袖后，该领袖所属组织下、玩家指挥范围内的同归属人口减员：

- 组织级统一累计：**4 人降 1 点**与该领袖的直接领袖关系
- 统计范围含 recruited 外部城城区居民、编组、玩家移动城内同归属人口等
- 与侵害他方（2 人降 1）**分轨**，不跨规则合并
- 恢复：完成委托或人口归属转化

### 委托与贸易

#### 领袖委托（已定 · 首版）

领袖委托是提升关系的主路径：

| 项 | 口径 |
|----|------|
| **展示数量** | 每位领袖恒展示 **3 条**；至少 **2 种**类型 |
| **刷新** | 完成或失效后立即补位 |
| **已接取失败** | 扣奖励关系的 **1/4** |
| **未接取失效** | 直接更换，不扣关系 |
| **奖励** | 直接关系 **+10～+20**（触发组织间接传导） |
| **招募委托** | 仅 **友善**（R ≥ 50）时补位刷新 |

#### 商队贸易（已定）

| 项 | 口径 |
|----|------|
| **入口** | **领袖页面**（唯一） |
| **创建方式** | 仅贸易创建；商队自动编组 |
| **己方商队** | 可指挥 |
| **可用档位** | 中立（常规价格）、友善（八折）、效忠（关闭） |
| **禁用** | 敌对（R ≤ −50）、未效忠、效忠后 |
| **旁路** | 停泊/野外贸易旁路 **deny** |

### 关系行动

#### 第五阶段（已定）

关系行动在回合五阶段框架中位于**环境行动之后**：

```
玩家指挥 → 玩家行动 → AI 行动 → 环境行动 → 关系行动 → 下一回合玩家指挥
```

#### 结算内容

| 行动 | 触发条件 | 效果 |
|------|----------|------|
| **强制脱离** | 未效忠已招募城，R ≤ −50 | 城区强制脱离、队伍转敌对、领袖回名下城市、原任职城区复位 |
| **效忠** | 已招募，R ≥ 100 | 城市消解、资源归玩家、关闭贸易、与关系解耦 |
| **档位跨越** | 中立↔友善、中立↔敌对等阈值跨越 | 下回合玩法生效（贸易/委托/敌对判定变化） |

#### 关系下降对话（已定）

当与某城市领袖的**直接领袖关系**发生**下降**时，由**剧情/脚本对话**框架触发该领袖对玩家的**警告**台词。对话为叙事反馈，不在机制结算前预告后果。
