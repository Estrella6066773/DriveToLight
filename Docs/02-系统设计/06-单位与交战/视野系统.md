## 侦察塔视野（已定）

侦察塔是一座**战略设施**（`L5_facility_defs.csv`，`category=auxiliary`，建造 30 金属 / 3 回合），建于野外合法格子。**不是**特殊城区模块。

| 项目 | 口径 |
|------|------|
| **被动** | 位于侦察塔所在格的**任何单位**，视野半径 **+3 格**（加算，在单位当前视野之上叠加；野外设施公有，不区分建造者与访客） |
| **主动** | **无** |
| **前提** | 侦察塔未被击毁 |
| **工作效率** | **不**受影响（被动固化为 +3，不随运维人数变化） |

**地形修正器（`L0_cross_modifiers`）**：所在格地形可叠加修正值——

| 地形 | 修正 | 说明 |
|------|------|------|
| **密林** | **−1** | 树冠遮蔽削弱目视 |
| **丘陵** | **+1** | 高处开阔 |
| **山地** | **+2** | 显著居高临下 |

修正器经 `L0_cross_modifiers`（`stat_key=watchtower_vision_bonus`）叠加后，单位视野半径 = 当前视野 + 3 + 地形修正值（可为负）。

与旧版的主要差异：旧侦察塔为特殊城区模块（势力级视野增益，被动 +2 格多座加算、主动 +50% 视野），现已统一为单一野外设施版本。

## 地图情报与实体揭示（已定）

### 资源点

权威规则见 [单位类型与视野 · 资源点揭示](./单位类型与视野.md#资源点揭示)（初始**未知**；进入视野后种类与储量一并即时揭示；离开视野后停留在上次看到的状态）。

### 他人城市与他人单位

敌对或中立的**他人城市**、**他人单位**（含敌对队伍）均须进入己方**单位视野**后才对玩家可见。

| 对象 | 揭示条件 | 离开视野之后 |
|------|----------|--------------|
| **他人城市** | 己方单位视野覆盖其占格（或可见 footprint） | **记入**战略地图：位置与已知信息保留，可再次导航 / 点选；地图上显示**名称**、**势力**、**关系档位**，可打开城市详情页 |
| **他人单位** | 己方单位视野覆盖其所在格 | **仅实时可见**：离开视野后地图上**不再保留**该单位标记 |

- **他人城市**：首次揭示后写入地图已知层；之后即使当前无视野覆盖，仍显示为已发现城市。地图上展示**名称**、**势力**、**关系档位**三个字段，可点选打开**城市详情页**。  
- **他人单位**：不写入持久已知层；只在当前有视野的格子上显示；移出或失去视野即从玩家地图上消失。  
- 己方城市与己方单位的显示不走本表（始终可知可控）。

### 城市详情页（已定）

从战略地图点选他人城市后，可打开**城市详情页**，展示以下信息与导航入口：

| 展示内容 | 说明 |
|----------|------|
| **名称** | 城市名称 |
| **势力** | 所属势力，可点开**势力详情页** |
| **关系档位** | 当前领袖关系档位 |
| **人口统计** | 城市人口概况 |
| **领袖简介** | 简短介绍，可点击进入**领袖页面** |

**城区详情页**：点选单个城区可打开城区详情页，其中含所属城市信息，可由此进入城市详情页。

### 领袖页面（已定）

领袖页面展示以下信息：

| 展示内容 | 说明 |
|----------|------|
| **绑定的人口与城市** | 该领袖所绑定城市的人口概况 |
| **所属势力** | 势力归属 |
| **自身特性** | 领袖能力与特质 |
| **自身介绍** | 领袖背景介绍文本 |

- 人口损失按 [分轨累计](../05-城市与领袖/势力系统.md#分轨累计已定) 折算后，当场更新受害城 **城市领袖** 对行为方 **城市领袖** 的 **直接领袖关系**，并触发 [组织间接传导](../05-城市与领袖/势力系统.md#组织)（×1/4）。

## 替代方案：放弃视野系统（已否决 · 备档）

> **2026-07-16 裁定**：采用**完整视野系统**（战争迷雾 + 单位视野 + 侦察塔增益）；本节方案**不采用**，仅留档备查（sy-36 已关闭）。

若放弃完整视野系统，可采用以下简化方案：

| 维度 | 规则 |
|------|------|
| **战争迷雾** | **全部揭示**——地图地形、他人城市、他人单位始终对玩家可见，无视野遮挡 |
| **资源点** | 默认标示格子**有**资源点存在；**不显示**类型与储量（进入单位视野后一并揭示） |

### 资源点揭示

资源点对玩家可见性遵循视野驱动模型：

- **初始**：所有格子默认**未知**。
- **进入视野**：己方单位视野覆盖时，种类与储量**一并**即时写入玩家战略地图已知层。
- **离开视野**：停留在**上次看到的状态**，不自动更新，不因离开而退回。

多队同时进入同一格时揭示不叠加。

### 受影响的系统

| 系统 | 变更 |
|------|------|
| **侦察塔** | 移除，或重新设计为其他城区能力 |
| **侦察队** | 移除"视野范围广"属性；能力改为"走到资源点格时揭示全部信息" |
| **他人城市 / 他人单位** | 始终全图可见，取消"进入视野后揭示"规则 |
| **单位 `visionRange` 字段** | 移除 |
| **战争迷雾渲染** | 移除 |
| **ResourceRevealService** | 视野揭示逻辑简化：进入视野 → 种类与储量一并写入 |

### 体验取舍

| 优势 | 代价 |
|------|------|
| 大幅降低开发与维护成本 | 失去探索中的空间悬念与逐步揭示感 |
| 玩家可全局规划路线与资源策略 | 敌人动向无策略模糊——无法通过侦察获取信息优势 |
| 降低新玩家认知负荷 | 与 4X 类游戏惯例脱节 |
