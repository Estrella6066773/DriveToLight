## 目标

定义玩家可组建的远征队伍模板、能力组合与执行规则，支撑探索、运输、资源开发与战斗玩法。

## 范围

- **包含**：队伍类型、职责、**队伍资产与组建门槛**、**人数对比率与能力衰减**、基础限制、战斗身份判定、队伍 AI 执行规则。
- **不包含**：具体数值平衡、逆风保护细则、程序实现（回合阶段见 [回合与行动表](../07-玩法循环/回合与行动表.md)）。

## 详细说明

### 概述

队伍系统是玩家将城市影响力扩展到荒野的核心机制。玩家组建各类队伍外出执行探索、运输、建设等任务。

> **详细单位定义**（属性、视野、通讯能力等）见 [单位类型与视野](./单位类型与视野.md)。
> 
> **视野同步与通讯机制**见 [通讯与飞信（草稿）](../../01-草稿/通讯与飞信.md)（非正式）；索引见 [通讯与飞信系统](./通讯与飞信系统.md)。

### 能力化设计原则

- 程序不写死“某队伍只能做某件事”，只提供可启用能力的执行通道。
- 队伍行为由 ScriptableObject 配置决定；开发者在引擎内手动配置能力与参数（可由 MCP 辅助执行）。
- 队伍“种类”在实现上视为预设模板，本质是能力组合。

### 队伍模板概览（默认预设）

**首版默认队伍类型以本表为准**（侦察、勘探、运输、工程、飞信五类）；早期草稿中的「步兵团」**不**纳入默认预设，武装交战由能力 SO 配置（如为某模板启用 `attack_strike`）实现，而非单独步兵模板。

| 队伍类型 | 核心职责 | 详细文档 |
|----------|----------|----------|
| **侦察队** | 快速探索大范围区域，揭示资源点[种类](./单位类型与视野.md#资源点揭示等级)；默认启用侦察能力 | [单位类型与视野 - 侦察队](./单位类型与视野.md#1-侦察队模板) |
| **勘探队** | 精确勘察资源点，揭示[具体储量](./单位类型与视野.md#资源点揭示等级)；默认启用勘探能力 | [单位类型与视野 - 勘探队](./单位类型与视野.md#2-勘探队模板) |
| **运输队** | 在资源点与城市之间[运输物资](./单位类型与视野.md#3-运输队模板)；默认启用搬运能力 | [单位类型与视野 - 运输队](./单位类型与视野.md#3-运输队模板) |
| **工程队** | [建造设施](./单位类型与视野.md#4-工程队模板)、[运维设施](./单位类型与视野.md#4-工程队模板)；默认启用建造与运维能力 | [单位类型与视野 - 工程队](./单位类型与视野.md#4-工程队模板) |
| **飞信** | [同步视野信息](../../01-草稿/通讯与飞信.md#飞信)、传递指令的一次性单位 | [单位类型与视野 - 飞信](./单位类型与视野.md#5-飞信模板) |

### 队伍资产与组建门槛

除 [人口编制](../04-资源与人口/人口与迁移.md#队伍与人口) 外，组建队伍实例还须满足 **队伍资产** 条件（框架 **已定**，具体清单 **单独设计**，见 OPEN-044）。

| 项 | 结论 |
|----|------|
| **定义** | 面向**编队**的可调配 **物品 / 装备组**（归属 [物品层](../03-图层与地点/地图图层.md) 或城市库存，存储层级 **待定**），用以表达「这支队伍凭什么具备某类能力」 |
| **粒度** | 以**队伍实例**为单位：组建时消耗或占用；解散 / 撤回时是否返还 **待定** |
| **与四类资源** | **金属 / 能源 / 食物 / 人口** 为宏观资源；队伍资产为**模板级组建门槛**，不与「维持人口」「远征食物开关」混谈 |
| **与能力 SO** | 模板 `ability_set` 决定**能做什么**；队伍资产决定**能否组建该模板**（或能否满编，细则 **待定**） |
| **具体内容** | 有哪些资产类型、各模板需要什么、如何生产或购买 **不在本文列举**；由后续专篇或配置表定义 |

程序占位见 [队伍与战斗数据结构 · team_asset_requirement_json](../../03-程序设计/03-数据字典/队伍与战斗数据结构.md#表-1team_template_config队伍模板配置so)。

### 默认编制与人数影响

> 默认值可在配置文件中调整，以下为当前设计：

| 队伍类型 | 默认人数 |
|----------|----------|
| 侦察队 | 3 人 |
| 勘探队 | 5 人 |
| 运输队 | 20 人 |
| 工程队 | 50 人 |

- 队伍人数可因战斗或事件**低于满编**，且不会仅因减员而立即解散（低于 `min_headcount` 的行为见 OPEN-008）。
- **满编人数** = 模板 `default_headcount`（上表默认值）；**人数比** = `current_headcount / default_headcount`（&lt; 1 时为减值）。

**人数对比率影响的能力通道（已定框架）**

| 受影响项 | 通道 | 说明 |
|----------|------|------|
| **视野范围** | `headcount_factor_curve` → 侦察 / 独立视野参数 | 人数越低，可揭示范围越小 |
| **负载上限** | 同上 → 搬运能力参数 | 运输与装卸上限随人数下降 |
| **工作效率** | **`work_efficiency` 数值通道**（人数比乘法项） | 工作时长变长；见 [工作效率与时长](../07-玩法循环/回合与行动表.md#工作效率与时长) |
| **攻击效果 / 战力** | 同上 → 攻击能力参数；交战公式中 **人数系数** | 仅 **具备武装** 的模板生效；结算见 [战斗结算流程](#战斗结算流程当前版本) |

- **移动速度不**随人数比变化；由模板 `unit_base_profile.base_move_speed`（及 Buff 等其它修正）决定，**不**经 `headcount_factor_curve`。
- 非工作类能力（视野、负载、攻击）经 **`headcount_factor_curve`** 统一映射：`F(headcount_ratio)` 写入 `ability_runtime_json`（程序见 [headcount_factor_curve](../../03-程序设计/03-数据字典/队伍与战斗数据结构.md#表-8headcount_factor_curve人数影响曲线)）。
- **工作效率**单独走数值通道，不重复套用同一曲线权重（与程序设计一致）。
- 各通道的**默认曲线、战力系数数值**归入 [04-数值框架/](../04-数值框架/)（配置模板搭建后设计，OPEN-008）。

详情与模板侧口径见 [单位类型与视野 · 默认编制与人数衰减影响](./单位类型与视野.md#默认编制与人数衰减影响)。

### 能力通道与参数配置

- 每项能力由独立能力配置 SO 定义，示例：
  - **建造能力**：二级选择设施类型；对照建造时长、资源消耗、按设施类型的建造条件（见 [单位类型与视野 · 工程队](./单位类型与视野.md#4-工程队模板)）；实际时长受 [工作效率](../07-玩法循环/回合与行动表.md#工作效率与时长) 影响；
  - **运维能力**：对已存在设施的运维；对照时长与效果受工作效率影响；
  - 搬运能力：装货/卸货为按回合推进的工作（对照时长各 1 回合）；实际时长受工作效率影响；负载上限、超载惩罚；
  - 攻击能力：打击范围、攻击效果、命中修正、反击许可。
- 队伍模板 SO 只负责“启用哪些能力”与“能力参数引用”，不在程序中写死。
- 程序层只提供能力执行入口（如 `TryBuild`、`TryTransport`、`TryAttack`），具体数值与开关完全读取 SO。

### 通讯与视野同步

外出队伍发现的视野信息需要通过特定机制同步回主城：

- **[即时通讯](../../01-草稿/通讯与飞信.md#即时通讯)**：在[通讯站](../03-图层与地点/建筑层/城区总览.md#特殊城区)信号覆盖范围内的单位，可实时同步视野信息。
- **[飞信](../../01-草稿/通讯与飞信.md#飞信)**：超出信号范围或不支持即时通讯的单位，定期发送[飞信](./单位类型与视野.md#5-飞信模板)传递视野记录。

详见 [通讯与飞信（草稿）](../../01-草稿/通讯与飞信.md) 与 [通讯与飞信系统](./通讯与飞信系统.md)。

### 战斗身份与关系判定

交战参与方类型（队伍、设施、城市）与敌我规则见 [交战系统](./交战系统.md)。本节保留队伍侧摘要：

- 是否加入战斗队伍与是否属于敌对势力，为两条独立判定。
- 单位是否判为敌人，主要依据 [势力系统](./势力系统.md) 中的双方关系。
- 加入战斗队伍只影响该单位是否进入当前战斗流程，不直接决定敌我关系。

### 队伍 AI 接管规则（连续指令）

- 玩家可下达连续指令，形成有先后顺序的**指令表**（程序实现为指令队列；见 [回合与行动表 · 指令表与自主执行](../07-玩法循环/回合与行动表.md#指令表与自主执行)）。
- 指令表列出**任务目标**；**如何执行**由 AI 自主决定（路径、等待、遇敌应对等）。
- 队伍 AI 按表顺序逐步执行任务，不跳过任务目标类型，不擅自改写任务目标。
- 当前指令受阻时，AI 可在同一任务目标内做可行调整（换位、等待、改路径、改攻击点）。
- 若持续受阻且无法推进，AI 将该条标记为未完成并进入下一条，同时写入战报说明原因。
- [遭遇意外](../07-玩法循环/回合与行动表.md#遭遇意外的中断与恢复)时，不打断指令表结构，仅打断当前条目的自主执行；恢复后从同一条目继续。

### 任务优先级模型

- 指令队列以玩家下达顺序为第一优先级。
- 单条指令内部采用「任务目标 > 生存判断 > 执行动作」的决策顺序。
- 队伍 AI 策略可设为：
  - **保守**：优先降低损失，再推进任务目标。
  - **激进**：优先推进任务目标，再处理损失控制。

### 遭遇敌人时的行为

单位在移动或执行指令过程中**遭遇敌对单位**（进入威胁范围、路径被阻断、或被动卷入交战）时，**不采用全局固定反应**（如一律回避或一律交战），而由当前 **`ai_strategy`** 决定优先级：

| 策略 | 遇敌优先级 | 典型动作（在同一任务目标内调整） |
|------|------------|----------------------------------|
| **保守** | 降低损失 **先于** 推进任务 | 改道绕行、脱离接触、暂停等待威胁离开；仅在损失可控且能继续推进时才恢复当前指令 |
| **激进** | 推进任务 **先于** 损失控制 | 优先维持当前指令方向与目标；必要时交战或穿越威胁区以完成任务，再处理减员 |

约束：

- 遇敌反应**不改写**指令队列中的任务目标类型，仅在当前 `team_command_item` 内做可行调整（与 [队伍 AI 接管规则](#队伍-ai-接管规则连续指令) 一致）。
- **无武装**队伍（默认侦察队、勘探队、运输队、工程队）不具备主动交战能力；激进策略下仍优先推进任务，但只能通过改路径、等待或接受风险通过等方式执行，**不得**凭空获得战斗能力。
- 若遇敌后持续无法按当前策略推进，按 AI 接管规则标记当前条目未完成或失败，并写入战报说明。

程序实现见 [队伍与指令执行 · 遇敌判定](../../03-程序设计/02-运行时逻辑/队伍与指令执行.md#遇敌判定与策略分支)。

### 队伍运作与食物补给

城外队伍在野外执行指令、推进工作时，默认从城市**食物**库存中持续消耗补给（与 [四种核心资源 · 食物与人口](../04-资源与人口/四种核心资源.md#食物与人口) 中「维持城市人口」的规则分开结算）。

| 项 | 结论 |
|----|------|
| **默认** | **启用**——城外队伍运作消耗食物 |
| **策略开关** | 玩家可在**存档设置**（新游戏或对局选项）中将 **`team_food_upkeep_enabled`** 设为 **false**，关闭后城外队伍运作**不再**扣减食物 |
| **不变项** | 关闭开关**不影响**城市人口的食物维持与供应不足衰减 |
| **待定** | 消耗量如何按人数、队伍类型、回合阶段计算；结算阶段（环境行动 / 玩家行动等） |

程序字段见 [回合与行动数据结构 · save_gameplay_options](../../03-程序设计/03-数据字典/回合与行动数据结构.md#save_gameplay_options存档对局选项)。

### 反击限制

- 仅当防守方具备反击能力，且攻击者位于防守方打击范围内，才触发反击。
- 反击按正常战斗流程结算。
- 反击不会触发新的反击响应（禁止反击连锁）。

### 战斗结算流程（当前版本）

完整交战管线（含设施、城市目标）见 [交战系统 · 战斗结算](./交战系统.md#战斗结算当前版本)。队伍间对攻按下列顺序结算（**不含士气**）：

1. 判定可攻击目标。
2. 发起攻击。
3. 计算攻击方战力：攻击能力系数 × **人数系数**。
4. 将攻击效果传递至防守方（受防御能力与**人数系数**影响）。
5. 结算目标**减员**、**建筑损伤**或**设施耐久**（按目标类型）。
6. 若满足反击条件，按同流程执行一次反击。

士气相关步骤已移入 [交战系统 · 士气（草稿）](./交战系统.md#士气草稿首版不纳入)，首版不启用。
