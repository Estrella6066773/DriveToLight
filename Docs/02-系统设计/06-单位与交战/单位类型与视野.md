## 目标

定义游戏中队伍模板、能力配置与视野机制，明确资源点揭示系统与能力化实现边界。

## 范围

- **包含**：单位类型定义、基础属性、视野机制、资源点揭示等级。
- **不包含**：具体数值平衡、AI行为逻辑、战斗系统（若有）。

## 详细说明

### 程序设计原则

> **所有单位类型都需要单独使用SO（ScriptableObject），可以被策划配置，并且能力也是可配置项，而不被写死在程序中。**
> 
> **允许变种单位使用不同结构的SO，并且可以兼容性读取同一基类的SO。**

- 单位模板通过 **ScriptableObject（SO）** 配置，而非硬编码。
- 单位能力（视野、移动速度、武装状态、建造、搬运、攻击等）均为可配置项。
- 支持创建**变种单位**：使用不同结构的SO，但继承同一基类，确保兼容性。
- 策划或开发者可通过引擎内配置自由调整单位参数，无需修改程序代码。

### 能力通道模型

- 程序层仅提供能力执行通道，不限制具体模板必须具备哪些能力。
- 能力是否启用与参数细节由能力 SO 决定。
- 当前已确认的基础能力通道：
  - 侦察能力（揭示种类）；
  - 勘探能力（揭示储量）；
  - 搬运能力（负载与装卸）；
  - 建造能力（建设与维修）；
  - 攻击能力（打击与反击）。

### 单位基础属性

| 属性 | 说明 | 配置类型 |
|------|------|----------|
| **移动速度** | 单位在地图上的移动效率 | 数值（可配置）；**不**随当前人数比衰减 |
| **武装状态** | 是否有武装/战斗能力 | 布尔（可配置） |
| **人数规模** | 单位承载的人口数量 | 数值（可配置） |
| **视野范围** | 单位能揭示的地图范围 | 数值（可配置） |
| **特殊能力** | 单位特有的功能能力 | SO引用（可配置） |

### 默认编制与人数比影响

> 以下为当前默认值；策划可在配置文件中调整。汇总口径见 [队伍系统 · 默认编制与人数影响](./队伍系统.md#默认编制与人数影响)。

| 队伍类型 | 编制人数（默认 = 上限） |
|----------|-------------------------|
| 侦察队 | **3** |
| 勘探队 | **5** |
| 运输队 | **15** |
| 工程队 | **10** |
| 民兵 | **15** |
| 步兵 | **10** |
| 弓手 | **10** |

- 队伍人数可因战斗或事件减少，且不会立即解散（`current_headcount ≥ 1` 即可维持；降至 **0** 细则 **待定**）。
- **已定**：默认人数 = 编制上限；**人数比** = `current_headcount / 编制人数`（满编 = 上表）。

人数下降同步影响：

| 能力 | 影响方式 |
|------|----------|
| **视野范围** | 满编视野 × **人数比系数**（首版 1:1 线性） |
| **负载上限** | 满编负载 × **人数比系数** |
| **工作效率** | **参与人数 × `work_efficiency`**（见 [回合与行动表 · 工作效率与工作量](../07-玩法循环/回合与行动表.md#工作效率与工作量)） |
| **攻击效果** | 满编战力 × **人数比系数** + 交战 **人数系数**；仅武装模板 |

**移动速度**不随人数比变化；见 [单位基础属性](#单位基础属性) 与模板 `base_move_speed`。

人数比系数映射由程序表 `headcount_factor_curve` 配置；首版全线性（系数 = 人数比）。各模板满编时的视野格数、负载、战力、移速等 **SO 默认参数待定**（sy-03；[04-数值框架/](../04-数值框架/) 未建）。

### 单位模板清单（默认预设）

#### 1. 侦察队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **极快** | 迅速探索大片区域 |
| 武装状态 | **无武装** | 无法主动交战；遇敌应对见 [队伍系统 · 遭遇敌人时的行为](./队伍系统.md#遭遇敌人时的行为) |
| 人数规模 | **少** | 轻装上阵，机动性强 |
| 视野范围 | **广** | 可揭示大范围内的资源点 |

**默认能力**：
- **揭示资源点种类**：进入视野内的资源点，可揭示其**种类**（是什么类型的资源点）。
- 无法揭示具体储量（需勘探队）。

#### 2. 勘探队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **较快** | 比侦察队慢，但仍有良好机动性 |
| 武装状态 | **无武装** | 无法战斗，需护卫或避开危险区域 |
| 人数规模 | **中** | 携带勘探设备，需要一定人力 |
| 视野范围 | **一般** | 视野不如侦察队广阔 |

**默认能力**：
- **揭示资源点储量**：对资源点执行 **勘探** 工作时，**50%** 成果至少 **揭示种类**，**100%** **揭示储量**；详见 [工作 · 勘探与分级揭示](../07-玩法循环/工作.md#特例勘探与分级揭示) 与 [资源点揭示等级](#资源点揭示等级)。
- 进入视野时仍可按能力即时提升揭示等级；与工作成果取**较高**等级，**不降级**。

#### 3. 运输队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **均衡** | 平衡速度与载重 |
| 武装状态 | **无武装** | 需要护卫保护 |
| 人数规模 | **中** | 负责运输作业 |
| 视野范围 | **一般** | 基础视野 |

**默认能力**：
- **运载物资**：可在资源点与城市之间运输资源。
- **载重能力**：有最大载重量限制，超载会影响移动速度。
- **装货 / 卸货**：各占一项 [工作中](../07-玩法循环/回合与行动表.md#工作中状态) 任务；所需工作量对照值各 **1**；每回合增量受 [工作效率与工作量](../07-玩法循环/回合与行动表.md#工作效率与工作量) 影响。

#### 4. 工程队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **均衡** | 携带工程设备，速度适中 |
| 武装状态 | **无武装** | 需要保护 |
| 人数规模 | **大** | 需要大量人力进行建设 |
| 视野范围 | **一般** | 基础视野 |

**默认能力**（两类**工作**，均占用 [工作中](../07-玩法循环/回合与行动表.md#工作中状态) 进度）：

| 工作 | 说明 | 玩家指挥 |
|------|------|----------|
| **建造设施** | 在合法位置新建设施 | 二级选设施类型；须指定 **被工作对象**（资源点 / 工地）；**工作对象**为工程队 |
| **运维设施** | 对已存在设施运维 | **被工作对象**为目标设施；**工作对象**为工程队 |

**建造设施 · 二级选择**

- 一级：指令类型 = 建造设施。
- 二级：选择具体**设施类型**（如桥梁、资源点开采站、驿站等）；各类型对应独立 SO 配置。
- **建造条件因设施而异**，程序通过 `facility_type_config` 读取，不在代码中写死。示例：
  - **桥梁**：须满足 [地图图层 · 裂谷与桥梁](../03-图层与地点/地图图层.md#裂谷与桥梁)（裂谷两端非裂谷、直线连接）。
  - **资源点设施**（如 **矿区 / 果园 / 能源站 / 征兵办**）：须落在对应 [资源层](../03-图层与地点/地图图层.md#图层一览) 资源点，且满足储量/揭示等级等条件；对照建造 **3 回合**（**细则待定**，sy-10）。
- 条件不满足时，指挥阶段即不可下达或提示失败（**UI 反馈方式待定**）。

**运维设施**

- 针对**已建造**的设施执行运维工作；所需工作量对照值默认 **3**；每回合增量受 [工作效率与工作量](../07-玩法循环/回合与行动表.md#工作效率与工作量) 影响。
- 工程队侧重荒野设施与资源点设施的建造 / 运维；对**城区**的修复为指定城区的多回合 [工作](../../07-玩法循环/工作.md)（**工作对象** = 工程队）。邻近 [城坞](../../03-图层与地点/建筑层/运作与居民.md#功能模块城坞侦察塔等) 时可获效率加成（**分工细则待定**，sy-11）。

程序实现见 [队伍与战斗数据结构 · 工程队工作](../../03-程序设计/03-数据字典/队伍与战斗数据结构.md#工程队工作建造与运维)。

### 资源点揭示等级

资源点（资源层）对玩家的可见性分为**三级**，程序字段 **`resource_reveal_level`**（与存档中每格 **`reveal_level`** 同义）：

| 等级 | 程序值 | 玩家可见（摘要） | 典型获取方式 |
|------|--------|------------------|--------------|
| **1 · 隐藏** | `hidden` | 仅知该格**有**资源点（图标/标记），**不知**类型与储量 | 默认；未纳入侦察/勘探视野 |
| **2 · 揭示种类** | `kind` | 知资源点**类型**（矿藏 / 果地 / 遗迹 / 村镇等），**不知**储量 | [侦察队](#1-侦察队模板) 视野；或 [勘探](#2-勘探队模板) 工作 **50%** 成果 |
| **3 · 揭示储量** | `amount` | 知**具体储量**（可采集量） | [勘探队](#2-勘探队模板) 勘探工作 **100%** 成果 |

> 草稿「三层揭示：1 隐藏 / 2 揭示类型 / 3 揭示类型储量」与上表**同义**；正式用语为 **隐藏 / 揭示种类 / 揭示储量**。

**合并规则（已定）**

揭示是**分等级**的序数，**不是**可叠加的 Buff：

| 程序值 | 序数（比较用） | 含义 |
|--------|----------------|------|
| `hidden` | 0 | 仅知有资源点 |
| `kind` | 1 | 知种类 |
| `amount` | 2 | 知储量 |

对同一资源点（同一地图格 `resource_reveal_level`）：

1. **只保留当前最高等级**——`new_level = max(已有等级, 本次来源等级)`（按上表序数比较）。
2. **不降级**——已 `kind` 不会因新来源只给到 `hidden` 而退回；已 `amount` 同理。
3. **不叠加**——多队、多来源**不会**把两级信息拼成更高一级以外的额外条目；例如两支队伍都达到 `kind`，结果仍是 `kind`，**不是**「双倍种类信息」。
4. **即时写入**——等级变化后**当场**写入玩家战略地图已知层；**不存在**视野延迟或待同步队列。

**来源与合并（示例）**

| 已有等级 | 新来源 | 合并后 |
|----------|--------|--------|
| `hidden` | 侦察队视野 → `kind` | `kind` |
| `hidden` | 勘探 50% → `kind` | `kind` |
| `kind` | 侦察队再次进入视野 | `kind`（不变） |
| `kind` | 勘探 100% → `amount` | `amount` |
| `amount` | 侦察队进入视野 | `amount`（不变） |

侦察队**即时**提升与工作成果、多队视野重叠，均走同一 **`max`、不降级** 规则（见 [工作 · 勘探与分级揭示](../07-玩法循环/工作.md#特例勘探与分级揭示)）。

**多队重叠发现同一格（已定）**

- 任意时刻、任意多支队伍同时对同一格产生揭示贡献时：先分别计算各来源**拟提升**的等级，再对玩家已知层执行**一次** `max(当前, 各来源…)`。
- **与发现顺序无关**——先写入侦察 `kind`、后写入勘探 `amount`，与同一回合内反序结算，最终均为 `amount`。
- **与队伍数量无关**——十支侦察队同时看见同一格，结果与一支相同：`kind`。
- UI 可在合并后**单次刷新**展示最终等级；刷新动画顺序**不改变**已知层结果（视觉细节归 sy-23）。

**揭示流程**

1. 格子生成或首次进入玩家相关视野时，默认为 **`hidden`**（若该格含资源点）。
2. **侦察队**进入视野 → 若具备侦察能力，提升至 **`kind`**（即时，不走工作进度）。
3. **勘探队**对资源点执行 **勘探** 工作 → [工作 · 勘探与分级揭示](../07-玩法循环/工作.md#特例勘探与分级揭示)：
   - **50%** 工作成果：等级至少 **`kind`**；
   - **100%** 工作成果：等级 **`amount`**。
4. 等级变化 **即时** 写入玩家战略地图已知层（**无**视野延迟；合并规则见上节）。

**与建造的关系**

- 建立 **矿区 / 果园 / 能源站 / 征兵办** 等资源点设施时，通常须 **`kind`** 及以上；具体门槛见 sy-10。

### 他人城市与他人单位（摘要）

他人城市、他人单位须进入己方**单位视野**才可见：**城市**揭示后**记入**战略地图；**单位**仅实时可见、离开视野后不保留。权威见 [视野系统 · 他人城市与他人单位](./视野系统.md#他人城市与他人单位)。
