## 目标

定义游戏中队伍模板、能力配置与视野机制，明确资源点揭示系统与能力化实现边界。

## 范围

- **包含**：单位类型定义、基础属性、视野机制、资源点揭示。
- **不包含**：具体数值平衡、AI行为逻辑、战斗系统（若有）。

## 详细说明

### 程序设计原则

> **所有单位类型都需要单独使用SO（ScriptableObject），可以被策划配置，并且能力也是可配置项，而不被写死在程序中。**
> 
> **允许变种单位使用不同结构的SO，并且可以兼容性读取同一基类的SO。**

- 单位模板通过 **ScriptableObject（SO）** 配置，而非硬编码。
- 单位能力（视野、移动速度、武装状态、建造、搬运、攻击等）均为可配置项。
- 支持创建**变种单位**：使用不同结构的SO，但继承同一基类，确保兼容性。
- 策划或开发者可通过引擎内配置自由调整单位参数，无需修改程序代码。

### 能力通道模型

- 程序层仅提供能力执行通道，不限制具体模板必须具备哪些能力。
- 能力是否启用与参数细节由能力 SO 决定。
- 当前已确认的基础能力通道：
  - 侦察能力（揭示全部信息）；
  - 搬运能力（负载与装卸）；
  - 建造能力（建设与维修）；
  - 攻击能力（打击与反击）。

### 单位基础属性

| 属性 | 说明 | 配置类型 |
|------|------|----------|
| **移动速度** | 单位在地图上的移动效率 | 数值（可配置）；**不**随当前人数比衰减 |
| **武装状态** | 是否有武装/战斗能力 | 布尔（可配置） |
| **人数规模** | 单位承载的人口数量 | 数值（可配置） |
| **视野范围** | 单位能揭示的地图范围 | 数值（可配置） |
| **特殊能力** | 单位特有的功能能力 | SO引用（可配置） |

### 默认编制与人数比影响

> 以下为当前默认值；策划可在配置文件中调整。汇总口径见 [队伍系统 · 默认编制与人数影响](./队伍系统.md#默认编制与人数影响)。

| 队伍类型 | 编制人数（默认 = 上限） |
|----------|-------------------------|
| 侦察队 | **3** |
| 运输队 | **15** |
| 工程队 | **10** |
| 民兵 | **15** |
| 步兵 | **10** |
| 弓手 | **10** |

- 队伍人数可因战斗或事件减少，且不会立即解散（`current_headcount ≥ 1` 即可维持；降至 **0** 细则 **待定**）。
- **已定**：默认人数 = 编制上限；**人数比** = `current_headcount / 编制人数`（满编 = 上表）。

人数下降同步影响：

| 能力 | 影响方式 |
|------|----------|
| **视野范围** | 满编视野 × **人数比系数**（首版 1:1 线性） |
| **负载上限** | 满编负载 × **人数比系数** |
| **工作效率** | **参与人数 × `work_efficiency`**（见 [回合与行动表 · 工作效率与工作量](../07-玩法循环/回合与行动表.md#工作效率与工作量)） |
| **攻击效果** | 满编战力 × **人数比系数** + 交战 **人数系数**；仅武装模板 |

**移动速度**不随人数比变化；见 [单位基础属性](#单位基础属性) 与模板 `base_move_speed`。

人数比系数映射由程序表 `headcount_factor_curve` 配置；首版全线性（系数 = 人数比）。各模板满编时的视野格数、负载、战力、移速等 **SO 默认参数待定**（sy-03；[04-数值框架/](../04-数值框架/) 未建）。

### 单位模板清单（默认预设）

#### 1. 侦察队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **极快** | 迅速探索大片区域 |
| 武装状态 | **无武装** | 无法主动交战；遇敌应对见 [队伍系统 · 遭遇敌人时的行为](./队伍系统.md#遭遇敌人时的行为) |
| 人数规模 | **少** | 轻装上阵，机动性强 |
| 视野范围 | **广** | 可揭示大范围内的资源点 |

**默认能力**：
- **揭示资源点信息**：进入视野内的资源点，可揭示其**种类与储量**（即时，不走工作进度）。

#### 2. 运输队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **均衡** | 平衡速度与载重 |
| 武装状态 | **无武装** | 需要护卫保护 |
| 人数规模 | **中** | 负责运输作业 |
| 视野范围 | **一般** | 基础视野 |

**默认能力**：
- **运载物资**：可在资源点与城市之间运输资源；一般**自律行动**（按指令表自主推进，不需玩家逐步微操）。
- **载重能力**：有最大载重量限制；**不允许超载**——人数不足导致载重上限下降时，**按比例自动丢弃**超出部分的货物。
- **装货 / 卸货**：各占一项 [工作中](../07-玩法循环/回合与行动表.md#工作中状态) 任务；所需工作量对照值各 **1**；每回合增量受 [工作效率与工作量](../07-玩法循环/回合与行动表.md#工作效率与工作量) 影响。

#### 3. 工程队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **均衡** | 携带工程设备，速度适中 |
| 武装状态 | **无武装** | 需要保护 |
| 人数规模 | **大** | 需要大量人力进行建设 |
| 视野范围 | **一般** | 基础视野 |

**默认能力**（两类**工作**，均占用 [工作中](../07-玩法循环/回合与行动表.md#工作中状态) 进度）：

| 工作 | 说明 | 玩家指挥 |
|------|------|----------|
| **建造设施** | 在合法位置新建设施 | 二级选设施类型；须指定 **被工作对象**（资源点 / 工地）；**工作对象**为工程队 |
| **运维设施** | 对已存在设施运维 | **被工作对象**为目标设施；**工作对象**为工程队 |

**建造设施 · 二级选择**

- 一级：指令类型 = 建造设施。
- 二级：选择具体**设施类型**（如桥梁、资源点开采站等）；各类型对应独立 SO 配置。
- **建造条件因设施而异**，程序通过 `facility_type_config` 读取，不在代码中写死。示例：
  - **桥梁**：须满足 [地图图层 · 裂谷与桥梁](../03-图层与地点/地图图层.md#裂谷与桥梁)（裂谷两端非裂谷、直线连接）。
  - **资源点设施**（如 **矿区 / 果园 / 能源站 / 征兵办**）：须落在对应 [资源层](../03-图层与地点/地图图层.md#图层一览) 资源点，且满足储量/揭示等条件；对照建造 **3 回合**（**细则待定**，sy-10）。
- 条件不满足时，指挥阶段即不可下达或提示失败（**UI 反馈方式待定**）。

**运维设施**

- 针对**已建造**的设施执行运维工作；所需工作量对照值见 [设施层 · 运维对照工作量](../03-图层与地点/设施层.md#运维对照工作量已定)（四档 2/3/4/5）。每回合增量受 [工作效率与工作量](../07-玩法循环/回合与行动表.md#工作效率与工作量) 影响。
- 工程队侧重荒野设施与资源点设施的建造 / 运维。

**修复城区**

城区修复有三种途径，详细规则见 [分离与拆解 · 修复城区](../03-图层与地点/建筑层/分离与拆解.md#修复城区)：

| 途径 | 执行者 | 效率 | 航行中 |
|------|--------|------|:------:|
| **自主修复** | 本地人口（≤15 人） | 初始效率（1.0/人） | 可 |
| **工程队修复** | 工程队 | 满编 1.5×（1.5/人） | 否 |
| **城坞** | 城坞模块 | 被动 10%/回合·相连城区均分 + 主动一次（20 金属 + 10 能源） | — |

程序实现见 [队伍与战斗数据结构 · 工程队工作](../../03-程序设计/03-数据字典/队伍与战斗数据结构.md#工程队工作建造与运维)。

| 属性 | 数值特征 | 说明 |

| 人数规模 | **上限 10** | 须消耗 **简陋兵甲** × **1**（队伍级） |

#### 6. 弓手模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **待定** | 轻装远程队 |
| 武装状态 | **有武装** | 远程；默认启用攻击能力 |
| 人数规模 | **上限 10** | 须消耗 **弓具** × **1**（队伍级） |

**组建门槛**：**弓具** × **1** + 人口；资产可于 [简易工坊](../03-图层与地点/设施层.md#简易工坊) 打造（消耗金属）。

### 资源点揭示

资源点对玩家可见性遵循视野驱动模型：所有格子初始为**未知**；己方单位进入该格视野时，**种类与储量一并即时揭示**，不走工作进度、不分阶段。

- 进入视野 → 全部信息同时写入玩家战略地图已知层。
- 离开视野后，该格信息**停留在上次看到的状态**，不自动更新，不因离开而退回。
- 多队同时进入同一格时揭示不叠加。
- 揭示后即可建造对应采集设施（建造条件见 sy-10）。

### 他人城市与他人单位（摘要）

他人城市、他人单位须进入己方**单位视野**才可见：**城市**揭示后**记入**战略地图；**单位**仅实时可见、离开视野后不保留。权威见 [视野系统 · 他人城市与他人单位](./视野系统.md#他人城市与他人单位)。
