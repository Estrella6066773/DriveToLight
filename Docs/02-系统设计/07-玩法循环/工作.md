## 定位

**工作**是跨系统的**机制模块**（不单列为与「地图」「队伍」并列的顶层系统）：凡需占用时间、以**工作进度**推进的任务，均通过同一套工作状态与时长规则结算；到达配置进度节点时发放**工作成果**。

工作可作用于：

| 被工作对象（示例） | 工作对象（执行者，示例） | 典型工作 |
|--------------------|--------------------------|----------|
| **核心区** | **核心区** | 停泊 ↔ 航行切换（骄阳之心驱动整城离港 / 入港） |
| 资源点 / 工地 | 工程队 | 建造采集设施 |
| 设施实例 | 工程队 | 运维设施 |
| 库存节点 | 运输队 | 装货 / 卸货 |
| 城区实例 | **工程队** / **本地人口** / **城坞模块** | **修复城区**（三种途径：自主 / 工程队 / 城坞；见 [分离与拆解 · 修复](../03-图层与地点/建筑层/分离与拆解.md#修复城区)） |
| 城区实例 | 工程队 / 本地人口 | **拆解结构**（完整度换金属，见 [分离与拆解 · 拆解](../03-图层与地点/建筑层/分离与拆解.md#玩家操作拆解结构)） |
| 城区 / 连接边 | 玩家指挥 + 本地人口 | **分离城区**、**再连接**（见 [分离与拆解 · 连接与分离](../03-图层与地点/建筑层/分离与拆解.md#玩家操作连接与分离)） |
| **特殊城区**模块 | 玩家指挥 / 本地人口 / 运维队伍 | **启用 / 关闭工作区**（模块停摆，见 [运作与居民 · 工作区](../03-图层与地点/建筑层/运作与居民.md#工作区特殊城区模块--一般城区设施)）；城区无人时运维队伍停留可代行（不须满编，人数影响效率） |
| **一般城区**设施实例 | 玩家指挥 / 城区运作人力 / 运维队伍 | **设施工作区**启停（与模块**同一套** UI 与逻辑，见 [设施层 · 设施即工作区](../03-图层与地点/设施层.md#一般城区--设施即工作区)）；同上，运维队伍停留可代行 |
| **城区实例** | **工作区**运作人力 | 维持模块或设施运转（见 [城市管理系统 · 工作区](../04-资源与人口/城市管理系统.md#工作区)） |
| **城区实例** | **待组建 / 在编队伍** | **创建队伍**（城区 `form_team`）、**补员**（当前格 `reinforce_team`；人员取自相邻格，见 [补员 · 相邻格人员来源](../04-资源与人口/人口与迁移.md#补员--相邻格人员来源已定)） |

**连接 / 分离与停泊、航行**

- **连接 / 分离**：**随时**可下达；**停泊**下经工作完成且**不**因该操作扣完整度；**航行**下每执行一次，**脱离的城区** **结构完整度各 −40%**（见 [分离与拆解 · 玩家操作连接与分离](../03-图层与地点/建筑层/分离与拆解.md#玩家操作连接与分离)）。
- **停泊 ↔ 航行**：属于 **核心区**职能的多回合工作（非连接 / 分离）；见 [连接与多核心 · 停泊与航行切换](../03-图层与地点/建筑层/连接与多核心.md#停泊与航行切换核心区工作)。
- **修复城区**：三种途径——**自主修复**（本地 ≤15 人、航行中可用）、**工程队修复**（满编效率 1.5×、仅停泊）、**城坞**（被动 10%/回合·相连城区均分 + 主动 20 金属 + 10 能源、冷却 5 回合立即一次）。修复为 n 回合工作、**每回合独立触发阶段性成果**（按本回合工作量比例结算）；完整度达 100% 自动终止（不可超过）；金属不足该轮自动终止。修复期间该城区设施与工作区自动暂停。详见 [分离与拆解 · 修复城区](../03-图层与地点/建筑层/分离与拆解.md#修复城区)。

**本地人口与运维队伍（已定）**

- 城区上的设施、城区的修复 / 拆解等工作，可**直接占用该城区本地居民人口**完成，**不须**具备运维能力的队伍到场。
- 城区**无人但状态仍有效**（未变废墟）、且有**具备运维能力的队伍**停留在该城区格 → 队伍可触发该城区工作区**被动**，并可代为下达**主动**能力。
- 详见 [运作与居民 · 工作区主动能力统一状态模型](../03-图层与地点/建筑层/运作与居民.md#主动能力统一状态模型已定) 与 [运作与居民 · 本地人口直接参与工作区工作](../03-图层与地点/建筑层/运作与居民.md#本地人口直接参与工作区工作已定)。

## 与行动、指令的关系

- **行动**：任意可指挥实体（移动城市、外出队伍、具备行动资格的设施等）在 [行动表](./回合与行动表.md#行动表) 中获得回合内执行顺序。
- **指令**：玩家在指挥阶段为实体填写任务目标；其中需多回合完成的条目进入 **工作中** 状态。
- 即时能力（移动一格、单次攻击等）不走工作进度；[交战系统](../06-单位与交战/交战系统.md) 中的交战结算亦不走工作进度。

## 城市内资源迁移与城外运送（已定）

玩家在详情面板为设施或工作区**选定工作**后，系统根据该工作所需资源的位置决定执行路径：

| 资源所在位置 | 行为 | 说明 |
|-------------|------|------|
| **同一城市内**（不同城区间） | **即时扣除** | 城市共用仓库内的资源在城区间迁移走即时结算，不须队伍运送 |
| **城市外**（野外资源点、村镇等） | **自动形成需求 → 运输队运送** | 玩家下达的指令作为**需求**挂入系统，自动形成指令链——先规划运输队伍将资源运回城市，再自动衔接实施工作 |

> 城市内各城区共享**同一共用仓库**（见 [四种核心资源 · 仓储容量](../04-资源与人口/四种核心资源.md#仓储容量已定框架)），仓库中任何位置的资源均可被任一城区即时取用，无须中间运输步骤。

### 指令 vs 需求：UI 区分（已定）

玩家下达意图时，系统按条件是否已满足给出不同的操作入口：

| 条件 | 入口按钮 | 行为 |
|------|----------|------|
| **条件已满足**（所需资源在城市仓库、人口充足、工作区可用等） | **下达指令** | 直接开始工作 |
| **条件未满足**（资源不在仓库、需要外部运送等） | **下达需求** | 系统自动挂起此项，转入筹备阶段——自动生成运输/采购等前置任务，条件满足后自动衔接并执行 |

两种按钮在 UI 上须有**明显视觉差异**（如颜色、图标、文案），以提示玩家当前是即时指令还是自动筹备链。

> 此规则属 [需求驱动指挥（工作计划）](../01-核心体验/平台与操作.md#需求驱动指挥工作计划) 体系（sy-37）。

## 工作进度与工作成果

工作拆成两个概念：

| 概念 | 说明 |
|------|------|
| **工作进度** | 一项工作进行到哪一步；UI 常以百分比表示（程序字段见 [完成度](./回合与行动表.md#完成度与推进)） |
| **工作成果** | 工作进度到达**指定阈值**时结算的回馈；由工作类型配置 |

### 一般规则

- 每类工作在 `work_type_config` 上配置一条或多条 **工作成果**，每条绑定一个 **进度阈值**（如 `0.5`、`1.0`）及 **阶段性成果重置策略**（见 [阶段性成果与重置策略](#阶段性成果与重置策略)）。
- 工作进度**首次达到**某阈值时，按该条成果的策略结算**一次**（或按策略允许再次结算）。
- **一般工作**仅配置 **100%** 阈值：进度满时发放该工作的完整成果（如设施建成、装货完成、停泊切换完成）。
- 工作进行中只累积进度；**不**在未满阈值时提前发放最终成果，除非该阈值已在配置中声明。

### 阶段性成果与重置策略

每条工作成果（`work_outcomes` 子项）配置 **`outcome_repeat_policy`**：

| 策略 | 含义 |
|------|------|
| **可重置**（`repeatable`） | 工作**中断并重启**同类任务后，进度再次达到该阈值时，**允许**再次结算此条成果 |
| **不可重置**（`once_per_subject`） | 该 **被工作对象** 上此成果**一旦结算过**，即使后续进度从 0 重新累计并再次达到阈值，**也不再**发放 |

- 已结算记录挂在 **被工作对象**（如 `work_subject_state.granted_outcome_ids` 或等价结构，程序字段 **待定**）。
- 其它工作类型是否配置中途成果，由各类 `work_type_config` 决定，**不**默认存在 50% 节点。

## 工作标签与中断（已定）

完整字段与程序入口见 [回合与行动表 · 工作中断与恢复](./回合与行动表.md#工作中断与恢复)。

### 工作类型不绑定执行者

- **工作类型**只约束 **被工作对象**（`work_subject_kind`），**不**锁定 **工作对象**（执行者）。
- 草稿「**不绑定实施者**」指：进度与成果挂在**被工作对象 / 工作槽位**上，**任意合格**工作对象均可推进；每次推进仍须归属某一 `work_actor_ref`。
- 与正式「必有执行者」**不矛盾**：每回合推进仍由具体实体执行；**可暂停**类工作允许暂停后**换队续作**。

### 中断行为

每项工作类型配置一个 **`interrupt_behavior`** 数值字段，用单一数值区分三种中断处置：

| 值 | 含义 | 示例 |
|----|------|------|
| **-1** | 不可暂停：中断后进度**归零**，须重新开始 | 食物生产工作 |
| **0** | 可暂停：中断后**保留**进度，可换执行者续作 | 金属收集 |
| **n（> 0）** | 需维持：中断后每回合**衰减 n** 进度，衰减至 0 后自动结束 | 能量收集（如 `interrupt_behavior=3` 表示每回合流失 3 进度） |

**三种行为对比**：

| 中断后 | -1（不可暂停） | 0（可暂停） | n > 0（需维持） |
|--------|----------------|-------------|-----------------|
| **进度** | **重置为 0** | **保留** | **逐渐衰减**（每回合 −n） |
| **换执行者** | 不适用（进度已归零） | **允许** | **允许**（衰减中可接续） |
| **持久化** | 按类型配置 | 建议 `persist_progress_on_subject=true` | 建议 `persist_progress_on_subject=true` |

### 精密工作标签

**`is_precision_work`**（精密工作）标签独立于中断行为标签，决定工作**何时被中断**：

| 条件 | 精密工作 | 非精密工作 |
|------|----------|------------|
| **被攻击时** | **停止** | 不因此中断（仍可受其他中断原因影响） |
| **城市航行移动时** | **停止** | 不因此中断 |

> 精密工作是触发中断的**附加条件**。中断发生后，进度处置由 [`interrupt_behavior`](#中断行为) 决定（-1→归零 / 0→保留 / n>0→衰减）。

**精密工作示例**：需要精确操作或无法在移动环境中维持的生产工作。

### 中断处置

每项工作类型配置 **`on_interrupt_disposition`**（意外 / 行动 / 主动中止等**中断**时生效）：

| 值 | 含义 |
|----|------|
| **待重试**（`retry`） | 保留工作条目（常为 `suspended`）；可恢复或由**另一工作对象**接续（须 `interrupt_behavior ≥ 0`） |
| **已结束**（`ended`） | **清除**本条工作事件 / 指令条目；不保留「进行中」占位 |

- `interrupt_behavior=-1` + 中断：进度必归零；处置为 **已结束** 或 **待重试** 由 SO 配置。
- `interrupt_behavior=0` + **待重试**：意外 / 行动打断后同条目或换队续作（见 [工作中断与恢复](./回合与行动表.md#工作中断与恢复)）。
- `interrupt_behavior=0` + **已结束**：中断即结束本次工作会话，下次重新下达时从 0 起算。
- `interrupt_behavior>0` + **待重试**：条目保留，每回合流失 `interrupt_behavior` 进度；衰减至 0 后自动转为 **已结束**。

## 移动城市与城区工作

城内工作分两类口径：

| 口径 | 说明 | 谁管理 |
|------|------|--------|
| **实体工作** | 工程队对指定城区修复、工程队运维设施、城内生产等多回合 [工作进度](#工作进度与工作成果) 任务 | 玩家通过 [行动表](./回合与行动表.md) 下达指令 |
| **城区运作** | 维持各城区供能与模块 / 设施运转所需的**劳动力** | **玩家城市**：为每城区指定由**哪一类人口**承担（见 [人口与迁移 · 玩家：城区工作分配](../04-资源与人口/人口与迁移.md#玩家城区工作分配)）；**非玩家城市**：由城市领袖抽象统管 |

- 城区上显示的**居民**人数**不**自动计入运作人力；运作不足时可导致城区停摆（与 [城市模块化](../03-图层与地点/建筑层/运作与居民.md#城区运作与居民人口) 联动，细则 **待定**）。
- 外出队伍执行的荒野工作（建造采集设施、运维设施等）走队伍 **工作对象** 规则，与城区运作人力**分开**结算。

## 核心规则（摘要）

完整字段、默认时长、中断与恢复见 [回合与行动表 · 工作中状态](./回合与行动表.md#工作中状态)。

| 概念 | 说明 |
|------|------|
| **工作对象** | 执行工作的实体（队伍、**核心区**、移动城市等） |
| **被工作对象** | 工作作用在其上的实体；可与工作对象相同 |
| **所需工作量** | `work_type_config.base_duration_turn`（对照额度） |
| **本回合增量** | 参与人数 × 工作效率；参与人数 = `max(1, current_headcount)` |
| **工作进度** | 已积累工作量 / 所需工作量；达阈值时检查并发放 **工作成果** |
| **工作成果** | 工作类型 SO 上按进度阈值配置的效果列表（`work_outcomes` 或等价结构，**待定**） |
| **中断行为** | `interrupt_behavior` — **-1**=不可暂停（进度归零）、**0**=可暂停（保留进度）、**n>0**=需维持（每回合衰减 n） |
| **精密工作** | `is_precision_work` — 被攻击或城市航行移动时触发中断 |
| **中断处置** | `on_interrupt_disposition`：**待重试** / **已结束** |
| **阶段性成果** | `work_outcomes` + 每条 `outcome_repeat_policy`：**可重置** / **不可重置** |
