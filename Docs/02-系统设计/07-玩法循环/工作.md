## 定位

**工作**是跨系统的**机制模块**（不单列为与「地图」「队伍」并列的顶层系统）：凡需占用时间、以**工作进度**推进的任务，均通过同一套工作状态与时长规则结算；到达配置进度节点时发放**工作成果**。

工作可作用于：

| 被工作对象（示例） | 工作对象（执行者，示例） | 典型工作 |
|--------------------|--------------------------|----------|
| **核心区** | **核心区** | 停泊 ↔ 航行切换（骄阳之心驱动整城离港 / 入港） |
| 资源点 / 工地 | 工程队 | 建造采集设施 |
| 资源点 | 勘探队 | 勘探（储量调查） |
| 设施实例 | 工程队 | 运维设施 |
| 库存节点 | 运输队 | 装货 / 卸货 |
| 城区实例 | **工程队** | **修复城区**（**被工作对象** = 指定城区；废墟 → 正常；**仅停泊**，见 [分离与拆解 · 修复](../03-图层与地点/建筑层/分离与拆解.md#修复城区)） |
| 城区实例 | 工程队 / 玩家指挥 | **拆解结构**（完整度换金属，见 [分离与拆解 · 拆解](../03-图层与地点/建筑层/分离与拆解.md#玩家操作拆解结构)） |
| 城区 / 连接边 | 玩家指挥 + 工程 | **分离城区**、**再连接**（见 [分离与拆解 · 连接与分离](../03-图层与地点/建筑层/分离与拆解.md#玩家操作连接与分离)） |
| **特殊城区**模块 | 玩家指挥 | **启用 / 关闭工作区**（模块停摆，见 [运作与居民 · 工作区](../03-图层与地点/建筑层/运作与居民.md#工作区特殊城区模块--一般城区设施)） |
| **一般城区**设施实例 | 玩家指挥 / 城区运作人力 | **设施工作区**启停（与模块**同一套** UI 与逻辑，见 [设施层 · 设施即工作区](../03-图层与地点/设施层.md#一般城区--设施即工作区)） |
| **城区实例** | **工作区**运作人力 | 维持模块或设施运转（见 [城市管理系统 · 工作区](../04-资源与人口/城市管理系统.md#工作区)） |
| **城区实例** | **待组建 / 在编队伍** | **创建队伍**（城区 `form_team`）、**补员**（当前格 `reinforce_team`；人员取自相邻格，见 [补员 · 相邻格人员来源](../04-资源与人口/人口与迁移.md#补员--相邻格人员来源已定)） |

**连接 / 分离与停泊、航行**

- **连接 / 分离**：**随时**可下达；**停泊**下经工作完成且**不**因该操作扣完整度；**航行**下每执行一次参与城区 **结构完整度各 −40%**（见 [分离与拆解 · 玩家操作连接与分离](../03-图层与地点/建筑层/分离与拆解.md#玩家操作连接与分离)）。
- **停泊 ↔ 航行**：属于 **核心区**职能的多回合工作（非连接 / 分离）；见 [连接与多核心 · 停泊与航行切换](../03-图层与地点/建筑层/连接与多核心.md#停泊与航行切换核心区工作)。
- **修复城区**：**工程队**对**指定城区**展开多回合工作（`work_subject_kind = district`）；移动城市 **航行**态 **不可**下达，须 **停泊**（见 [地图与移动 · 航行态](../02-地图与世界/地图与移动.md#航行态占格单位边界与禁用功能-sy-19)）。目标城区邻近 [城坞](../03-图层与地点/建筑层/运作与居民.md#功能模块城坞通讯站等) 时可获效率加成，**不**改变执行者为工程队。

## 与行动、指令的关系

- **行动**：任意可指挥实体（移动城市、外出队伍、具备行动资格的设施等）在 [行动表](./回合与行动表.md#行动表) 中获得回合内执行顺序。
- **指令**：玩家在指挥阶段为实体填写任务目标；其中需多回合完成的条目进入 **工作中** 状态。
- 即时能力（移动一格、单次攻击等）不走工作进度；[交战系统](../06-单位与交战/交战系统.md) 中的交战结算亦不走工作进度。

## 工作进度与工作成果

工作拆成两个概念：

| 概念 | 说明 |
|------|------|
| **工作进度** | 一项工作进行到哪一步；UI 常以百分比表示（程序字段见 [完成度](./回合与行动表.md#完成度与推进)） |
| **工作成果** | 工作进度到达**指定阈值**时结算的回馈；由工作类型配置 |

### 一般规则

- 每类工作在 `work_type_config` 上配置一条或多条 **工作成果**，每条绑定一个 **进度阈值**（如 `0.5`、`1.0`）及 **阶段性成果重置策略**（见 [阶段性成果与重置策略](#阶段性成果与重置策略)）。
- 工作进度**首次达到**某阈值时，按该条成果的策略结算**一次**（或按策略允许再次结算）。
- **一般工作**仅配置 **100%** 阈值：进度满时发放该工作的完整成果（如设施建成、装货完成、停泊切换完成）。
- 工作进行中只累积进度；**不**在未满阈值时提前发放最终成果，除非该阈值已在配置中声明。

### 阶段性成果与重置策略

每条工作成果（`work_outcomes` 子项）配置 **`outcome_repeat_policy`**：

| 策略 | 含义 |
|------|------|
| **可重置**（`repeatable`） | 工作**中断并重启**同类任务后，进度再次达到该阈值时，**允许**再次结算此条成果 |
| **不可重置**（`once_per_subject`） | 该 **被工作对象** 上此成果**一旦结算过**，即使后续进度从 0 重新累计并再次达到阈值，**也不再**发放 |

- 已结算记录挂在 **被工作对象**（如 `work_subject_state.granted_outcome_ids` 或等价结构，程序字段 **待定**）。
- **可重置** 常用于勘探等：中断后新开一项同类工作，仍可在 50% 再次获得阶段性揭示（与资源点当前揭示等级取较高、不降级规则并行，见 [资源点揭示等级](../06-单位与交战/单位类型与视野.md#资源点揭示等级)）。
- **不可重置** 常用于破译等：中断前进度归零，已领取的阶段性回报**不**因再次推进而重复发放。

### 特例：勘探与分级揭示

[勘探队](../06-单位与交战/单位类型与视野.md#2-勘探队模板) 对资源点执行 **勘探** 类工作时，可在进度中途发放阶段性成果，与 [资源点揭示等级](../06-单位与交战/单位类型与视野.md#资源点揭示等级) 联动：

| 进度阈值 | 工作成果（示例） | `outcome_repeat_policy` | 说明 |
|----------|------------------|-------------------------|------|
| **50%** | **至少揭示种类** | **可重置** | 若当前为 `hidden`，提升至 **`kind`**；已 `kind` 或 `amount` 时不降级 |
| **100%** | **揭示储量** | **待定**（建议 `once_per_subject` 或与揭示等级合并取高） | 等级 **`amount`** |

**勘探中断示例（已定）**：勘探队推进至 **50%** 后中断勘探并移动 → 本条工作按 **`on_interrupt_disposition=ended`** 视为**已结束**，清除当前工作事件；下次对同一资源点重新发起勘探时，进度从 0 累计，再次到达 **50%** 时可**再次**结算 50% 阶段性成果（**可重置**）。

- 侦察队通过**视野能力**即时提升至 **`kind`**；勘探队通过**勘探工作**按上表推进；二者取**较高**等级，**不降级**。
- 其它工作类型是否配置中途成果，由各类 `work_type_config` 决定，**不**默认存在 50% 节点。

### 特例：设施破译（示例）

需多阶段回报的**破译**类工作（`work_type` **待定**）典型配置：

| 项 | 口径 |
|----|------|
| **`allow_suspend`** | **`false`**（**不可暂停**）· 中断则**进度重置为 0** |
| **阶段性成果** | 进度中途可配置多档回报 |
| **`outcome_repeat_policy`** | **不可重置** · 已领取的阶段性回报不因再次推进而重复发放 |
| **中断后** | 进度归零；已结算成果保留为已领取；新推进至相同阈值**不**再发放 |

## 工作对象、可暂停与中断（已定）

完整字段与程序入口见 [回合与行动表 · 工作中断与恢复](./回合与行动表.md#工作中断与恢复)。

### 工作类型不绑定执行者

- **工作类型**只约束 **被工作对象**（`work_subject_kind`），**不**锁定 **工作对象**（执行者）。
- 草稿「**不绑定实施者**」指：进度与成果挂在**被工作对象 / 工作槽位**上，**任意合格**工作对象均可推进；每次推进仍须归属某一 `work_actor_ref`。
- 与正式「必有执行者」**不矛盾**：每回合推进仍由具体实体执行；**可暂停**类工作允许暂停后**换队续作**。

### 可暂停与不可暂停

由 `work_type_config.allow_suspend` 区分（与草稿「可暂停 / 不可暂停」对齐）：

| | **可暂停**（`allow_suspend=true`） | **不可暂停**（`allow_suspend=false`） |
|---|-----------------------------------|----------------------------------------|
| **中断后进度** | **保留** | **重置为 0** |
| **换执行者** | **允许** · 暂停后另一合格单位可接续同一被工作对象上的进度 | 不适用同一进度接续（任务已重置） |
| **持久化** | 建议 `persist_progress_on_subject=true` | 按类型配置 |

### 中断处置

每项工作类型配置 **`on_interrupt_disposition`**（意外 / 行动 / 主动中止等**中断**时生效）：

| 值 | 含义 |
|----|------|
| **待重试**（`retry`） | 保留工作条目（常为 `suspended`）；可恢复或由**另一工作对象**接续（须 **可暂停**） |
| **已结束**（`ended`） | **清除**本条工作事件 / 指令条目；不保留「进行中」占位 |

- **不可暂停** + 中断：进度必归零；处置为 **已结束** 或 **待重试** 由 SO 配置（破译等多用 **已结束** + 成果 **不可重置**）。
- **可暂停** + **待重试**：意外 / 行动打断后同条目或换队续作（见 [工作中断与恢复](./回合与行动表.md#工作中断与恢复)）。
- **可暂停** + **已结束**：如勘探 — 中断即结束本次工作会话，下次重新下达时从 0 起算，阶段性成果按 **可重置** 策略可再次发放。

## 移动城市与城区工作

城内工作分两类口径：

| 口径 | 说明 | 谁管理 |
|------|------|--------|
| **实体工作** | 工程队对指定城区修复、工程队运维设施、城内生产等多回合 [工作进度](#工作进度与工作成果) 任务 | 玩家通过 [行动表](./回合与行动表.md) 下达指令 |
| **城区运作** | 维持各城区供能与模块 / 设施运转所需的**劳动力** | **玩家城市**：为每城区指定由**哪一类人口**承担（见 [人口与迁移 · 玩家：城区工作分配](../04-资源与人口/人口与迁移.md#玩家城区工作分配)）；**非玩家城市**：由城市领袖抽象统管 |

- 城区上显示的**居民**人数**不**自动计入运作人力；运作不足时可导致城区停摆（与 [城市模块化](../03-图层与地点/建筑层/运作与居民.md#城区运作与居民人口) 联动，细则 **待定**）。
- 外出队伍执行的荒野工作（建造采集设施、勘探等）走队伍 **工作对象** 规则，与城区运作人力**分开**结算。

## 核心规则（摘要）

完整字段、默认时长、中断与恢复见 [回合与行动表 · 工作中状态](./回合与行动表.md#工作中状态)。

| 概念 | 说明 |
|------|------|
| **工作对象** | 执行工作的实体（队伍、**核心区**、移动城市等） |
| **被工作对象** | 工作作用在其上的实体；可与工作对象相同 |
| **所需工作量** | `work_type_config.base_duration_turn`（对照额度） |
| **本回合增量** | 参与人数 × 工作效率；参与人数 = `max(1, current_headcount)` |
| **工作进度** | 已积累工作量 / 所需工作量；达阈值时检查并发放 **工作成果** |
| **工作成果** | 工作类型 SO 上按进度阈值配置的效果列表（`work_outcomes` 或等价结构，**待定**） |
| **可暂停** | `allow_suspend`：**可暂停**保留进度、允许换执行者；**不可暂停**则中断后进度归零 |
| **中断处置** | `on_interrupt_disposition`：**待重试** / **已结束** |
| **阶段性成果** | `work_outcomes` + 每条 `outcome_repeat_policy`：**可重置** / **不可重置** |
