## 目标



定义正六边形格子上多层内容的组成方式、各图层的堆叠规则，以及单位与环境在他人行动阶段的响应机制。



## 范围



- **包含**：图层类型、默认可叠加规则、各图层堆叠逻辑、图层间影响规则与优先级、响应触发条件与示例。

- **不包含**：影响规则的具体数值、完整规则表、程序实现。



## 详细说明



### 基本原则



- 一个格子可同时含有多个图层的内容。

- 一般情况下，不同图层的内容可在同一格内叠加。

- 在此基础之上，存在图层之间的**影响逻辑**；各图层内部也有各自的重叠逻辑。



### 地形层与环境层的分工



两层的划分以**是否描述地面形状**为准，而非是否「自然形成」：



| 图层 | 职责 | 典型内容 | 变更特征 |
|------|------|----------|----------|
| **地形层** | 格子的**地面形状**与地貌类型：决定通过性、占格、桥梁/裂隙等空间规则 | 平原、丘陵、山地、河流、裂隙、沼泽等 | 每格一种，作为底层；通常随地图生成确定，较少因短期行为改变 |
| **环境层** | 格子上的**表层状况**与**大气/区域效果**：地面植被与覆盖、灾害天气、移动城市轨迹等 | 树林、焦土、弹坑；沙暴、暴雨、迷雾；移动城市轨迹等 | 可随时间或单位行为产生、变更或消退（存续时间可长可短） |



- **树林、密林**等植被覆盖属于**环境层**（表层状况），不属于地形层；同一格可同时有丘陵（地形）与密林（环境）。
- **天气类**效果（沙暴、暴雨、迷雾等）与环境层中的**地面表层**效果（焦土、弹坑等）同层叠放，各自类型自有重叠逻辑。
- 旧稿将密林列在地形层的写法**废止**；影响规则中凡「密林削弱视野」类条目，触发条件改为**环境层**。



### 图层间影响逻辑



图层之间默认可堆叠；当多种图层内容同时存在时，按**影响规则**修正互动结果。



**优先级（必须）**



- 每一条影响规则**必须**绑定独立的优先级。

- 结算时按优先级顺序依次判定；优先级数值越小，越先结算。

- 即使两条规则对调顺序不会改变最终结果（例如密林与丘陵对哨塔视野的叠加），每条规则仍须各自持有优先级，不得省略。



### 影响规则示例



| 优先级 | 触发条件 | 效果 |

|--------|----------|------|

| 10 | 地形层为**裂隙** | 该格**不可**在设施层建立任何设施 |

| 20 | 地形层为**裂隙**，且存在任意一对**对向格**（裂隙两侧相对的两格），两格的地形均**非裂隙** | 允许在该裂隙上建立**桥梁**类设施 |

| 30 | 设施层存在**哨塔**，环境层为**密林**（或等效的树林/植被覆盖） | 削弱哨塔视野 |

| 40 | 设施层存在**哨塔**，地形层为**丘陵** | 强化哨塔视野（程度待定） |

| 50 | 设施层存在**哨塔**，地形层为**山脉** | 强化哨塔视野（程度高于丘陵，数值待定） |



说明：



- 优先级 10 与 20 同时判定时：先排除一般设施，再评估是否满足桥梁例外。

- 优先级 30、40、50 同时作用于同一哨塔时，按序依次结算；具体叠加方式（累加 / 取极值等）待定，但各条规则的优先级不可合并省略。



### 图层一览



由下至上，典型叠放顺序如下：



| 图层 | 内容 | 堆叠规则 |

|------|------|----------|

| 地形层 | **地面形状**：平原、丘陵、山地、河流、沼泽、裂隙等（见 [地形层与环境层的分工](#地形层与环境层的分工)） | 每格一种，作为底层 |

| 环境层 | **表层状况与区域效果**：树林、焦土、弹坑；沙暴、暴雨、迷雾；移动城市轨迹等 | 各环境类型自有重叠逻辑 |

| 资源层 | 资源点（村镇、矿藏、果地、遗迹）及**储量**；可建立对应**采集设施** | 不堆叠同类；多种资源点可并存；统计储量 |

| 建筑层 | **城区**（见 [建筑层与城区](#建筑层与城区)） | 不可堆叠 |

| 设施层 | 功能设施：**采集设施**（征兵办、矿区、果园、能源站等）、哨塔、驿站、桥梁、陷阱等 | 堆叠规则受已有设施影响 |

| 物品层 | 可被单位拾取的物品 | 堆叠上限受其他图层影响 |

| 单位层 | 队伍单位 | 默认不可堆叠；部分设施/建筑可容纳多个单位 |



### 资源层与采集设施

- **资源层**格含 **储量** 与资源点类型（见 [荒野地点 · 资源点与采集设施](../04-资源与人口/荒野地点.md#资源点与采集设施)）。
- **采集设施**建于资源点，通过 [工作](../07-玩法循环/工作.md) 将储量转化为物品层资产；类型包括 **征兵办**、**矿区**、**果园**、**能源站**。
- **遗迹**（资源层）：**前文明遗迹**，**不可重建**；储量产出能源，仅可建 **能源站** 等采集设施。
- **废墟**不是独立图层对象，而是**城区**的一种**状态**（见 [建筑层与城区](#建筑层与城区)）；与资源层 **遗迹** 的区分见 [荒野地点 · 遗迹与废墟](../04-资源与人口/荒野地点.md#遗迹与废墟术语强制区分)。

### 建筑层与城区

**建筑层存储的内容就是城区**——二者划等号，不另设与城区平行的「城市建筑类型」。

| 字段 | 说明 |
|------|------|
| **对象** | **城区**：规则见 [城市模块化](../03-图层与地点/建筑层/README.md) |
| **类别** | **特殊城区** / **一般城区**（正交于状态） |
| **状态** | **正常** / **废墟** |
| **堆叠** | 每格至多一个城区实例 |

- **正常**：按类别提供能力；可承载人口（若配置允许）。
- **废墟**：被击毁或默认刷新的损毁城区；玩家须能**认出**其为废墟；**结构完整度 低于 20%** 强制转废墟；**无法工作**；**不可迁入**、已有居民**仅可迁出**；**须修复为正常**后机制上才与正常城区同等对待。
- **外部城市**、玩家**移动城市**占格：由一格或多格**城区**组成或挂载于建筑层；移动城市的城区连接关系另见 [城市模块化](../03-图层与地点/建筑层/README.md) 与 [地图与移动](./地图与移动.md)。

### 图层与地点类型的关系

- **建筑层**上的城区、外部城市占格，与 [荒野地点 · 建筑层城区](../04-资源与人口/荒野地点.md#建筑层城区) 对应；玩家**移动城市**见 [城市模块化](../03-图层与地点/建筑层/README.md)。
- **设施层**中的采集设施、驿站等，与 [探索与扩张](./探索与扩张.md) 中的建设行为对应。

- **资源层**与 [四种核心资源](../04-资源与人口/四种核心资源.md) 的产出来源相关。



### 响应



环境或单位可具备**响应**机制：



- 在**其他单位**的[行动阶段](../07-玩法循环/回合与行动表.md#回合阶段)内，若有符合条件的单位执行了符合条件的行动，则触发该环境或单位的响应行为。

- 响应在目标单位的行动环节内即时结算，可打断其当前行动。



**示例（陷阱）**



- 陷阱属于设施层的一种设施。

- 某单位在自己的行动环节内，从陷阱**相邻格**移动进入陷阱所在格时，陷阱响应：

  1. 打断目标单位行动；

  2. 对目标造成伤害；

  3. 消耗该陷阱的耐久。
