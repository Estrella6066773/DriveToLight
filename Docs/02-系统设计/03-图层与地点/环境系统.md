## 目标

定义环境系统的运行时框架：各图层的环境效果如何挂载、太阳照射区与光带如何推移、跨图层修正规则如何结算、章节环境规则如何切换。

## 范围

- **包含**：环境效果（天气、地面表层）的配置与结算管线；太阳/光带（黄昏带、暗渊带）的运行时模型；跨图层修正规则（`cross_modifiers`）的结算框架；章节级环境规则配置切换。
- **不包含**：各地形类型的具体数值（属 [地图图层 · 地形类型清单](./地图图层.md#地形类型清单)）；设施建造条件（属 [设施层](./设施层.md)）；资源点揭示规则（属 [单位类型与视野](../06-单位与交战/单位类型与视野.md)）；太阳具体 v₀、a、带宽度（sy-01）。

## 详细说明

### 系统定位

环境系统是 **回合制运行时引擎**的组成部分，在 [环境行动阶段](../07-玩法循环/回合与行动表.md#回合阶段) 结算：

| 职责 | 说明 |
|------|------|
| **太阳/光带推移** | 每回合（或按格修正周期）更新光带位置；全图暗渊时停用 |
| **环境效果结算** | 天气、地面表层等挂载在格子上的效果逐格/逐区域结算 |
| **跨图层修正** | 按优先级依次判定各条 `cross_modifiers` 规则 |
| **响应检测触发** | 环境变化作为检测条件，经响应检测器触发行为通道 |
| **章节环境配置** | 按章节切换环境规则集（日照启用/停用、压力来源切换等） |

### 模块结构

环境系统划分为四个子模块：

| 子模块 | 职责 | 结算阶段 |
|--------|------|----------|
| **光带引擎** | 太阳照射区推移、黄昏带/暗渊带格级标记、`sun_motion_enabled` | 环境行动 · 第一步 |
| **环境效果结算器** | 天气循环、地面表层效果到期/刷新、移动城市轨迹管理 | 环境行动 · 第二步（光带之后） |
| **跨图层修正器** | 按 `L0_cross_modifiers.csv` 逐条判定并写入修正值 | 环境行动 · 第三步 |
| **章节环境开关** | 按章节切换环境配置集；控制日照启用/停用、天气可用列表等 | 章节切换时触发 |

### 光带引擎

#### 太阳照射区与光带模型

玩法层使用 **黄昏带**、**暗渊带** 表示太阳照明前沿在地图上的位置；与设定层民众口中的 [暗渊](../../04-设定/03-地点与场景/暗渊.md) 区分（详见 [地图与移动 · 黄昏带与暗渊带](../02-地图与世界/地图与移动.md#黄昏带与暗渊带玩法)）。

太阳沿卷轴**纵向**推移，采用 `v(t)=v₀+a·t` 运动学模型（**已定**），经格修正工具（xy 执行层）每环境回合结算一次。

#### 运行时状态

每回合环境行动结算时，光带引擎按以下顺序执行：

```
1. 读取当前 t，计算 v(t) = v₀ + a·t
2. 格残差 accum += v(t)
3. step_cells = floor(accum)
4. 向上推移光带 step_cells 格
5. accum -= step_cells
6. 检测照射区是否完全移出地图边界 → 若是，sun_motion_enabled = false
7. t += 1
```

#### 光带状态枚举

每个格子按其在光带中的位置，属于以下之一：

| 光带状态 | 含义 | 格级效果 |
|----------|------|----------|
| **日光带** | 太阳直接照射区 | 无额外惩罚 |
| **黄昏带** | 太阳余晖前沿过渡带 | 轻度 debuff（具体 sy-01） |
| **暗渊带** | 已脱离日照 | 重度 debuff（具体 sy-01） |
| **全图暗渊** | `sun_motion_enabled=false`，全局均属暗渊带 | 常驻 debuff（具体 sy-01） |

### 环境效果结算器

#### 效果类型

环境层的效果分为三类，均挂载于格子环境层：

| 类别 | 子类型 | 持续时间 | 变更特征 |
|------|--------|----------|----------|
| **地面表层** | 树林、密林、焦土、弹坑 | 树林默认持续；焦土/弹坑按配置回合到期 | 可因单位行为产生或变更 |
| **天气** | 沙暴、暴雨、迷雾 | 按配置回合持续后刷新或消退 | 区域级，影响多格 |
| **城市轨迹** | 移动城市轨迹 | 按配置回合到期 | 航行经过时产生 |

#### 结算管线

```
foreach 格子（按需，非全图）：
  1. 地面表层效果到期检测 → 移除到期项
  2. 天气区域到期检测 → 刷新或移除
  3. 城市轨迹到期检测 → 移除
  4. 新触发条件检测（单位行为引发的地面变更等）
```

环境效果通过 [GAS-lite 能力管线](../../03-程序设计/02-运行时逻辑/Effect与能力解析.md) 挂载于格子 ASC（`AbilitySystemComponent`）：效果配置走 `GameplayEffectConfigSO`，程序只管管道开关，不硬编码 `if (weather==sandstorm)` 类分支。

#### 天气系统（首版框架）

三种天气类型枚举已定（sandstorm、heavy_rain、fog）；首版仅保留**区域级**天气：

| 天气 | 效果（待定） | 影响范围 |
|------|-------------|----------|
| **沙暴** | 单位视野下降、路径消耗增加 | 区域（多格） |
| **暴雨** | 单位路径消耗增加、部分地形通行性变化 | 区域（多格） |
| **迷雾** | 单位视野大幅下降 | 区域（多格） |

具体影响数值走 GE/GA 配置；天气刷新与持续时间走 [L2_environment_defs.csv](../../03-程序设计/03-数据字典/tables/L2_environment_defs.csv)。

### 跨图层修正器

跨图层修正规则（`cross_modifiers`）在**环境结算的第三步**执行，按**优先级**依次判定：

| 优先级 | 触发条件 | 修正目标 | 效果 |
|--------|----------|----------|------|
| 30 | 环境层：密林 + 设施层：哨塔 | 哨塔视野加成 | **−1 格** |
| 40 | 地形层：丘陵 + 设施层：哨塔 | 哨塔视野加成 | **+1 格** |
| 50 | 地形层：山地 + 设施层：哨塔 | 哨塔视野加成 | **+2 格** |

结算框架：每条规则按 `(when_layer, when_id)` 匹配 → 修正目标 `(require_layer, require_id)` → 写入 `stat_key` 的 `op` 修正值。程序不写死具体规则，走 CSV 驱动（`L0_cross_modifiers.csv`）。

### 章节环境规则

章节切换时，环境系统加载对应当前章节的环境配置集：

| 阶段 | 光照 | 太阳移动 | 天气 |
|------|------|----------|------|
| **第一、二章** | 存在日照/黄昏带/暗渊带梯度 | **启用** `v(t)=v₀+a·t` | 可用 |
| **第三章前期** | 最后余晖消退过渡 | **启用**→照射区移出地图后停用 | 可用 |
| **第三章主体及以后** | 全局暗渊带 | **停用** `sun_motion_enabled=false` | 可用（无日照相关天气可选） |

环境配置集为 ScriptableObject（`ChapterEnvironmentProfileSO`），包含：

- `sun_motion_enabled`
- `v₀`、`a`（若启用）
- 可用天气白名单
- 光带宽度（黄昏带、暗渊带过渡宽度，sy-01）

### 与环境行动阶段的衔接

环境系统各子模块在 [环境行动阶段](../07-玩法循环/回合与行动表.md#回合阶段) 内的结算顺序：

```
环境行动（EnvironmentAction）：
  1. 光带引擎：太阳/光带推移
  2. 环境效果结算器：天气/地面表层/城市轨迹
  3. 跨图层修正器：cross_modifiers 规则
  4. 周总结钩子（turn % 7 == 0 时）
```
