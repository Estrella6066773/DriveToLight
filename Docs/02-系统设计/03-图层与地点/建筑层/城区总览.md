# 城区总览

## 术语

- **[移动城市](../../02-地图与世界/地图与移动.md)**即**玩家主城**；地图上仅此一座可整体移动。
- **城区**：见下节；与 [地图图层 · 建筑层](../地图图层.md#建筑层与城区)**划等号**——建筑层格上存储的内容就是城区这一机制对象，不再另设平行的「建筑类型」。
- **[骄阳之心（反应堆）](../../../04-设定/05-隐秘真相/骄阳之心.md)**：驱动城市的能源核心，在游戏机制中体现为**核心区**（特殊城区的一种）。
- **核心区**：拥有骄阳之心的城区，是城市的能源心脏。
- **移动城区**：与核心区保持连接、属于移动城市一部分的城区。
- 荒野格上的城区、外部城市占格等，同为建筑层上的城区实例；未接入玩家移动城市连接网络的，见 [荒野地点](../../04-资源与人口/荒野地点.md)。

## 城区：定义与资产定位

- **城区**是 [建筑层](../地图图层.md#建筑层与城区) 上的核心内容：每格建筑层至多一个城区实例（不可堆叠）。
- 玩法定位类似《杀戮尖塔》中的**遗物**：具有长期战略价值；**类别**决定能力形态，**状态**决定当前是否可用。
- **不可通过常规建造获得**：玩家不能像搭设施一样在空地「新建」一座城区；获得新城区须通过探索、事件、贸易、修复废墟、拆解回收或剧情等**稀有途径**（细则 **待定**，见 [待细化追踪 · sy-14](../../../00-规范/待细化追踪-系统.md#当前开放项)）。
- 玩家可**改造、修复、拆解、再连接**已有城区，但不可凭空创造城区**类别**。

## 城区配置属性（骨架）

与草稿 `n49`（住宅 / 能力 / **负载成本**）对齐：下列为**城区类别 SO**（`district_type_config` 或等价 ScriptableObject **待定**）上的配置维度；**数值与完整名单**可后续追加，**负载成本规则已定**。

| 属性 | 说明 | 文档 |
|------|------|------|
| **住宅（居民承载）** | **基础承载**（每城区必有）+ **屋舍**设施叠加；见 [§居民承载](#居民承载) | [人口与迁移 · 城区承载](../../04-资源与人口/人口与迁移.md#城区承载与居民安置)、[设施层 · 屋舍](../设施层.md#屋舍) |
| **城区能力** | **仅特殊城区**；[可激活](运作与居民.md#城区能力激活) 的模块功能 | [运作与居民 · 城区能力激活](运作与居民.md#城区能力激活)（规则已定；清单 sy-25） |
| **负载成本** | 移动城市**航行**时，该城区随整城移动而累积的结构损伤 | 见 [§负载成本](#负载成本) |

### 居民承载

**每个城区**（特殊 / 一般均可，状态**正常**）在 `district_type_config` 上配置 **基础居民承载**（`base_resident_cap` 或等价字段 **待定**）——**不**依赖是否建造设施即有该基础上限。

| 来源 | 说明 |
|------|------|
| **城区基础** | 类别 SO 自带的 **基础住宅数量**；首版默认 **`base_resident_cap = 50`**（特殊 / 一般同值；废墟行除外） |
| **屋舍**（设施） | 建于 [一般城区](#一般城区) **设施建造位**的 **占格类** [屋舍](../设施层.md#屋舍)；在基础上限之上 **叠加**额外居民承载；首版单座 **`extra_resident_cap = 15`**（多座可累加） |
| **合计上限** | `居民承载上限 = 基础居民承载 + Σ(有效屋舍提供的额外承载)`；拆除屋舍后立即扣减加成（超额居民须迁出或停迁入，细则 sy-14） |

- **居民**安置占用合计上限；与 **工作人口**、**队伍编制**分开（见 [人口与迁移 · 居民与工作人口](../../04-资源与人口/人口与迁移.md#居民与工作人口)）。
- 玩家在 [城市管理系统 · 人口分配（住房）](../../04-资源与人口/城市管理系统.md#人口分配住房) 中分配哪类居民入住哪座城区，**不得**超过该城区当前合计承载。
- **废墟**城区 **不可迁入**；基础承载与屋舍加成在修复前 **不适用**于新居民（见 [废墟](#废墟)）。

### 负载成本

**负载成本**指：移动城市在 **航行**状态下每移动一定格数，**各连接城区**（含特殊 / 一般）**分别**累积须用 **金属** **修复**的结构损伤——**不是** [队伍系统 · 负载上限](../../06-单位与交战/队伍系统.md#默认编制与人数影响)（运输队载重），**也不是**整城移动即时耗能（首版**不设**即时油耗）。

| 项 | 首版默认 |
|----|----------|
| **触发** | **航行**中整城每前进 **1** 格计入累计；满 **`move_grids_per_load_step = 3`** 格结算一步 |
| **按城区结算** | 每个仍接入连接网络的城区**独立**结算 |
| **每步损伤** | 该城区 **结构完整度 −2%**（与交战等**同一**完整度池） |
| **修复当量** | **`load_repair_cost = 4`** 金属（= 2% × [修复单价 2 金属/1%](../../04-资源与人口/四种核心资源.md#金属与能源消耗锚已定)）；`load_repair_kind=metal` |
| **修复** | 停泊后经 [修复城区](分离与拆解.md#修复城区) 消耗金属恢复完整度；**低于 20%** → **废墟** |
| **停泊** | **不**因移动累积负载成本 |
| **航行放弃** | 参与城区完整度 **−40%**（见 [分离与拆解](分离与拆解.md#玩家操作连接与分离)） |

```mermaid
flowchart LR
    Move[航行：整城移动] --> Acc[每3格一步]
    Acc --> Dmg[各城区完整度-2%]
    Dmg --> Fix[停泊修复：2金属/1%]
```

- **例**：4 城区连接航行 **12** 格 → 4 步 × 4 区 × 4 金属 = **64** 金属当量待修。
- 数值总表见 [金属与能源消耗锚](../../04-资源与人口/四种核心资源.md#金属与能源消耗锚已定)。

## 玩家可执行的城区操作（分轨）

下列操作**相互独立**；勿与草稿旧称「激活/关闭城区」混淆——后者指 [工作区](运作与居民.md#工作区特殊城区模块--一般城区设施)（模块或设施），**不是**连接拓扑。

| 操作 | 作用对象 | 主要效果 | 文档 |
|------|----------|----------|------|
| **连接 / 分离** | 与核心区的**拓扑链接** | 移动城区 ↔ 落地图分离；航行中**放弃城区** **30%～50%/次** 完整度惩罚 | [分离与拆解 · 连接与分离](分离与拆解.md#玩家操作连接与分离) |
| **城区占格迁移** | footprint 内**占格位置** | **仅停泊**；多回合工作 + 目标格**地基占位** → 迁入 | [分离与拆解 · 城区占格迁移](分离与拆解.md#城区占格迁移) |
| **停泊 ↔ 航行** | **核心区**（骄阳之心） | **1 回合**切换整城离港 / 入港；进度绑定核心区 | [连接与多核心 · 停泊与航行切换](连接与多核心.md#停泊与航行切换核心区工作) |
| **城市管理系统** | **己方 + 占领**城区 | 资源存储、住房、城区能力人力三类分配 | [城市管理系统](../../04-资源与人口/城市管理系统.md) |
| **启用 / 关闭工作区** | **特殊城区**模块；**一般城区**按**设施** | 关闭后停摆，节省能源 / 金属 / 运作人口；**核心区不可关闭**；与设施**共用 UI** | [运作与居民 · 工作区](运作与居民.md#工作区特殊城区模块--一般城区设施) |
| **修复** | **结构完整度**、**废墟 → 正常** | **工程队**对**指定城区**的多回合 [工作](../../07-玩法循环/工作.md)；**仅停泊** | [分离与拆解 · 修复城区](分离与拆解.md#修复城区) |
| **拆解** | **主动拆解工作** → 换金属 | 每次 **−10%** / **+10** 金属；**被动损伤不回收**；过低 → **废墟** | [分离与拆解 · 拆解](分离与拆解.md#玩家操作拆解结构) |

## 类别：特殊城区与一般城区

城区按玩法职责分为两类（与**状态**正交，见下节）：

| 类型 | 职责 | 能力来源 |
|------|------|----------|
| **特殊城区** | 承载**固定、独特**的城市能力 | 城区模块**自带**；不依赖玩家在内部另建常规设施来体现核心功能 |
| **一般城区** | 提供**可建造设施的空地**；**本身不提供** [城区能力](运作与居民.md#城区能力与设施效果) | 玩家在允许列表内建造的 [设施](../设施层.md)；**随移动城市迁移** |

### 特殊城区

- 每个特殊城区对应一种**不可替代**的模块能力，例如：
  - **核心区**（骄阳之心 / 能源心脏）
  - **巨炮**等城市武装或标志性功能模块
  - **城坞**（高效城区修复）、**通讯站**（被动 + 主动视野增益）
  - 外部城市的**势力主城区**（城市领袖默认任职、势力特色能力）
- 特殊城区可携带 **0～2 个词条**（见 [运作与居民.md · 城区词条](运作与居民.md#城区词条)）；**势力主城区**词条定死并与**势力领袖**对齐；其余特殊城区规则 **待定**。
- **城区能力**：特殊城区模块与一般城区设施均可配置 **被动能力**、**主动能力**或**仅其一**（见 [运作与居民 · 城区能力（被动 / 主动）](运作与居民.md#城区能力被动--主动)）；具体模块名单 **后续** 配置（sy-25，优先级低）。示例：
  - **巨炮**（**仅主动能力**）：远程大量伤害；**激活开销** **80** 能源 + **40** 金属（**日常开销**留空）
  - **学院**（**被动**）：工作区开启时放宽归属转化目标；**日常开销** **8** 能源 / 3 回合
  - **通讯站**（**被动 + 主动能力**）：势力级视野增益；**日常** **10** 能源 / 3 回合；**激活** **20** 能源
  - **核心区**：**日常** **15** 能源 / 3 回合（不可关）；**城坞**日常 **10** 能源 / 3 回合

### 一般城区

**设施可接入移动城市**：与核心区保持连接的**移动城区**中，若存在**一般城区**，玩家可在其**设施建造位**上建设配置允许的常规设施；所建设施归属**移动城市**，随城航行与停泊切换而保留（见 [设施层 · 接入移动城市](../设施层.md#接入移动城市一般城区)）。

- 一般城区提供若干**设施建造位**（空地 / 槽位；数量、布局与**可建设施白名单** **待定**），玩家可在其中建造仓库、生产、防御等**常规设施**。
- **一般城区本身不提供城区能力**——不提供特殊城区那种模块级固定能力（单位技能、宏观模块生产等）；其玩法价值来自**设施组合**，而非城区本体。
- **消耗分轨**（见 [运作与居民 · 城区消耗与设施消耗](运作与居民.md#城区消耗与设施消耗)）：
  - **城区自身消耗**：由该一般城区的**类别、状态、词条、承载**等**自身条件**决定（能源 / 金属 / 运作人口等 **待定** 具体项）；**不**因上面建了哪些设施而增减。
  - **设施消耗**：各设施按 [设施层](../设施层.md) 独立结算建造、运行、运维与耐久；**不计入**该一般城区的城区账单。
- **与荒野设施的对比**：
  - **城内（一般城区）**：**占格类**设施绑定**移动城市**，适合长期保留的产能、仓储与辅助功能；投资可随城带走。
  - **荒野（地图设施层）**：设施绑定**世界地图格子**（资源点、桥梁、驿站等）；城市离开后通常**无法整体带走**，资源点采尽或路线变更时往往只能**弃置**或**拆解回收**（见 [探索与扩张](../../07-玩法循环/探索与扩张.md#设施建设)）。
- 一般城区**不**自带特殊城区那种**模块级固定能力**；战略价值来自玩家选择建造并随城携带的**设施**（每座设施 = 工作区，可配被动 / 主动）。

## 状态：正常与废墟

城区按当前可用性分为两种**状态**（与**类别**正交：特殊 / 一般均可处于正常或废墟）：

| 状态 | 说明 |
|------|------|
| **正常** | **特殊城区**：按类别提供**被动 / 主动**能力（经工作区）；**一般城区**：**不**提供模块级固定能力，设施即工作区；均可承载**居民**（若配置允许）；须有人口参与**城区运作**（维持城区本体，**非**设施运行） |
| **废墟** | **结构完整度**过低或遭击毁；**无法工作**（城区运作与供能停止）；**设施**与**特殊城区能力**失能（见 [§废墟失能](#废墟失能)）；**不可迁入**居民，已有居民**仅可迁出**；须修复为**正常**后才视为正常城区 |

### 废墟

- **废墟**是对**状态为废墟的城区**的统称——不是与「城区」并列的另一种建筑层对象。
- **玩家辨识**：地图上须能**一眼认出**废墟与状态**正常**的城区——通过占格外观、图标、信息面板或揭示等级等（具体表现 **待定**），避免与正常城区、资源层 **遗迹** 混淆。
- **机制口径**：在修复完成前，程序与规则**只**将其视为**废墟**状态的城区，**不**按正常城区结算运作、供能、[设施](../设施层.md) 运行、[城区能力](运作与居民.md#城区能力被动--主动)、领袖任职（若仍任职 **待定**）、连接网络等能力。
- **结构完整度与废墟阈值**：
  - 每个城区维护 **结构损伤累计**（与人口 **同一单位**；程序字段名 **待定**）及 **结构完整度**（0～100%，**展示层**），由结构 **容量**（SO）与累计损伤换算。
  - **交战**造成的结构损伤按 **1:1** 写入累计（见 [交战系统 · 受到伤害](../06-单位与交战/交战系统.md#受到伤害遭受方-已定)）。
  - **负载成本、分离、拆解**等非交战来源是否写入 **同一累计池** **待定**（见交战系统 [待对齐项](../06-单位与交战/交战系统.md#战斗结算待对齐项)）。
  - 当 **结构完整度低于 20%** 时，城区**强制**转为**废墟**状态。
- **废墟状态下的人口**：**无法工作**（不参与城区运作）；**不允许迁入**新居民；已在该城区的居民**仅允许迁出**至其他合法城区（见 [人口与迁移 · 废墟与居民迁出](../../04-资源与人口/人口与迁移.md#废墟与居民迁出)）。**招募 · 未效忠**势力城废墟 **除外**——**禁止从住宅迁出**优先（见 [废墟 × 招募 · 未效忠](../../05-城市与领袖/领袖与势力.md#废墟--招募--未效忠已定)）。

#### 废墟失能

**废墟失能**指：状态为**废墟**的城区，其**设施**与**特殊城区能力**在修复为**正常**前**一律不可用**；与既有「**无法工作**」对齐并扩展如下。

| 失能对象 | 口径 | 文档 |
|----------|------|------|
| **城区运作**（特殊城区） | **无法工作**：模块工作区**不可开启** | [运作与居民 · 工作区](运作与居民.md#工作区特殊城区模块--一般城区设施) |
| **设施**（一般城区） | **停止运行**（设施工作区停摆）；**保留占格** | [设施层](../设施层.md)、[废墟失能](#废墟失能) |
| **被动 / 主动能力** | **被动**停用；**主动**不可用（工作区不可开启） | [运作与居民 · 城区能力](运作与居民.md#城区能力被动--主动) |
| **设施**（一般城区建造位及占格类 / 辅助类） | **无法运行**：**不**产出、**不**执行生产 / 仓储 / 防御等运行时效果；**不**接受玩家下达的运维 / 升级等工作指令；既有设施实例**保留占格**，修复城区为**正常**后方可恢复运行（是否须额外「设施修复」步 **待定**） | [设施层](../设施层.md)、[运作与居民 · 城区消耗与设施消耗](运作与居民.md#城区消耗与设施消耗) |

- **示例**（特殊城区）：**巨炮**、**通讯站**、**学院**、**城坞**、**势力主城区**模块等——废墟期间工作区不可开启，**被动 / 主动**均不可用。
- **示例**（一般城区）：仓库、生产、屋舍等**设施**——废墟期间**停止运行**；**不**因废墟自动拆除占格。
- **与占领**：敌对城市**打为废墟**是 [废墟占领](../../05-城市与领袖/领袖与势力.md#废墟占领敌对--已定框架) 的**前置**（占格细则 **待定**，sy-34）；废墟失能**不**等于城市**消亡**（城市整体失能条件 **待定**，sy-31）。
- **粮食**：废墟人口**仍计**周总结粮食需求；未分到同样**半数减员**；与「能否工作」**脱钩**（见 [粮食与周总结（草稿）](../../../01-草稿/粮食与周总结/README.md)）。
- **来源**：
  - **结构完整度 低于 20%**（含 **负载成本**（航行移动）、主动拆解结构、交战损伤、航行中**放弃城区** **30%～50%/次**、灾害等）；
  - **地图默认刷新**：荒野格子上也可**默认刷新**已为废墟状态的城区（未修复前视为残骸占格）。
- **修复为正常**：玩家指定**工程队**对目标城区展开修复**工作**（消耗金属等 **待定**）；完整度 **≥ 20%** 且状态由**废墟**变为**正常**后，方可迁入居民、启用工作区与连接等；邻近 **城坞** 可影响该项工作效率（**待定**）。
- **拆解**（**拆解结构**，见 [分离与拆解.md](分离与拆解.md)）：对城区**结构**发起**主动拆解工作**，削减完整度并回收金属；**被动损伤不回收**；与**分离城区**是**不同概念**。
- 与 [资源层 · 遗迹](../../04-资源与人口/荒野地点.md#遗迹与废墟术语强制区分) **不是**同一类存在：遗迹为前文明资源点，**不可**修复为城区。

### 城区类型示例

#### 特殊城区

| 类型 | 定位 |
|------|------|
| **核心区** | 拥有骄阳之心的特殊城区；与核心区连接的城区才是移动城区 |
| **巨炮** | **仅主动能力**：远程重击；**激活开销**大量 **能源 / 金属** |
| **学院** | **被动**：工作区开启时放宽本城区人口归属转化目标；**日常开销**维持被动 |
| **城坞** | 城区修复中心；开销与能力配置 **待定**（sy-25） |
| **通讯站** | **被动 + 主动能力**：势力级视野增益；**日常开销** / **激活开销**分别配置（数值 sy-25） |
| **势力主城区**（外部城市） | 城市领袖默认任职；势力特色城区能力；招募后定义不变（见 [领袖与势力](../../05-城市与领袖/领袖与势力.md)） |

下列模块若仅提供固定加成、不可在内自由铺建常规设施，也归入**特殊城区**（如部分**居民区**仅承担人口承载 **待定** 分类）。

#### 一般城区

| 定位 | 说明 |
|------|------|
| **设施空地** | 提供设施建造位；**不提供城区能力**；所建设施随城市移动，消耗独立结算 |
| **可改造** | 玩家选择仓库、生产、辅助等设施组合，形成「随城携带」的功能区 |

> **城坞特殊规则**：一般情况下城市不允许重叠，但城坞属于额外条件——允许修复与自身重叠但未连接在同一城市的城区。

> **通讯站**：势力级视野增益（见上表）。规则见 [通讯与视野系统 · 通讯站](../../06-单位与交战/通讯与视野系统.md#通讯站视野已定)；主动消耗数值见 sy-25。
