# 城区总览

## 术语

- **[移动城市](../../02-地图与世界/地图与移动.md)**即**玩家主城**；地图上仅此一座可整体移动。
- **城区**：见下节；与 [地图图层 · 建筑层](../地图图层.md#建筑层与城区)**划等号**——建筑层格上存储的内容就是城区这一机制对象，不再另设平行的「建筑类型」。
- **[骄阳之心（反应堆）](../../../04-设定/05-隐秘真相/骄阳之心.md)**：驱动城市的能源核心，在游戏机制中体现为**核心区**（特殊城区的一种）。
- **核心区**：拥有骄阳之心的城区，是城市的能源心脏。
- **移动城区**：与核心区保持连接、属于移动城市一部分的城区。
- 荒野格上的城区、外部城市占格等，同为建筑层上的城区实例；未接入玩家移动城市连接网络的，见 [荒野地点](../../04-资源与人口/荒野地点.md)。

## 城区：定义与资产定位

- **城区**是 [建筑层](../地图图层.md#建筑层与城区) 上的核心内容：每格建筑层至多一个城区实例（不可堆叠）。
- 玩法定位类似《杀戮尖塔》中的**遗物**：具有长期战略价值；**类别**决定能力形态，**状态**决定当前是否可用。
- **不可通过常规建造获得**：玩家不能像搭设施一样在空地「新建」一座城区；获得新城区须通过探索、事件、贸易、修复废墟、拆解回收或剧情等**稀有途径**（细则 **待定**，见 [待细化追踪 · OPEN-033](../../../00-规范/待细化追踪.md)）。
- 玩家可**改造、修复、拆解、再连接**已有城区，但不可凭空创造城区**类别**。

## 类别：特殊城区与一般城区

城区按玩法职责分为两类（与**状态**正交，见下节）：

| 类型 | 职责 | 能力来源 |
|------|------|----------|
| **特殊城区** | 承载**固定、独特**的城市能力 | 城区模块**自带**；不依赖玩家在内部另建常规设施来体现核心功能 |
| **一般城区** | 提供**可建造设施的空地** | 玩家在空地内建造的 [设施](../设施层.md)；随城市迁移 |

### 特殊城区

- 每个特殊城区对应一种**不可替代**的模块能力，例如：
  - **核心区**（骄阳之心 / 能源心脏）
  - **巨炮**等城市武装或标志性功能模块
  - **城坞**（高效城区修复）、**通讯站**（即时通讯中枢）
  - 外部城市的**势力主城区**（城市领袖默认任职、势力特色能力）
- 特殊城区可携带 **1～3 个词条**（见 [运作与居民.md](运作与居民.md)），并与 [领袖与势力](../../05-城市与领袖/领袖与势力.md) 中的任职条件对齐。
- **城区能力**（单位技能、宏观生产、设施激活等）主要由特殊城区**本体**提供。

### 一般城区

- 一般城区提供若干**设施建造位**（空地 / 槽位；数量与布局 **待定**），玩家可在其中建造仓库、生产、防御等**常规设施**。
- 建于一般城区内的设施**随移动城市整体迁移**——城市航行或停泊切换时，这些设施仍在城内，无需在荒野重复建设。
- **与荒野设施的对比**：
  - **城内（一般城区）**：设施绑定**移动城市**，适合长期保留的产能、仓储与辅助功能；投资可随城带走。
  - **荒野（地图设施层）**：设施绑定**世界地图格子**（资源点、桥梁、驿站等）；城市离开后通常**无法整体带走**，资源点采尽或路线变更时往往只能**弃置**或**拆解回收**（见 [探索与扩张](../../07-玩法循环/探索与扩张.md#设施建设)）。
- 一般城区**不**自带与特殊城区同级的独特模块能力；其战略价值来自玩家选择建造并随城携带的设施组合。

## 状态：正常与废墟

城区按当前可用性分为两种**状态**（与**类别**正交：特殊 / 一般均可处于正常或废墟）：

| 状态 | 说明 |
|------|------|
| **正常** | 城区按类别提供能力；可承载**居民**（若该类别 / 配置允许）；须有人口参与**城区运作**方能供能运转；一般城区内可建造设施 |
| **废墟** | **结构完整度**过低或遭击毁；**无法工作**（城区运作与供能停止）；**不可迁入**居民，已有居民**仅可迁出**；须修复为**正常**后才视为正常城区 |

### 废墟

- **废墟**是对**状态为废墟的城区**的统称——不是与「城区」并列的另一种建筑层对象。
- **玩家辨识**：地图上须能**一眼认出**废墟与状态**正常**的城区——通过占格外观、图标、信息面板或揭示等级等（具体表现 **待定**），避免与正常城区、资源层 **遗迹** 混淆。
- **机制口径**：在修复完成前，程序与规则**只**将其视为**废墟**状态的城区，**不**按正常城区结算运作、供能、领袖任职（若仍任职 **待定**）、连接网络等能力。
- **结构完整度与废墟阈值**：
  - 每个城区维护 **结构完整度**（0～100%，程序字段名 **待定**），表示当前建筑结构剩余比例；与 [拆解结构](分离与拆解.md#拆解结构) 中的累计损伤联动（完整度下降 ↔ 损伤上升 **待定**换算）。
  - 当 **结构完整度低于 20%** 时，城区**强制**转为**废墟**状态（战斗击毁、事件或主动拆解导致的过低完整度均适用）。
- **废墟状态下的人口**：**无法工作**（不参与城区运作）；**不允许迁入**新居民；已在该城区的居民**仅允许迁出**至其他合法城区（见 [人口与迁移 · 废墟与居民迁出](../../04-资源与人口/人口与迁移.md#废墟与居民迁出)）。
- **来源**：
  - **结构完整度 低于 20%**（含主动拆解结构、交战损伤、灾害等）；
  - **地图默认刷新**：荒野格子上也可**默认刷新**已为废墟状态的城区（未修复前视为残骸占格）。
- **修复为正常**：须花费 **金属** 等资源并完成对应 [工作](../../07-玩法循环/工作.md)（细则 **待定**），将**结构完整度**恢复至 **20% 及以上**且状态由**废墟**变为**正常**后，方**视为**正常城区——可迁入居民、参与城区运作、启用类别能力并接入移动城市连接（若合法）；城坞等可影响修复效率。
- **拆解**（**拆解结构**，见 [分离与拆解.md](分离与拆解.md)）：对城区**结构**进行拆解，用**累计损伤**换取金属等资源；与**分离城区**是**不同概念**。
- 与 [资源层 · 遗迹](../../04-资源与人口/荒野地点.md#遗迹与废墟术语强制区分) **不是**同一类存在：遗迹为前文明资源点，**不可**修复为城区。

### 城区类型示例

#### 特殊城区

| 类型 | 定位 |
|------|------|
| **核心区** | 拥有骄阳之心的特殊城区；与核心区连接的城区才是移动城区 |
| **巨炮**等特殊武装 / 功能模块 | 城市级独特战斗或功能能力（示例 **待定** 完整名单） |
| **城坞** | 城区修复中心；高效修复、允许重叠修复未连接城区的特殊规则 |
| **通讯站** | 即时通讯中枢；为离城单位提供信号覆盖 |
| **势力主城区**（外部城市） | 城市领袖默认任职；势力特色城区能力；招募后定义不变（见 [领袖与势力](../../05-城市与领袖/领袖与势力.md)） |

下列模块若仅提供固定加成、不可在内自由铺建常规设施，也归入**特殊城区**（如部分**居民区**仅承担人口承载 **待定** 分类）。

#### 一般城区

| 定位 | 说明 |
|------|------|
| **设施空地** | 提供设施建造位；所建设施随城市移动 |
| **可改造** | 玩家选择仓库、生产、辅助等设施组合，形成「随城携带」的功能区 |

> **城坞特殊规则**：一般情况下城市不允许重叠，但城坞属于额外条件——允许修复与自身重叠但未连接在同一城市的城区。

> **通讯站特殊规则**：即时通讯需要信号覆盖。离开城市的单位能否使用即时通讯取决于是否有通讯站支持。
