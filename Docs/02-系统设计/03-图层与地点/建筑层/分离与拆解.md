# 分离与拆解

## 关键概念（分轨）

| | **分离城区** | **拆解结构** |
|---|-------------|-------------|
| **做什么** | **解除**城区与移动城市（核心区网络）的**链接** | 对城区**结构**进行拆解 |
| **结果** | 城区脱离随城移动，以建筑层实例落入地图；类别 / 状态一般不变 | **累计损伤**换取金属等资源 |
| **资源** | **不**因分离本身回收资源 | 按累计损伤比例换资源 |
| **废墟** | 分离**不会**自动变废墟 | **累计损伤过高** → 状态变为**废墟** |
| **形态调整** | **先分离、再连接**（收窄、改布局） | 与之无关 |

二者可独立发生：可只分离不拆解，也可在仍连接或已分离状态下拆解结构。

## 分离城区

**分离**指**解除城区与移动城市的链接**（脱离核心区连接网络），**不是**拆解结构、**不是**用损伤换资源。

| 要点 | 说明 |
|------|------|
| **操作本质** | 断开拓扑连接；城区实例仍保留，类别与（分离前）状态一般不变 |
| **结果** | 该城区不再随移动城市整体移动；以**建筑层城区**落入世界地图（占格 **待定**） |
| **典型用途** | 让路通过狭窄地形、临时派出城区执行任务、牺牲连接以保全其余部分等 |
| **与拆解** | **分离**不必然产生回收资源；**拆解结构**不必然导致分离（可原地拆解 **待定**） |

- **分离规则（被动）**：若连接网络被切断，凡**不能连接到任意核心区**的城区，一律视为**已分离**，落入世界地图。
- **调整城市形态**（收窄、改布局）：以**先分离、再连接**为主流程，须玩家**手动逐步操作**（多回合工作）；**不**与「拆解结构换资源」混为同一操作。

## 拆解结构

**拆解结构**指对城区**建筑结构**进行拆解：**累计损伤**可换取 **金属** 等资源；与**分离城区**是**独立机制**。

| 要点 | 说明 |
|------|------|
| **操作本质** | 削减城区**结构完整度**，按规则将部分损伤转化为可回收资源 |
| **结构完整度** | 0～100%；与累计损伤联动（**待定**字段映射）；**低于 20%** 时城区转为**废墟** |
| **换资源** | 主动拆解或允许损伤结算时，按完整度 / 损伤比例获得金属等（公式 **待定**） |
| **转废墟** | **结构完整度 低于 20%** → 状态**废墟**（见 [城区总览.md · 废墟](城区总览.md#废墟)）；无法工作；人口仅可迁出、不可迁入 |
| **与分离** | 可在仍连接移动城市时拆解，也可对已分离的城区拆解；**分离不会自动**触发拆解换资源 |

- 状态已为**废墟**的城区仍可继续**拆解结构**以回收剩余资源（比例 **待定**），与「修复为正常」为不同选择。
