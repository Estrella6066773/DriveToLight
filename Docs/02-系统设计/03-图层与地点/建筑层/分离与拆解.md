# 分离与拆解

## 关键概念（分轨）

| | **分离城区** | **拆解结构** |
|---|-------------|-------------|
| **做什么** | **解除**城区与移动城市（核心区网络）的**链接** | 对城区**结构**进行拆解 |
| **结果** | 城区脱离随城移动，以建筑层实例落入地图；类别 / 状态一般不变 | **累计损伤**换取金属等资源 |
| **资源** | **不**因分离本身回收资源 | 按累计损伤比例换资源 |
| **废墟** | 分离**不会**自动变废墟 | **累计损伤过高** → 状态变为**废墟** |
| **形态调整** | **先分离、再连接**（收窄、改布局） | 与之无关 |

二者可独立发生：可只分离不拆解，也可在仍连接或已分离状态下拆解结构。

## 分离城区

**分离**指**解除城区与移动城市的链接**（脱离核心区连接网络），**不是**拆解结构、**不是**用损伤换资源。

| 要点 | 说明 |
|------|------|
| **操作本质** | 断开拓扑连接；城区实例仍保留，类别与（分离前）状态一般不变 |
| **结果** | 该城区不再随移动城市整体移动；以**建筑层城区**落入世界地图（占格 **待定**） |
| **典型用途** | 让路通过狭窄地形、临时派出城区执行任务、牺牲连接以保全其余部分等 |
| **与拆解** | **分离**不必然产生回收资源；**拆解结构**不必然导致分离（可原地拆解 **待定**） |

- **分离规则（被动）**：若连接网络被切断，凡**不能连接到任意核心区**的城区，一律视为**已分离**，落入世界地图。
- **调整城市形态**（收窄、改布局）：**城区占格迁移**（见 [§城区占格迁移](#城区占格迁移)）或 **先分离、再连接**；须玩家**手动逐步操作**；**停泊**下经多回合 [工作](../../07-玩法循环/工作.md)，**航行**下可即时断连但承受高额结构损耗（见下节）。

## 城区占格迁移

在移动城市 **停泊** 状态下，玩家可将仍连接在核心区网络上的城区**迁移到 footprint 内另一合法占格**——**不是** [分离](#分离城区)（脱离网络），而是**改变该城区在城廓内的相对位置**。

| 项 | 规则 |
|----|------|
| **前提** | 移动城市处于 **停泊**；目标格合法、无冲突 |
| **工作形态** | 多回合 [工作](../../07-玩法循环/工作.md)；`work_type` **待定**（如 `district_relocate`） |
| **对照工作量** | **3～5**（具体默认 **待定**）；每回合增量受 [工作效率与工作量](../../07-玩法循环/回合与行动表.md#工作效率与工作量) 影响 |
| **开始时** | 在目标格于 [建筑层](../地图图层.md#建筑层与城区) **预占地基**（占位标记；占用规则 **待定**） |
| **完成时** | 城区实例**直接迁入**目标占格；更新连接拓扑与 footprint |
| **与分离** | **占格迁移**不断开与核心区的链接；**分离**则城区脱离随城移动 |
| **航行** | **不可**发起城区占格迁移 |

**典型用途**：通过狭窄地形前在 footprint 内重新排布模块，无需先分离再连接。

## 玩家操作：连接与分离

### 何时可操作

- 玩家在**指挥阶段****随时**可对**己方**移动城市所管辖的城区发起**连接**（接回核心区网络 / 与其它城区连边）或**分离**（断开指定拓扑边）。
- **占领城区**是否适用同一套操作 **待定**。
- 被动规则不变：连接网络被切断后，凡**不能连接到任意核心区**的城区一律视为**已分离**（见上节）。

### 停泊 vs 航行（已定）

移动城市处于 [停泊](../../02-地图与世界/地图与移动.md#停泊与航行) 或 **航行** 时，同一连接/分离指令的**代价模型不同**：

| 状态 | 完成方式 | 结构损耗 |
|------|----------|----------|
| **停泊** | 须占用多回合 **工作**（`work_type`、对照时长 **待定**，sy-15）；**不可**一键完成 | **无**因连接/分离本身触发的结构完整度惩罚 |
| **航行** | **允许**随时**放弃城区**（分离）；是否仍占回合 / 是否即时 **待定** | **每执行一次**，**参与该次操作的所有城区** **结构完整度各一次性下降 30%～50%**（具体数值 **待定**；累计；**低于 20%** → **废墟**，见 [城区总览 · 废墟](城区总览.md#废墟)） |

**设计意图**：**停泊**用**时间**换安全；**航行**用**高额结构损伤**换机动期间的形态调整自由度（含**放弃城区**以通过地形或减负）。

- 此损伤与 [拆解结构](#拆解结构) 主动换资源的损伤**分开结算**，但均写入同一 **结构完整度** 字段。
- 数值 **30%～50%** 为当前对照区间；是否随章节或难度缩放 **待定**（数值框架延后）。

### 再连接前置

- 将已**分离**的城区接回网络前，须在合法位置**对齐**占格；不可一键自动拼合。
- 对齐规则、占格冲突、工程队是否必须到场 **待定**（sy-15）。

## 拆解结构

**拆解结构**指对城区**建筑结构**进行拆解：**累计损伤**可换取 **金属** 等资源；与**分离城区**是**独立机制**。

| 要点 | 说明 |
|------|------|
| **操作本质** | 削减城区**结构完整度**，按规则将部分损伤转化为可回收资源 |
| **结构完整度** | 0～100%；与累计损伤联动（**待定**字段映射）；**低于 20%** 时城区转为**废墟** |
| **损伤来源** | 含 **负载成本**（航行移动，见 [城区总览 · 负载成本](城区总览.md#负载成本)）、交战、拆解、航行中**放弃城区**（分离）**30%～50%/次** 等，**同一**完整度池 |
| **换资源** | 主动拆解或允许损伤结算时，按完整度 / 损伤比例获得金属等（公式 **待定**） |
| **转废墟** | **结构完整度 低于 20%** → 状态**废墟**（见 [城区总览.md · 废墟](城区总览.md#废墟)）；无法工作；人口仅可迁出、不可迁入 |
| **与分离** | 可在仍连接移动城市时拆解，也可对已分离的城区拆解；**分离不会自动**触发拆解换资源 |

- 状态已为**废墟**的城区仍可继续**拆解结构**以回收剩余资源（比例 **待定**），与「修复为正常」为不同选择。

## 修复城区

**修复**指恢复城区**结构完整度**与**可用状态**，使**正常**机制（含 [工作区启用](../运作与居民.md#工作区启用与关闭)、[城区运作](../运作与居民.md#城区运作与居民人口)）可再次生效；**不是** [分离](#分离城区) 的逆操作名称混用——修复针对**结构与健康度**，连接针对**拓扑**。

| 要点 | 说明 |
|------|------|
| **主要目标** | **废墟 → 正常**：将 **结构完整度** 恢复至 **20% 及以上**，状态由**废墟**变为**正常** |
| **资源** | 消耗 **金属** 等（具体清单 **待定**）；**能源** 是否预付 **待定** |
| **执行** | 多回合 [工作](../../07-玩法循环/工作.md)；执行者通常为 **工程队** 或 **城坞**（城坞效率加成见 [运作与居民 · 城坞](../运作与居民.md#功能模块城坞通讯站等)） |
| **与拆解** | 修复**提高**完整度；拆解**降低**完整度换资源；二者互斥进行中的规则 **待定** |
| **与连接** | 已**分离**的城区可修复，但接回移动城市须另做 [再连接](#玩家操作连接与分离) |
| **停泊 / 航行** | 细则 **待定**（默认按一般城区工作在城内进行；是否限制航行中修复 sy-22） |

- 完整度已在 **20% 以上**但仍受损的**正常**城区，是否支持「局部修复」**待定**（数值框架延后）。

## 玩家操作：拆解结构

在 [拆解结构](#拆解结构) 规则之上，玩家主动发起**拆解**：

| 要点 | 说明 |
|------|------|
| **何时** | 状态为**正常**或**废墟**均可（废墟可继续拆至回收殆尽 **待定**） |
| **结果** | **结构完整度**下降；按规则获得 **金属** 等资源；**低于 20%** → **废墟** |
| **执行** | 多回合 **工作**（`work_type`、对照时长 **待定**）；与 [修复城区](#修复城区) 分轨 |
| **与分离** | 可在仍**连接**移动城市时拆解，也可对**已分离**城区拆解；**不**自动触发分离 |
| **与关闭工作区** | 关闭工作区**不**等于拆解；拆解是永久性结构回收，关闭是能力停摆 |
