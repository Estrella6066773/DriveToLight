# 分离与拆解

## 关键概念（分轨）

| | **分离城区** | **拆解结构** |
|---|-------------|-------------|
| **做什么** | **拆除链接**，使城区脱离核心网络 | 对城区**结构**进行拆解 |
| **结果** | 城区脱离随城移动，以建筑层实例落入地图；类别 / 状态一般不变 | **主动拆解工作**削减完整度并回收金属 |
| **资源** | **不**因分离本身回收资源 | **仅**主动拆解回收金属；**被动损伤不回收** |
| **废墟** | 分离**不会**自动变废墟 | **累计损伤过高** → 状态变为**废墟** |
| **形态调整** | **先分离、再连接**（收窄、改布局） | 与之无关 |

二者可独立发生：可只分离不拆解，也可对**已归属玩家**的城区拆解结构——**拆解只看所有权，不看链接**（无论城区处于连接还是分离状态，只要所有权满足即可拆解）。**敌方城区不属玩家**——不可直接拆解。

## 分离城区

**分离**指某城区**不能**经 **链接** 的路径连到**任意核心区**（见 [连接与多核心 · 链接与连接拓扑](连接与多核心.md#链接与连接拓扑)），从而脱离随城移动；**不是**拆解结构、**不是**用损伤换资源。

| 要点 | 说明 |
|------|------|
| **操作本质** | **拆除**一条或多条**链接**；或被动因其它拆除导致该城区与所有核心区**不连通** |
| **结果** | 该城区不再随移动城市整体移动；以**建筑层城区**落入世界地图（占格 **待定**） |
| **典型用途** | 让路通过狭窄地形、临时派出城区执行任务、牺牲连接以保全其余部分等 |
| **与拆解** | **分离**不必然产生回收资源；**拆解结构**不必然导致分离——详见 [§拆解结构](#拆解结构)。**拆解的前提**：城区须为**中立**或**归属于玩家**；招募（未效忠）的城区**不允许**拆解 |

- **分离判定（已定）**：凡**不能**经链接连接到**任意核心区**的城区，一律视为**已分离**，落入世界地图。
- **调整城市形态**（收窄、改布局）：**迁移城区**（见 [§迁移城区](#迁移城区)）或 **先分离、再修建链接**；须玩家**手动逐步操作**；链接本身为**即时操作**（消耗资源后立即生效），**航行**下放弃城区仍承受高额结构损耗（见下节）。

## 迁移城区

在移动城市 **停泊** 状态下，玩家可将仍连接在核心区网络上的城区**迁移到 footprint 内另一合法占格**——**不是** [分离](#分离城区)（脱离网络），而是**改变该城区在城廓内的相对位置**。

| 项 | 规则 |
|----|------|
| **前提** | 移动城市处于 **停泊**；目标格合法、无冲突 |
| **工作形态** | 多回合 [工作](../../07-玩法循环/工作.md)；**对自己进行工作**——工作对象与被工作对象相同（城区自身）；**与其他工作互斥**——迁移期间该城区不可同时执行其它工作 |
| **对照工作量** | **3 回合**；每回合增量受 [工作效率与工作量](../../07-玩法循环/回合与行动表.md#工作效率与工作量) 影响 |
| **开始时** | 在目标格于 [建筑层](../地图图层.md#建筑层与城区) **预占地基**（占位标记；已定规则见下） |
| **完成时** | 城区实例**直接迁入**目标占格；释放原位占格；更新连接拓扑与 footprint |
| **中断/取消** | 撤销目标格预占，自动释放；原位占格与城区状态恢复 |
| **与分离** | **迁移城区**不断开与核心区的链接；**分离**则城区脱离随城移动 |
| **航行** | **不可**迁移城区 |

**迁移期间的基本约束（已定）**：

- **目标格预占**：开始迁移时在目标格放置预占标记。预占格相当于一个「在迁移中」的城区占位，工作内容是「将原位城区迁移到此处」。预占期间该格**禁止其它用途**——不可被另一座城区占用，不可建造设施，不可作为其它迁移的目标。
- **原位城区**：迁移期间视为「工作中」状态，工作内容是「将本城区迁移至目标位置」。**禁止所有正常功能**——工作区暂停、设施暂停、被动能力不触发、不可对其发起其它工作指令。
- **中断/取消处置**：玩家取消迁移或被外部因素打断时，撤销目标格预占标记，自动释放两格；原位城区恢复迁移前状态。

迁移期间的链接行为见 [§链接再配置](#链接再配置)。

**链接再配置**：迁移城区时，与该城区相连的现有链接**自动拆除**；玩家在规划迁移时须**手动指定新位置与哪些相邻城区之间重建链接**。所需资源**在规划时提前预约**（新建链接消耗 15 金属/条，拆除不回收迁移期间的自动拆除），在迁移工作完成结算时**即时**拆除已有链接并按方案一并修建新链接。不须工程队在场。

**典型用途**：通过狭窄地形前在 footprint 内重新排布模块，无需先分离再连接。

## 玩家操作：连接与分离

### 何时可操作

- 玩家在**指挥阶段****随时**可对**己方**移动城市所管辖的城区发起 **连接**（**修建链接** / 接回核心网络）或 **分离**（**拆除链接**）。
- **中立城区**：武装单位在城区格完成「**宣称占领**」工作后，该城区脱离原城市变为中立无主城区（归属态变更；与损坏态（正常/废墟）独立），可被玩家连接（是否需要连接工作 **待定**，sy-15）。（已定，sy-34）
- 被动规则不变：拆除链接后，凡**不能**经链接连到**任意核心区**的城区一律视为**已分离**（见 [连接与多核心 · 分离判定](连接与多核心.md#移动城区与分离判定已定)）。
- **即时性**（已定）：链接修建与拆除均为**即时操作**——消耗资源后立即生效；不须工程队、不须占用城区人口。修建消耗 **15 金属**，拆除回收 **1 金属**（10%），重建 **12 金属**（80%），规则与 [设施](../设施层.md#残骸回收与重建已定) 一致。

### 停泊 vs 航行（已定）

移动城市处于 [停泊](../../02-地图与世界/地图与移动.md#停泊与航行) 或 **航行** 时，同一连接/分离指令的**代价模型不同**：

| 状态 | 完成方式 | 结构损耗 |
|------|----------|----------|
| **停泊** | 即时消耗资源并生效；**不可**一键完成（须逐条手动操作） | **无**因连接/分离本身触发的结构完整度惩罚 |
| **航行** | **允许**随时**放弃城区**（分离），链接拆除本身即时 | **每执行一次**，**脱离的城区** **结构完整度各一次性下降 40%**（已定；累计；**低于 20%** → **废墟**）；损伤按每回合结算（与通用规则一致） |

**设计意图**：**停泊**用**时间**换安全；**航行**用**高额结构损伤**换机动期间的形态调整自由度（含**放弃城区**以通过地形或减负）。

- 此损伤与 [拆解结构](#拆解结构) 主动换资源的损伤**分开结算**，但均写入同一 **结构完整度** 字段。
- 航行放弃完整度下降取 **40%**（已定）；补回约 **80** 金属（按 2 金属/1%）。

### 再连接前置

将已**分离**的城区接回网络前，须在合法位置**对齐**占格；不可一键自动拼合。链接修建即时消耗资源并生效，不须工程队。

**占格对齐规则（已定）**：

- **仅一个相邻格**：直接与该格建立链接，不须玩家点选。
- **多个相邻格**：玩家在可选相邻格中点选，敲定需要连接哪些格。

### 编辑连接

对仍连接在网络中的某个**单个城区**，玩家可进入**「编辑连接」**模式，精细化调整该城区与各相邻城区的链接关系：

1. 进入编辑状态后，高亮显示该城区所有**相邻城区格**。
2. 玩家**点选**相邻格，切换是否设置链接（当前有链接 → 标记为拆除；当前无链接 → 标记为新建；不变的不作标记）。
3. **确认**时，系统按新设置**比较当前链接状态**，自动执行：
   - **新建**：原来没有，新设了 → 消耗 15 金属/条
   - **保留**：原来已有，新设不变 → 无消耗、无回收
   - **拆除**：原来已有，新设取消 → 回收 1 金属/条
4. 单次确认可同时包含新建、保留、拆除多条链接，均在确认时**即时结算**。

## 拆解结构

**拆解结构**指对城区**建筑结构**进行**主动拆解工作**，削减完整度并回收 **金属**；与**分离城区**是**独立机制**。

| 要点 | 说明 |
|------|------|
| **操作本质** | 削减城区**结构完整度**，仅在**主动拆解工作完成时**转化为可回收金属 |
| **所有权** | 城区须为**中立**或**归属于玩家**；敌对城区不可直接拆解；招募（未效忠）的城区**不允许**拆解（见 [领袖与势力 · 势力城区管辖](../05-城市与领袖/领袖与势力.md#招募后势力城区管辖已定)） |
| **结构完整度** | 0～100%；与累计损伤联动（**待定**字段映射）；**低于 20%** 时城区转为**废墟** |
| **损伤来源** | 含 **负载成本**（航行每 3 格 −2%/区）、交战、拆解、航行放弃 **−40%/次** 等，**同一**完整度池 |
| **换资源** | **仅主动拆解**（玩家工作）：每次完成 **−10%** 完整度 → **+10** 金属（见 [玩家操作：拆解结构](#玩家操作拆解结构)） |
| **被动损伤** | 负载、交战、航行放弃等导致的完整度下降：**不**回收金属（已定） |
| **转废墟** | **结构完整度 低于 20%** → 状态**废墟**（见 [城区总览.md · 废墟](城区总览.md#废墟)）；无法工作；人口仅可迁出、不可迁入 |
| **与分离** | 可在仍连接移动城市时拆解，也可对已分离的城区拆解；**分离不会自动**触发拆解换资源 |

- 状态已为**废墟**的城区仍可继续**拆解结构**以回收剩余资源（已定，sy-22：**按城区分开算**，不须整城全废墟），与「修复为正常」为不同选择。

## 修复城区

**修复**指恢复城区**结构完整度**与**可用状态**，使**正常**机制（含 [工作区](../运作与居民.md#工作区特殊城区模块--一般城区设施)、设施工作区）可再次生效；**不是** [分离](#分离城区) 的逆操作。

**城区修复期间**，该城区上的**设施**与**工作区**自动暂停——不可工作、不触发被动能力、不消耗日常开销；修复完成后自动恢复。

修复有三种途径——**自主修复**、**工程队修复**、**城坞被动修复**。三者均为对该城区的**工作操作**，同一城区同一时刻**仅可运行一种**，**不可并行**。切换途径须先停止当前工作（正在进行的城坞被动视为「该城区正在被城坞修复」）。

修复行为为 **n 回合工作**，**每回合独立触发阶段性成果**——每回合结算一次本回合的工作进度、消耗与恢复量，而非积满后一次性触发。

> **共通规则**：
> - **终止条件 ①**：城区完整度**恢复至 100%** → 修复工作**自动终止**，**不允许**超过 100%（超出部分舍弃，不另补金属）。
> - **终止条件 ②**：回合结算时城市仓库金属**不足本轮消耗** → 该轮**自动终止**，不扣金属、不恢复完整度。

### 自主修复

玩家城区**自行调度本地人口**进行修复，**不占工程队行动表**：

| 项 | 口径 |
|---|------|
| **触发** | 玩家对己方城区下达「自主修复」指令 |
| **人口** | 最多 **15** 本地居民（不足 15 人以实际为准） |
| **效率** | 初始效率（每人 `work_efficiency = 1.0`）；15 人满员时每回合积攒 **15** 工作量 |
| **单次完成** | 每回合独立结算：本回合工作量 ÷ 30 × 10% = 本回合恢复完整度；相应扣除 20 ×（本回合工作量 ÷ 30）金属（向下取整） |
| **航行中** | **允许**航行中继续（不依赖外部队伍到场） |

### 工程队修复

玩家**指挥工程队**前往指定城区修复：

| 项 | 口径 |
|---|------|
| **执行者** | 工程队（`work_subject_kind = team`，`work_subject_ref` 指向工程队实例） |
| **被工作对象** | 目标城区（`work_subject_kind = district`） |
| **效率** | 满编 20 人时，每人效率 **1.5**（= 初始效率 × 1.5），满编时每回合积攒 **30** 工作量 |
| **单次完成** | 每回合独立结算：本回合工作量 ÷ 30 × 10% = 本回合恢复完整度；相应扣除 20 ×（本回合工作量 ÷ 30）金属（向下取整） |
| **停泊 / 航行** | **仅停泊**可下达；**航行中不可**下达工程队修复 |

> 满编工程队（20 人 × 效率 1.5 = 30/回合）每回合恰好恢复 **10%** 完整度、扣 **20** 金属。自主修复 15 人满员时每回合恢复约 **5%** 完整度、扣约 **10** 金属。

### 城坞被动 · 主动

城坞模块**工作区开启**时生效：

| 方式 | 效果 | 消耗 |
|------|------|------|
| **被动** | 每回合自动向**相连城区**（均分）恢复价值 **20 金属**的结构损伤（= 合计 **10%** 完整度）。相连 2 城区各 +5%、相连 3 城区各约 +3.3%；完整度已达 100% 的城区不参与均分；**正在自主修复或工程队修复的城区不纳入分配**（同一城区不可并行两种修复工作）。**受工作效率修正**——取决于城坞工作区实际上岗人数，计算结果向下取整 | 无额外消耗（已含于城坞日常开销 **10 能源/3 回合**） |
| **主动** | 指定城区**立刻**完成一次维修（+10% 完整度），不计工作量、不占行动表。同样受终止条件约束——城区已达 100% 时不可激活；金属不足时拒绝激活 | **20 金属 + 10 能源** |

- 主动效果不需工程队在场；冷却 **5** 回合（回合结束后开始计算）。
- 被动与主动可同时生效（同一回合被动先结算，主动后触发）。

## 玩家操作：拆解结构

在 [拆解结构](#拆解结构) 规则之上，玩家主动发起**拆解**：

| 要点 | 说明 |
|------|------|
| **何时** | 城区须为**中立**或**归属于玩家**；状态为**正常**或**废墟**均可（按城区分开算，不须整城全废墟） |
| **结果** | 每次完成：完整度 **−10%**；获得 **10** 金属；**低于 20%** → **废墟** |
| **执行** | 工作量 **30**（3 回合）；`work_type_id=dismantle_district`；与 [修复城区](#修复城区) **不可同时**对同一城区进行 |
| **与分离** | **拆解只看所有权，不看链接**——城区归属于玩家（含已效忠）或中立即可拆解，不论处于连接还是分离状态。敌方城区**所有权不属玩家**，不可拆解；招募（未效忠）城区**禁止拆解** |
| **与关闭工作区** | 关闭工作区**不**等于拆解；拆解是永久性结构回收，关闭是能力停摆 |
