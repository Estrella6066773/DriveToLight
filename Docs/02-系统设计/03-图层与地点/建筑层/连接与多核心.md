# 连接与多核心

## 核心区与骄阳之心

> **引用设定**
>
> "反应堆在游戏机制上体现为「核心区」，与核心区相连接的城区为移动城区。"

- **核心区**对应设定中的**骄阳之心（反应堆）**，是城市的能源心脏；在类型上属于**特殊城区**（见 [城区总览.md · 特殊城区](城区总览.md#特殊城区)）。
- 每颗骄阳之心对应一个核心区（详见[骄阳之心分布](../../../04-设定/05-隐秘真相/骄阳之心.md)）。
- 城市**至少需要一个核心区**才能运转和移动。
- 与核心区经 **链**（见 [§链与连接拓扑](#链与连接拓扑)）连通的城区被视为**移动城区**，记录在移动城市的连接网络中，并随城市整体移动。
- **停泊 ↔ 航行**状态切换属于**核心区**的固定职能（骄阳之心驱动整城离港 / 入港），见 [停泊与航行切换（核心区工作）](#停泊与航行切换核心区工作)。
- 建筑层上仍用同一套**城区**数据表示；停泊占格、连接拓扑等由 [地图与移动](../../02-地图与世界/地图与移动.md) 与程序数据结构补充，**不**另设「模块化城区」与「地图城区」两套定义。

## 停泊与航行切换（核心区工作）

**停泊 ↔ 航行**不是「移动城市抽象实体」对自身的无名工作，而是 **核心区**（拥有骄阳之心的特殊城区）所执行的**切换工作**：

| 项 | 说明 |
|----|------|
| **被工作对象** | **核心区**城区实例（`work_subject_ref` → 核心区 ID） |
| **工作对象** | 默认与**被工作对象**相同——由**核心区模块**执行离港 / 入港流程 |
| **玩家入口** | 玩家在指挥阶段对**移动城市**下达切换指令；行动表仍可用 `actor_kind=mobile_city`，但工作进度绑定**核心区** |
| **对照时长** | 各 **1 回合**（`dock_to_sail` / `sail_to_dock`）；见 [回合与行动表 · 工作中状态](../07-玩法循环/回合与行动表.md#工作中状态) |
| **多核心** | **各核心独立**——停泊↔航行切换由各自所属核心区独立控制。因为连接两个移动城市的前提是两者都处于**停泊**状态，所以连接为一个整体时两者的状态天然一致（sy-40 已定） |

切换完成后，整城 `mobile_city_dock_state` 更新，所有仍连接该核心网络的**移动城区**一并进入目标状态（见 [地图与移动 · 停泊与航行](../../02-地图与世界/地图与移动.md#停泊与航行)）。

## 链接与连接拓扑

城区之间的**物理连通**由 **城区链接**（简称 **链接**）表示：两座城区之间**至多一条**链接。

**链接是移动城市专属功能**——链接仅存在于玩家的**移动城市**内部，用于判断城区是否随城市整体移动及是否遭受分离处罚。**NPC 城市不存在链接**，其城区间的相邻关系仅看空间格子接壤。

**链接的作用范围**：**资源转移不须链接**——城区间资源流通仅须空间上相邻城区组成连续路径即可，同一共用仓库内的资源可被任一城区即时取用，无须中间运输步骤（见 [工作 · 城市内资源迁移](../07-玩法循环/工作.md#城市内资源迁移与城外运送已定)）。

链接的**修建与拆除均为即时操作**——消耗资源后立即生效，**不**经过多回合工作推进；**但**建立与拆除的**时机**由上下文行为决定（直接建造时即时结算；迁移城区时提前预约资源、在迁移完成时一并执行，见 [分离与拆解 · 迁移城区](分离与拆解.md#迁移城区)）。修建消耗 **15 金属**；拆除回收 **1 金属**（建造金属的 10%，向下取整）；拆除后在原址重建仅需 **12 金属**（80%）。拆除回收与重建折扣规则与 [设施](../设施层.md#残骸回收与重建已定) 一致。

| 项 | 口径 |
|----|------|
| **节点** | 城区实例（含核心区、一般城区、特殊城区） |
| **边（链接）** | 无向；城区 A 与城区 B 之间**至多一条**链接 |
| **修建链接** | 玩家发起 **连接** 操作，在合法占格对齐后即时新建一条链接（见 [分离与拆解 · 连接与分离](分离与拆解.md#玩家操作连接与分离)）；消耗 **15 金属**，即时生效；不须工程队、不须占用城区人口 |
| **拆除链接** | 玩家发起 **分离** 操作，即时拆除指定链接；不须工程队、不须占用城区人口 |

**拓扑示例（已定）**

- 三座城区**两两相邻**（三角形）：最多 **3** 条链接（A—B、B—C、A—C）。
- 三座城区**排成直线**：最多 **2** 条链接（仅相邻对 A—B、B—C；A 与 C **不**直连）。

```
三角形（3 链接）          直线（2 链接）

    B                      A — B — C
   / \
  A—C
```

### 移动城区与分离判定（已定）

- **移动城区**：经一条或多条**链接**的路径，能到达**至少一座核心区**的城区。
- **已分离**：**不能**经任何链接的路径连到**任意**核心区的城区 → 脱离随城移动，以建筑层实例落入世界地图（见 [分离与拆解 · 分离城区](分离与拆解.md#分离城区)）。
- **被动规则**：拆除链接后，程序对连通分量重算；凡落入「与所有核心区不连通」分量的城区，**自动**视为已分离（**不**须玩家逐座点名）。
- **多核心**：两个分量若**各自**仍连到**不同**核心区，则**两半都仍是移动城市**（各自随所属核心网络移动）；仅与所有核心区断开的分量才分离。

**再连接**：将已分离城区在合法位置对齐后，**修建**新链接接回核心网络；即时消耗资源并生效，不须工程队、不须占用城区人口；不可一键自动拼合。

再连接时，玩家须指定新位置与哪些相邻城区建立链接：

- **仅一个相邻格**：自动与该格建立链接，不须玩家点选。
- **多个相邻格**：玩家在可选相邻格中点选，敲定需要连接哪些格。

此外，对仍连接在网络中的**单个城区**，玩家可进入**「编辑连接」**模式：点选相邻城区格，切换是否设置链接。确认时按新设置**比较当前链接状态**，自动执行新建（原来没有→新设）、保留（原来已有→新设不变）、或拆除（原来已有→新设取消）。单次确认可同时新建、保留与拆除多条链接，结算规则与单条操作一致（新建 15 金属/条、拆除回收 1 金属/条，即时生效）。

（细则 sy-15，见 [分离与拆解 · 再连接前置](分离与拆解.md#再连接前置)。）

## 玩家操作：连接与分离（摘要）

玩家在指挥阶段**随时**可对己方城区（及 **占领城区** 的管理规则 **待定**）发起 **连接**（**修建链接**）或 **分离**（**拆除链接**）。完整流程、停泊/航行差异见 [分离与拆解 · 玩家操作：连接与分离](分离与拆解.md#玩家操作连接与分离)。

| 移动城市状态 | 连接 / 分离 |
|--------------|-------------|
| **停泊** | 即时消耗资源并生效；**不**叠加航行结构损耗惩罚 |
| **航行** | **允许**操作；**放弃城区**（分离）时，**脱离的城区** **结构完整度一次性下降 40%**（已定；**低于 20%** → **废墟**） |

## 多核心城市（第二章后）

> **引用设定**
>
> "移动城市不唯一。如果一个城市被分成两半，只要与任意核心区链接的，都是移动城市的一部分，只有不与核心区连接的，才被视为分离的城区。"

- 玩家在第二章从 [铁巢（IronNest）](../../../04-设定/03-地点与场景/铁巢.md) 获取**第二颗骄阳之心**后，可建立**第二个核心区**。
- **多核心优势**：
  - 城市可以被**分成两半**而两部分都仍属移动城市（只要各自连接到一个核心区）。
  - 允许构建**模块化城市结构**，例如分离部分城区执行特定任务后重新连接。
  - 提高城市整体的**容错性**（一个核心区失效不会导致全城瘫痪）。
- **移动判定**：只要城区经**链接**能到达**任意核心区**，它就属于移动城市的一部分，随所属连通分量整体移动。

## 粮食与相连分区

- 程序按 **链接** 连通分量（且该分量内**至少含一座核心区**）分配 **`mobile_city_id`**；**粮食周总结**按该 ID **独立**结算。链接决定城区归入哪个移动分区——此机制仅适用于玩家移动城市（NPC 城市不存在链接，其城区不参与 `mobile_city_id` 分区）。链接的跨功能用途仅限于此：其余如资源转移仅看空间相邻，不须链接。
- **不**跨 `mobile_city_id` 自动均分粮食；跨分区补给走资源迁移——城区间空间相邻即可形成运输路径，**不须**链接（见 [工作 · 城市内资源迁移](../../07-玩法循环/工作.md#城市内资源迁移与城外运送已定)）。
- 详见 [粮食与周总结（草稿）](../../../01-草稿/归档/粮食与周总结/README.md)。

## 与分离/拆解的关系

- **分离**（见 [分离与拆解.md](分离与拆解.md)）：城区**不能**经链接连到任意核心区；分离后的城区落入世界地图，但**不**改变状态（可为正常或废墟）。
- **拆解**（见 [分离与拆解.md](分离与拆解.md)）：削减结构完整度换取资源；与分离是**独立机制**。
