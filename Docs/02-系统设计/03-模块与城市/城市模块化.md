## 目标

定义城市由多个可拆分、重组、牺牲的城区组成的核心规则，使“取舍”成为玩家能清楚感受到的策略选择。

## 范围

- **包含**：城区定义（与 [建筑层](../01-核心系统/地图图层.md#建筑层与城区) 划等号）、**类别**（特殊 / 一般）、**状态**（正常 / 废墟）、**城区运作与居民人口**、核心区与骄阳之心机制、连接拓扑、**分离城区**与**拆解结构**（分轨）、再连接规则、城区稀有资产定位、城区类型示例、多核心城市。
- **不包含**：城区获取途径细则、模块数值、UI 交互流程。

## 详细说明

### 术语

- **[移动城市](../01-核心系统/地图与移动.md)**即**玩家主城**；地图上仅此一座可整体移动。
- **城区**：见下节；与 [地图图层 · 建筑层](../01-核心系统/地图图层.md#建筑层与城区)**划等号**——建筑层格上存储的内容就是城区这一机制对象，不再另设平行的「建筑类型」。
- **[骄阳之心（反应堆）](../../04-设定/05-隐秘真相/骄阳之心.md)**：驱动城市的能源核心，在游戏机制中体现为**核心区**（特殊城区的一种）。
- **核心区**：拥有骄阳之心的城区，是城市的能源心脏。
- **移动城区**：与核心区保持连接、属于移动城市一部分的城区。
- 荒野格上的城区、外部城市占格等，同为建筑层上的城区实例；未接入玩家移动城市连接网络的，见 [荒野地点](../02-资源循环/荒野地点.md)。

### 城区：定义与资产定位

- **城区**是 [建筑层](../01-核心系统/地图图层.md#建筑层与城区) 上的核心内容：每格建筑层至多一个城区实例（不可堆叠）。
- 玩法定位类似《杀戮尖塔》中的**遗物**：具有长期战略价值；**类别**决定能力形态，**状态**决定当前是否可用。
- **不可通过常规建造获得**：玩家不能像搭设施一样在空地「新建」一座城区；获得新城区须通过探索、事件、贸易、修复废墟、拆解回收或剧情等**稀有途径**（细则 **待定**，见 [待细化追踪 · OPEN-033](../../00-规范/待细化追踪.md)）。
- 玩家可**改造、修复、拆解、再连接**已有城区，但不可凭空创造城区**类别**。

### 类别：特殊城区与一般城区

城区按玩法职责分为两类（与**状态**正交，见下节）：

| 类型 | 职责 | 能力来源 |
|------|------|----------|
| **特殊城区** | 承载**固定、独特**的城市能力 | 城区模块**自带**；不依赖玩家在内部另建常规设施来体现核心功能 |
| **一般城区** | 提供**可建造设施的空地** | 玩家在空地内建造的 [设施](../01-核心系统/地图图层.md#图层一览)；随城市迁移 |

#### 特殊城区

- 每个特殊城区对应一种**不可替代**的模块能力，例如：
  - **核心区**（骄阳之心 / 能源心脏）
  - **巨炮**等城市武装或标志性功能模块
  - **城坞**（高效城区修复）、**通讯站**（即时通讯中枢）
  - 外部城市的**势力主城区**（城市领袖默认任职、势力特色能力）
- 特殊城区可携带 **1～3 个词条**（见下节），并与 [领袖与势力](../01-核心系统/领袖与势力.md) 中的任职条件对齐。
- **城区能力**（单位技能、宏观生产、设施激活等）主要由特殊城区**本体**提供。

#### 一般城区

- 一般城区提供若干**设施建造位**（空地 / 槽位；数量与布局 **待定**），玩家可在其中建造仓库、生产、防御等**常规设施**。
- 建于一般城区内的设施**随移动城市整体迁移**——城市航行或停泊切换时，这些设施仍在城内，无需在荒野重复建设。
- **与荒野设施的对比**：
  - **城内（一般城区）**：设施绑定**移动城市**，适合长期保留的产能、仓储与辅助功能；投资可随城带走。
  - **荒野（地图设施层）**：设施绑定**世界地图格子**（资源点、桥梁、驿站等）；城市离开后通常**无法整体带走**，资源点采尽或路线变更时往往只能**弃置**或**拆解回收**（见 [探索与扩张 · 设施建设](../01-核心系统/探索与扩张.md#设施建设)）。
- 一般城区**不**自带与特殊城区同级的独特模块能力；其战略价值来自玩家选择建造并随城携带的设施组合。

### 状态：正常与废墟

城区按当前可用性分为两种**状态**（与**类别**正交：特殊 / 一般均可处于正常或废墟）：

| 状态 | 说明 |
|------|------|
| **正常** | 城区按类别提供能力；可承载**居民**（若该类别 / 配置允许）；须有人口参与**城区运作**方能供能运转；一般城区内可建造设施 |
| **废墟** | **结构完整度**过低或遭击毁；**无法工作**（城区运作与供能停止）；**不可迁入**居民，已有居民**仅可迁出**；须修复为**正常**后才视为正常城区 |

#### 废墟

- **废墟**是对**状态为废墟的城区**的统称——不是与「城区」并列的另一种建筑层对象。
- **玩家辨识**：地图上须能**一眼认出**废墟与状态**正常**的城区——通过占格外观、图标、信息面板或揭示等级等（具体表现 **待定**），避免与正常城区、资源层 **遗迹** 混淆。
- **机制口径**：在修复完成前，程序与规则**只**将其视为**废墟**状态的城区，**不**按正常城区结算运作、供能、领袖任职（若仍任职 **待定**）、连接网络等能力。
- **结构完整度与废墟阈值**：
  - 每个城区维护 **结构完整度**（0～100%，程序字段名 **待定**），表示当前建筑结构剩余比例；与 [拆解结构](#拆解结构) 中的累计损伤联动（完整度下降 ↔ 损伤上升 **待定**换算）。
  - 当 **结构完整度低于 20%** 时，城区**强制**转为**废墟**状态（战斗击毁、事件或主动拆解导致的过低完整度均适用）。
- **废墟状态下的人口**：**无法工作**（不参与城区运作）；**不允许迁入**新居民；已在该城区的居民**仅允许迁出**至其他合法城区（见 [人口与迁移 · 废墟与居民迁出](../02-资源循环/人口与迁移.md#废墟与居民迁出)）。
- **来源**：
  - **结构完整度 低于 20%**（含主动拆解结构、交战损伤、灾害等）；
  - **地图默认刷新**：荒野格子上也可**默认刷新**已为废墟状态的城区（未修复前视为残骸占格）。
- **修复为正常**：须花费 **金属** 等资源并完成对应 [工作](../02-玩法循环/工作.md)（细则 **待定**），将**结构完整度**恢复至 **20% 及以上**且状态由**废墟**变为**正常**后，方**视为**正常城区——可迁入居民、参与城区运作、启用类别能力并接入移动城市连接（若合法）；城坞等可影响修复效率。
- **拆解**（**拆解结构**，见下节）：对城区**结构**进行拆解，用**累计损伤**换取金属等资源；与**分离城区**是**不同概念**。
- 与 [资源层 · 遗迹](../02-资源循环/荒野地点.md#遗迹与废墟术语强制区分) **不是**同一类存在：遗迹为前文明资源点，**不可**修复为城区。

### 城区运作与居民人口

状态为**正常**的城区要维持**运转与供能**（模块能力生效、一般城区内设施持续运行、与 [能源](../02-资源循环/四种核心资源.md) 等资源的衔接 **待定**），须有人口参与**城区运作**——由劳动力按规则到岗，**不是**城区自动常驻生效。

| 概念 | 说明 |
|------|------|
| **城区人口**（显示在城区上） | 只表示**居民**——在该城区居住、占用承载上限的人口；**不**表示已指派上岗的**工作人口** |
| **城区运作** | 维持城区供能与运转所需的劳动；在玩家城市由玩家指定**哪一类人口**承担（见 [人口与迁移 · 玩家：城区工作分配](../02-资源循环/人口与迁移.md#玩家城区工作分配)） |
| **废墟** | **无法工作**；城区运作与供能**停止**；**不可迁入**居民，已有居民**仅可迁出** |

- **非玩家势力**的外部城市：每城仅 **1 名城市领袖**，城区运作与人口调度由领袖**统管抽象**，不向玩家暴露城区级工作分配界面（见 [领袖与势力 · 非玩家城市](../01-核心系统/领袖与势力.md#非玩家城市与人口调度)）。
- **玩家移动城市**：须主动管理全城 [工作](../02-玩法循环/工作.md)，并为**每个城区**规定完成该城区运作的**人口类型**（归属 / 文化类别 **待定** 完整名单）。

### 城区词条

- 每个城区携带 **1～3 个词条**，描述其文化、设施与人口气质（具体词条表 **待定**）。
- **城市领袖**携带与之对应的**特质词条**；领袖须任职于词条要求得到满足的城区（匹配规则见 [领袖与势力 · 城区词条与领袖特质](../01-核心系统/领袖与势力.md#城区词条与领袖特质)）。
- 外部城市的**特殊城区**（势力主城区）在领袖未被招募前，**默认满足**该领袖的词条要求；玩家改造或更换任职城区后须主动对齐词条。

### 城区能力与领袖能力

- **特殊城区能力**：由城区**本体**提供——单位技能、宏观资源生产、模块级功能等。
- **一般城区**：不单独提供模块级独特能力；生效的是其内部**已建造设施**的能力（仍受设施层规则约束）。
- **领袖能力**：为领袖名下人口提供消耗减免、战斗或生产强化、归属转化等——见 [领袖与势力 · 能力：领袖与城区](../01-核心系统/领袖与势力.md#能力领袖与城区)。
- 领袖能力、特殊城区能力与一般城区内设施效果**作用范围不同**；同一效果是否可叠加 **待定**。

### 核心区与骄阳之心

> **引用设定**
>
> "反应堆在游戏机制上体现为「核心区」，与核心区相连接的城区为移动城区。"

- **核心区**对应设定中的**骄阳之心（反应堆）**，是城市的能源心脏；在类型上属于**特殊城区**。
- 每颗骄阳之心对应一个核心区（详见[骄阳之心分布](../../04-设定/05-隐秘真相/骄阳之心.md)）。
- 城市**至少需要一个核心区**才能运转和移动。
- 与核心区**物理连接**的城区被视为**移动城区**，记录在移动城市的连接结构中，并随城市整体移动。
- 建筑层上仍用同一套**城区**数据表示；停泊占格、连接拓扑等由 [地图与移动](../01-核心系统/地图与移动.md) 与程序数据结构补充，**不**另设「模块化城区」与「地图城区」两套定义。

### 连接拓扑

- 与**核心区**保持连接的城区属于**移动城区**，记录在移动城市的连接网络中，随城整体移动（见 [核心区与骄阳之心](#核心区与骄阳之心)）。
- **再连接**：玩家将已**分离**的城区在合法位置对齐后，通过连接工作接回核心区网络；不可一键自动拼合（对应多回合 [工作](../02-玩法循环/工作.md)，细则 **待定**，OPEN-034）。

### 分离城区

**分离**指**解除城区与移动城市的链接**（脱离核心区连接网络），**不是**拆解结构、**不是**用损伤换资源。

| 要点 | 说明 |
|------|------|
| **操作本质** | 断开拓扑连接；城区实例仍保留，类别与（分离前）状态一般不变 |
| **结果** | 该城区不再随移动城市整体移动；以**建筑层城区**落入世界地图（占格 **待定**） |
| **典型用途** | 让路通过狭窄地形、临时派出城区执行任务、牺牲连接以保全其余部分等 |
| **与拆解** | **分离**不必然产生回收资源；**拆解结构**不必然导致分离（可原地拆解 **待定**） |

- **分离规则（被动）**：若连接网络被切断，凡**不能连接到任意核心区**的城区，一律视为**已分离**，落入世界地图。
- **调整城市形态**（收窄、改布局）：以**先分离、再连接**为主流程，须玩家**手动逐步操作**（多回合工作）；**不**与「拆解结构换资源」混为同一操作。

### 拆解结构

**拆解结构**指对城区**建筑结构**进行拆解：**累计损伤**可换取 **金属** 等资源；与**分离城区**是**独立机制**。

| 要点 | 说明 |
|------|------|
| **操作本质** | 削减城区**结构完整度**，按规则将部分损伤转化为可回收资源 |
| **结构完整度** | 0～100%；与累计损伤联动（**待定**字段映射）；**低于 20%** 时城区转为**废墟** |
| **换资源** | 主动拆解或允许损伤结算时，按完整度 / 损伤比例获得金属等（公式 **待定**） |
| **转废墟** | **结构完整度 低于 20%** → 状态**废墟**（见 [废墟](#废墟)）；无法工作；人口仅可迁出、不可迁入 |
| **与分离** | 可在仍连接移动城市时拆解，也可对已分离的城区拆解；**分离不会自动**触发拆解换资源 |

- 状态已为**废墟**的城区仍可继续**拆解结构**以回收剩余资源（比例 **待定**），与「修复为正常」为不同选择。

### 多核心城市（第二章后）

> **引用设定**
>
> "移动城市不唯一。如果一个城市被分成两半，只要与任意核心区链接的，都是移动城市的一部分，只有不与核心区连接的，才被视为分离的城区。"

- 玩家在第二章从 [铁巢](../../04-设定/03-地点与场景/铁巢.md) 获取**第二颗骄阳之心**后，可建立**第二个核心区**。
- **多核心优势**：
  - 城市可以被**分成两半**而两部分都仍属移动城市（只要各自连接到一个核心区）。
  - 允许构建**模块化城市结构**，例如分离部分城区执行特定任务后重新连接。
  - 提高城市整体的**容错性**（一个核心区失效不会导致全城瘫痪）。
- **移动判定**：只要城区与**任意核心区**保持连接，它就属于移动城市的一部分，随城市整体移动。

### 城区类型示例

#### 特殊城区

| 类型 | 定位 |
|------|------|
| **核心区** | 拥有骄阳之心的特殊城区；与核心区连接的城区才是移动城区 |
| **巨炮**等特殊武装 / 功能模块 | 城市级独特战斗或功能能力（示例 **待定** 完整名单） |
| **城坞** | 城区修复中心；高效修复、允许重叠修复未连接城区的特殊规则 |
| **通讯站** | 即时通讯中枢；为离城单位提供信号覆盖 |
| **势力主城区**（外部城市） | 城市领袖默认任职；势力特色城区能力；招募后定义不变（见 [领袖与势力](../01-核心系统/领袖与势力.md)） |

下列模块若仅提供固定加成、不可在内自由铺建常规设施，也归入**特殊城区**（如部分**居民区**仅承担人口承载 **待定** 分类）。

#### 一般城区

| 定位 | 说明 |
|------|------|
| **设施空地** | 提供设施建造位；所建设施随城市移动 |
| **可改造** | 玩家选择仓库、生产、辅助等设施组合，形成「随城携带」的功能区 |

> **城坞特殊规则**：一般情况下城市不允许重叠，但城坞属于额外条件——允许修复与自身重叠但未连接在同一城市的城区。

> **通讯站特殊规则**：即时通讯需要信号覆盖。离开城市的单位能否使用即时通讯取决于是否有通讯站支持。
