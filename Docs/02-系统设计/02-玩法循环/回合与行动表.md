## 目标

定义回合制的时间推进方式、每回合的阶段划分，以及单位如何通过行动表与可叠加指令完成多回合任务。

## 范围

- **包含**：回合阶段、行动表规则、环境结算顺序、指令队列与多回合执行。
- **不包含**：单位属性对执行效率的具体数值、程序实现、界面交互细节。

## 参考

整体节奏类似《文明 6》的回合制，但阶段划分与指令系统不同。

## 详细说明

### 回合阶段

每个回合按以下顺序推进：

| 阶段 | 说明 |
|------|------|
| 玩家指挥 | 回合开始时游戏等待玩家输入；玩家可自由修改自身阵营单位的指令。指挥环节结束后，本回合进入行动阶段。 |
| 玩家行动 | 玩家阵营单位按行动表顺序，执行当前回合的指令。 |
| AI 行动 | 每个[外部城市](../01-核心系统/外部城市与组织关系.md)按预生成顺序轮流行动，直至全部城市各行动一次。动画可跳过或加速；视野区外的行动直接跳过动画。 |
| 环境行动 | 非玩家、非城市的自然机制在此结算，如太阳移动、环境对单位的影响。 |

```mermaid
flowchart LR
    A[玩家指挥] --> B[玩家行动]
    B --> C[AI 行动]
    C --> D[环境行动]
    D --> A
```

### 行动表

行动表是一个队列，决定当前回合内单位的行动先后顺序。

- 任意一方在行动阶段中，单位均按行动表顺序执行**当前回合**的指令。
- 新输入的指令默认插入行动表指定位置；玩家可从本回合起调整后续行动顺序。
- AI 程序自行生成并维护各自的行动表。
- 玩家[移动城市](../01-核心系统/地图与移动.md)、外部城市及其下属单位，在逻辑与实现上均适用同一套行动表规则。

### 环境结算顺序

环境行动阶段固定顺序：

1. **太阳移动**（优先）。
2. **环境效果结算**：从阴影侧向太阳侧，按顺序逐格结算；面向太阳时，同一排的格子从右向左依次结算。（太阳在六边形地图上斜向移动。）

### 指令系统

- 每个单位存有一套**指令队列**，不是单一指令。
- 指令可叠加、可组合，构成一条完整的任务规划。例如：派出队伍时可一次性设定全程行为。
- 单位在未遭遇意外时，按指令规划自行推进，无需玩家逐步操作。（意外包括被 [响应](../01-核心系统/地图图层.md#响应) 打断等。）
- 一条指令可跨越多个回合才能完成；单位对指令的执行效率由自身属性决定（属性规则另行讨论）。
- 回合开始时，玩家可修改指令；单位在后续若干回合内按顺序执行，直至全部完成，或在后续回合中被再次修改。
- 玩家指挥环节结束后，各单位在行动阶段按行动表顺序，从指令队列中取出并执行当前回合应完成的部分。
