## 定位

**工作**是跨系统的**机制模块**（不单列为与「地图」「队伍」并列的顶层系统）：凡需占用时间、以**工作进度**推进的任务，均通过同一套工作状态与时长规则结算；到达配置进度节点时发放**工作成果**。

工作可作用于：

| 被工作对象（示例） | 工作对象（执行者，示例） | 典型工作 |
|--------------------|--------------------------|----------|
| 移动城市 | 移动城市 | 停泊 ↔ 航行切换 |
| 资源点 / 工地 | 工程队 | 建造采集设施 |
| 资源点 | 勘探队 | 勘探（储量调查） |
| 设施实例 | 工程队 | 运维设施 |
| 库存节点 | 运输队 | 装货 / 卸货 |
| 城区实例 | 工程队 / 城坞 | 修复城区（废墟 → 正常） |
| 城区实例 | 工程队 / 玩家指挥 | **拆解结构**（累计损伤换资源） |
| 城区 / 连接边 | 玩家指挥 + 工程 | **分离城区**、**再连接**（见 [城市模块化 · 分离城区](../03-模块与城市/城市模块化.md#分离城区)） |
| 城区实例 | **城区运作人力** | 维持城区供能与运转（见 [城市模块化 · 城区运作](../03-模块与城市/城市模块化.md#城区运作与居民人口)） |

## 与行动、指令的关系

- **行动**：任意可指挥实体（移动城市、外出队伍、具备行动资格的设施等）在 [行动表](./回合与行动表.md#行动表) 中获得回合内执行顺序。
- **指令**：玩家在指挥阶段为实体填写任务目标；其中需多回合完成的条目进入 **工作中** 状态。
- 即时能力（移动一格、单次攻击等）不走工作进度；[交战系统](../01-核心系统/交战系统.md) 中的交战结算亦不走工作进度。

## 工作进度与工作成果

工作拆成两个概念：

| 概念 | 说明 |
|------|------|
| **工作进度** | 一项工作进行到哪一步；UI 常以百分比表示（程序字段见 [完成度](./回合与行动表.md#完成度与推进)） |
| **工作成果** | 工作进度到达**指定阈值**时结算的回馈；由工作类型配置 |

### 一般规则

- 每类工作在 `work_type_config` 上配置一条或多条 **工作成果**，每条绑定一个 **进度阈值**（如 `0.5`、`1.0`）。
- 工作进度**首次达到**某阈值时，结算该条工作成果**一次**；已发放的成果不因中断或倒退而重复发放（除非工作被重置，规则 **待定**）。
- **一般工作**仅配置 **100%** 阈值：进度满时发放该工作的完整成果（如设施建成、装货完成、停泊切换完成）。
- 工作进行中只累积进度；**不**在未满阈值时提前发放最终成果，除非该阈值已在配置中声明。

### 特例：勘探与分级揭示

[勘探队](../01-核心系统/单位类型与视野.md#2-勘探队模板) 对资源点执行 **勘探** 类工作时，可在进度中途发放阶段性成果，与 [资源点揭示等级](../01-核心系统/单位类型与视野.md#资源点揭示等级) 联动：

| 进度阈值 | 工作成果（示例） | 说明 |
|----------|------------------|------|
| **50%** | **初级揭示** | 如揭示资源点**种类**（或等价的部分信息） |
| **100%** | **揭示储量** | 该勘探工作的完整成果 |

- 侦察队通过**视野能力**揭示种类、勘探队通过**勘探工作**逐级推进，二者可并存；同一资源点已满足的揭示等级**不降级**。
- 其它工作类型是否配置中途成果，由各类 `work_type_config` 决定，**不**默认存在 50% 节点。

## 移动城市与城区工作

城内工作分两类口径：

| 口径 | 说明 | 谁管理 |
|------|------|--------|
| **实体工作** | 工程队修复、城坞运维、城内生产等多回合 [工作进度](#工作进度与工作成果) 任务 | 玩家通过 [行动表](./回合与行动表.md) 下达指令 |
| **城区运作** | 维持各城区供能与模块 / 设施运转所需的**劳动力** | **玩家城市**：为每城区指定由**哪一类人口**承担（见 [人口与迁移 · 玩家：城区工作分配](../02-资源循环/人口与迁移.md#玩家城区工作分配)）；**非玩家城市**：由城市领袖抽象统管 |

- 城区上显示的**居民**人数**不**自动计入运作人力；运作不足时可导致城区停摆（与 [城市模块化](../03-模块与城市/城市模块化.md#城区运作与居民人口) 联动，细则 **待定**）。
- 外出队伍执行的荒野工作（建造采集设施、勘探等）走队伍 **工作对象** 规则，与城区运作人力**分开**结算。

## 核心规则（摘要）

完整字段、默认时长、中断与恢复见 [回合与行动表 · 工作中状态](./回合与行动表.md#工作中状态)。

| 概念 | 说明 |
|------|------|
| **工作对象** | 执行工作的实体（队伍、移动城市等） |
| **被工作对象** | 工作作用在其上的实体；可与工作对象相同 |
| **对照时长** | `work_type_config.base_duration_turn` |
| **实际时长** | 对照时长 ÷ 工作效率（锁定后不重算） |
| **工作进度** | 有效推进回合 / 实际时长；达阈值时检查并发放 **工作成果** |
| **工作成果** | 工作类型 SO 上按进度阈值配置的效果列表（`work_outcomes` 或等价结构，**待定**） |
| **可中止** | 由工作类型 `allow_suspend` 决定；中止后是否保留工作进度 |
