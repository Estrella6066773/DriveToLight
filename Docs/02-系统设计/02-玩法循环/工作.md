## 定位

**工作**是跨系统的**机制模块**（不单列为与「地图」「队伍」并列的顶层系统）：凡需占用时间、以完成度推进的任务，均通过同一套工作状态与时长规则结算。

工作可作用于：

| 被工作对象（示例） | 工作对象（执行者，示例） | 典型工作 |
|--------------------|--------------------------|----------|
| 移动城市 | 移动城市 | 停泊 ↔ 航行切换 |
| 资源点 / 工地 | 工程队 | 建造开采站 |
| 设施实例 | 工程队 | 运维设施 |
| 库存节点 | 运输队 | 装货 / 卸货 |
| 城区实例 | 工程队 / 城坞 | 修复城区 |
| 分离中的城区 | 玩家指挥 + 工程 | 拆解、再连接（见 [城市模块化](../03-模块与城市/城市模块化.md)） |

## 与行动、指令的关系

- **行动**：任意可指挥实体（移动城市、外出队伍、具备行动资格的设施等）在 [行动表](./回合与行动表.md#行动表) 中获得回合内执行顺序。
- **指令**：玩家在指挥阶段为实体填写任务目标；其中需多回合完成的条目进入 **工作中** 状态。
- 即时能力（移动一格、单次攻击等）不走工作完成度；[交战系统](../01-核心系统/交战系统.md) 中的交战结算亦不走工作完成度。

## 核心规则（摘要）

完整字段、默认时长、中断与恢复见 [回合与行动表 · 工作中状态](./回合与行动表.md#工作中状态)。

| 概念 | 说明 |
|------|------|
| **工作对象** | 执行工作的实体（队伍、移动城市等） |
| **被工作对象** | 工作作用在其上的实体；可与工作对象相同 |
| **对照时长** | `work_type_config.base_duration_turn` |
| **实际时长** | 对照时长 ÷ 工作效率（锁定后不重算） |
| **完成度** | 有效推进回合 / 实际时长 |
| **可中止** | 由工作类型 `allow_suspend` 决定；中止后是否保留完成度 |
