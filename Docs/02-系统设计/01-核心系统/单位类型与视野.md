## 目标

定义游戏中队伍模板、能力配置与视野机制，明确资源点揭示系统与能力化实现边界。

## 范围

- **包含**：单位类型定义、基础属性、视野机制、资源点揭示等级。
- **不包含**：具体数值平衡、AI行为逻辑、战斗系统（若有）。

## 详细说明

### 程序设计原则

> **所有单位类型都需要单独使用SO（ScriptableObject），可以被策划配置，并且能力也是可配置项，而不被写死在程序中。**
> 
> **允许变种单位使用不同结构的SO，并且可以兼容性读取同一基类的SO。**

- 单位模板通过 **ScriptableObject（SO）** 配置，而非硬编码。
- 单位能力（视野、移动速度、武装状态、建造、搬运、攻击等）均为可配置项。
- 支持创建**变种单位**：使用不同结构的SO，但继承同一基类，确保兼容性。
- 策划或开发者可通过引擎内配置自由调整单位参数，无需修改程序代码。

### 能力通道模型

- 程序层仅提供能力执行通道，不限制具体模板必须具备哪些能力。
- 能力是否启用与参数细节由能力 SO 决定。
- 当前已确认的基础能力通道：
  - 侦察能力（揭示种类）；
  - 勘探能力（揭示储量）；
  - 搬运能力（负载与装卸）；
  - 建造能力（建设与维修）；
  - 攻击能力（打击与反击）。

### 单位基础属性

| 属性 | 说明 | 配置类型 |
|------|------|----------|
| **移动速度** | 单位在地图上的移动效率 | 数值（可配置） |
| **武装状态** | 是否有武装/战斗能力 | 布尔（可配置） |
| **人数规模** | 单位承载的人口数量 | 数值（可配置） |
| **视野范围** | 单位能揭示的地图范围 | 数值（可配置） |
| **特殊能力** | 单位特有的功能能力 | SO引用（可配置） |

### 默认编制与人数衰减影响

> 以下为当前默认值；策划可在配置文件中调整。

| 队伍类型 | 默认人数 |
|----------|----------|
| 侦察队 | 3 人 |
| 勘探队 | 5 人 |
| 运输队 | 20 人 |
| 工程队 | 50 人 |

- 队伍人数可因战斗或事件减少，且不会立即解散。
- 人数下降会同步影响以下能力：
  - **视野范围**：人数越低，视野范围越小。
  - **工作效率**：参与 [工作中](../02-玩法循环/回合与行动表.md#工作效率与时长) 结算时，按 **当前人数 / 满编人数**（满编 = 模板默认编制）作为乘法减值；效率越低，实际工作时长越长。
  - **负载上限**：人数越低，可运输与可处理的资源上限越低。
  - **攻击效果**：仅对具备武装能力的队伍生效，人数越低，攻击效果越弱。

### 单位模板清单（默认预设）

#### 1. 侦察队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **极快** | 迅速探索大片区域 |
| 武装状态 | **无武装** | 无法主动交战；遇敌应对见 [队伍系统 · 遭遇敌人时的行为](./队伍系统.md#遭遇敌人时的行为) |
| 人数规模 | **少** | 轻装上阵，机动性强 |
| 视野范围 | **广** | 可揭示大范围内的资源点 |

**默认能力**：
- **揭示资源点种类**：进入视野内的资源点，可揭示其**种类**（是什么类型的资源点）。
- 无法揭示具体储量（需勘探队）。

#### 2. 勘探队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **较快** | 比侦察队慢，但仍有良好机动性 |
| 武装状态 | **无武装** | 无法战斗，需护卫或避开危险区域 |
| 人数规模 | **中** | 携带勘探设备，需要一定人力 |
| 视野范围 | **一般** | 视野不如侦察队广阔 |

**默认能力**：
- **揭示资源点储量**：进入视野内的资源点，可揭示其**具体储量**（有多少资源可供采集）。
- 需先由侦察队或其他方式揭示资源点种类。

#### 3. 运输队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **均衡** | 平衡速度与载重 |
| 武装状态 | **无武装** | 需要护卫保护 |
| 人数规模 | **中** | 负责运输作业 |
| 视野范围 | **一般** | 基础视野 |

**默认能力**：
- **运载物资**：可在资源点与城市之间运输资源。
- **载重能力**：有最大载重量限制，超载会影响移动速度。
- **装货 / 卸货**：各占一项 [工作中](../02-玩法循环/回合与行动表.md#工作中状态) 任务；对照时长各 **1 回合**，实际时长受 [工作效率](../02-玩法循环/回合与行动表.md#工作效率与时长) 影响。

#### 4. 工程队模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **均衡** | 携带工程设备，速度适中 |
| 武装状态 | **无武装** | 需要保护 |
| 人数规模 | **大** | 需要大量人力进行建设 |
| 视野范围 | **一般** | 基础视野 |

**默认能力**（两类**工作**，均占用 [工作中](../02-玩法循环/回合与行动表.md#工作中状态) 进度）：

| 工作 | 说明 | 玩家指挥 |
|------|------|----------|
| **建造设施** | 在合法位置新建设施 | 二级选设施类型；须指定 **被工作对象**（资源点 / 工地）；**工作对象**为工程队 |
| **运维设施** | 对已存在设施运维 | **被工作对象**为目标设施；**工作对象**为工程队 |

**建造设施 · 二级选择**

- 一级：指令类型 = 建造设施。
- 二级：选择具体**设施类型**（如桥梁、资源点开采站、驿站等）；各类型对应独立 SO 配置。
- **建造条件因设施而异**，程序通过 `facility_type_config` 读取，不在代码中写死。示例：
  - **桥梁**：须满足 [地图图层 · 裂隙与桥梁](./地图图层.md#影响规则示例)（裂隙格 + 对向格条件）。
  - **资源点设施**（如矿区/果地开采站）：须落在对应 [资源层](./地图图层.md#图层一览) 资源点，且满足储量/揭示等级等条件；对照建造 **3 回合**（**细则待定**，OPEN-028）。
- 条件不满足时，指挥阶段即不可下达或提示失败（**UI 反馈方式待定**）。

**运维设施**

- 针对**已建造**的设施执行运维工作；对照时长默认 **3 回合**，实际时长受 [工作效率](../02-玩法循环/回合与行动表.md#工作效率与时长) 影响。
- 城坞对**城区**的修复效率高于工程队；工程队侧重荒野设施与资源点设施（**分工细则待定**，OPEN-029）。

程序实现见 [队伍与战斗数据结构 · 工程队工作](../../03-程序设计/03-数据字典/队伍与战斗数据结构.md#工程队工作建造与运维)。

#### 5. 飞讯模板

| 属性 | 数值特征 | 说明 |
|------|----------|------|
| 移动速度 | **极快** | 一次性快速传递单位 |
| 武装状态 | **无** | 无任何武装 |
| 人数规模 | **无** | 非人员单位，信息载体 |
| 视野范围 | **独立视野** | 有自身的独立视野，但不提供战略视野 |

**默认能力**：
- **更新指令**：一次性单位，从核心区出发，飞往指令对应的单位在预测中的目的地。
- **搜索机制**：
  - 在目的地搜寻目标单位。
  - 如果在目的地没找到，会在周围搜索。
  - 搜索持续一定回合数后消失。
- **视野特性**：
  - **不提供战略视野**：不会为主城或其他单位提供持续视野。
  - **有独立视野**：在飞行路径上有自身视野，用于搜索目标单位。

> **飞讯机制背景**：不支持或未启用即时通讯的单位会定期向核心区发送飞讯，飞讯记录着这一次要同步的视野信息。支持且启用即时通讯的单位，会在存在通讯站的情况下及时同步信息，否则不同步（同步失败）。详见 [通讯与飞讯系统](./通讯与飞讯系统.md)。

### 资源点揭示等级

| 揭示等级 | 揭示内容 | 需要单位 |
|----------|----------|----------|
| **不揭示** | 仅知道此处有资源点（图标标记） | 无 |
| **揭示种类** | 知道是什么类型的资源点（矿区/果地/废墟等） | 侦察队 |
| **揭示储量** | 知道具体有多少资源可供采集 | 勘探队 |

**揭示流程**：
1. 单位移动或探索时，将资源点纳入视野。
2. 根据单位能力判定揭示等级。
3. 侦察队先揭示种类，勘探队后揭示储量。
4. 揭示信息通过飞讯或即时通讯同步回主城。
