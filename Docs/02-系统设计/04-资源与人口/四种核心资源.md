## 目标

定义游戏基础资源类型、用途与来源，降低上手门槛的同时保留策略深度。

## 范围

- **包含**：四种资源的定义、用途、典型来源；食物与人口的关系原则；**首版产出 / 消耗数值锚**（见下文各节）。
- **不包含**：库存上限总表、贸易价、章节刷新密度；可另开 [04-数值框架/](../04-数值框架/) 补全。

## 详细说明

基础资源包括：

| 资源 | 主要用途 | 典型来源 |
|------|----------|----------|
| **金属** | **设施**与**城区**的建造、修补、升级；**资产生产** | 矿藏、[废墟状态城区（拆解回收）](./荒野地点.md#遗迹与废墟术语强制区分)、外部城市 |
| **食物** | **维持人口**；远征队补给 | 果地、外部城市、后期 [温室](./温室.md) |
| **能源** | **城区自身**维持（**城区能力**仅**特殊城区**）；**设施**独立运行；**资产生产**；后期温室 **能源→食物** | 遗迹、外部城市 |
| **人口** | **居民**安置、**城区运作**劳动力、组建队伍；见 [人口与迁移 · 居民与工作人口](./人口与迁移.md#居民与工作人口) | [村镇](./荒野地点/村镇.md)（资源点储量）、外部城市 |

### 四类资源产出锚（已定）

共用 **3 回合** / 工作量 **30**（满编工程队专精一拍）节拍；与 [消耗锚](#金属与能源消耗锚已定)、[粮食产量阶段](./荒野地点.md#粮食产量阶段已定) 对齐。

| 资源 | 设施 | 单次产出 | 输入 | 储量区间 | 数量感（例） |
|------|------|----------|------|----------|--------------|
| **食物** | [果园](./荒野地点/果园.md) | **25** | 果地储量 | 果地 **400～700** | 前期 **3～5** 座 ≈ **4** 区 |
| **食物** | [温室](./温室.md) | **150** | **50** 能源 | —（城内） | 后期 **1～2** 座 ≈ **7** 区 |
| **金属** | [矿区](./荒野地点/矿区.md) | **25** | 矿藏储量 | 矿藏 **400～700** | 前期 **2～3** 座撑航行修复 |
| **能源** | [能源站](./荒野地点/能源站.md) | **40** | 遗迹储量 | 遗迹 **500～900** | 前期 **1**、后期 **2～4**（视温室） |
| **人口** | [征兵办](./荒野地点/征兵办.md) | **15** 人 | 村镇储量 | 村镇 **60～120** 人 | 前期 **2～3** 座扩编 |

**建造金属（采集 / 温室 · 已定）**：果园 / 矿区 / 能源站 / 征兵办各 **30** 金属；温室 **80** 金属；屋舍 **20** 金属。建造对照 **3** 回合（与既有 `build_turns` 一致）。

**拆解回收（城区结构 · 已定）**：仅**主动拆解工作**回收金属——工作量 **30**，完整度 **−10%**，获得 **10** 金属（为修复单价之半，鼓励修而非拆）。**被动损伤**（负载、交战、航行放弃等）**不**回收金属。细则见 [分离与拆解](../03-图层与地点/建筑层/分离与拆解.md)。

### 仓储容量（已定框架）

金属、食物、能源**共用**同一份仓库空间，**不**按资源种类分仓上限。玩家须在有限容量内决定囤粮、囤金属还是囤能源。

| 项 | 口径 |
|----|------|
| **占用** | 节点内 `占用 = 金属存量 + 食物存量 + 能源存量`（**1:1**，单位同为资源数量） |
| **上限** | `占用 ≤ capacity_max`；超限**不可入库**（采集 / 转化 / 卸货拒绝并提示；首版） |
| **人口** | **不**占仓库；走 [居民承载](../03-图层与地点/建筑层/城区总览.md#居民承载) |
| **队伍载荷** | **不**并入城仓；与城市仓库**分池** |
| **扩容** | 一般城区可有**基础容量**；[仓储类 · 仓库](../03-图层与地点/设施层.md#仓储类仓库) 叠加抬高 `capacity_max`（数值 **待补**，sy-23） |
| **废止** | 建材仓 / 粮仓 / 燃料库**分种类上限**；改为统一 **仓库** |

细则与 CMS 策略见 [城市管理系统 · 资源存储分配](./城市管理系统.md#资源存储分配)。

### 食物与人口

- **食物**用于维持城市人口总量；日常编制分配、队伍组建**不**等同于把人口从总量中「消耗掉」（见 [人口与迁移 · 人口不是消耗品](./人口与迁移.md#人口不是消耗品)）。
- **粮食结算**在 **周总结**（每 7 回合、环境行动结束后）统一进行，**不**每回合扣粮。规则详见 [粮食与周总结（草稿）](../../01-草稿/粮食与周总结/README.md)（**无饥饿版**）。
- **供应不足**时：本周期**未分到** 1 粮食/人/周的个体 → **⌊人数×50%⌋ 减员**；**无**饥饿中间态、**无**跨周 cohort。
- **结算单元**：按 `mobile_city_id`（相连分区）独立；**城市仓库扣减**与**队伍载荷粮食**分池。
- **废止**：`team_food_upkeep_enabled` 每回合队伍扣粮（原 OPEN-042 每回合路径）；队伍粮食并入周总结载荷容器。
- **产量**：见上表；城内物品转化（工坊等）**不**产粮。

### 金属与能源

- **金属**面向**结构性与生产性**消耗：**设施**与**城区**的建造、修补、升级，以及资产生产；航行 **负载成本** 与主动修复以金属结算（见下节）。
- **能源**面向**运转性**消耗，分轨：
  - **特殊城区日常开销**（工作区开启期间，**每 3 回合**结算）
  - **激活开销**（主动能力触发时）
  - **设施**：首版荒野采集与仓储**无**持续耗能；[温室](./温室.md) 仅在转化完成时扣能源
  - **不**设整城移动**即时**耗能（首版）；与负载成本分轨

### 金属与能源消耗锚（已定）

与 [粮食产量阶段](./荒野地点.md#粮食产量阶段已定) 共用 **3 回合**生产/结算节拍；住宅 **50**、前期约 **4** 区 / 后期约 **7** 区。

#### 能源 · 日常开销（每 3 回合，工作区开启时）

| 模块 | `routine_cost`（每 3 回合） | 约合周耗 | 备注 |
|------|------------------------------|----------|------|
| **核心区** | **15** 能源 | ~35 | **不可关闭**；城市保底耗能 |
| **通讯站** | **10** 能源 | ~23 | 维持被动视野 |
| **学院** | **8** 能源 | ~19 | 维持被动 |
| **城坞** | **10** 能源 | ~23 | 开启时 |
| **巨炮** | **留空** | 0 | 仅激活扣费 |

**激活开销（单次）**

| 模块 | `activation_cost` |
|------|-------------------|
| **通讯站**主动视野 | **20** 能源 |
| **巨炮**开火 | **80** 能源 + **40** 金属 |

**前期（例 · 4 区，仅核心 + 通讯站开启）**：约 **58** 能源/周 → **1** 座 [能源站](./荒野地点/能源站.md) 专精（~93/周）可覆盖并留缓冲。  
**后期（例 · 核心 + 通讯 + 1 温室）**：约 **175** 能源/周 → 须 **2** 座能源站专精；**2** 温室则约 **292**/周 → **3～4** 座。

能源站：单次 **40** 能源 / 工作量 **30**；见 [能源站](./荒野地点/能源站.md)。

#### 金属 · 负载与修复

| 参数 | 首版默认 | 说明 |
|------|----------|------|
| `move_grids_per_load_step`（x） | **3** | 航行每前进 **3** 格结算一步 |
| 每步完整度下降 | **−2%** / 城区 | 各连接城区**分别**结算 |
| 修复单价 | **2** 金属 / **1%** 完整度 | 一步损伤当量 = **4** 金属（=`load_repair_cost` y） |
| `load_repair_kind` | `metal` | |
| 修复工作 | 工作量 **30**（3 回合）；恢复 **10%**；扣 **20** 金属 | `work_type_id=repair_district` |
| 航行放弃城区 | 完整度 **−40%**（取 30%～50% 中值） | 补回约 **80** 金属 + 4 次修复工作 |

**航行例（4 城区满编连接，航行 12 格）**：12÷3=4 步 × 4 城区 × 4 金属 = **64** 金属当量待修。  
**矿区**单次 **25** 金属 / 工作量 **30**（~58/周/座）→ **2** 座专精约可覆盖上述一趟中距航行的修复囤积。

**首版不定**：整城移动额外即时耗能 / 耗金属（与负载分轨的「油耗」）——**关闭**；仅走负载损伤 → 停泊后金属修复。

细则见 [城区总览 · 负载成本](../03-图层与地点/建筑层/城区总览.md#负载成本)、[分离与拆解 · 修复](../03-图层与地点/建筑层/分离与拆解.md#修复城区)、[运作与居民 · 日常开销](../03-图层与地点/建筑层/运作与居民.md#特殊城区--工作区启停)。
