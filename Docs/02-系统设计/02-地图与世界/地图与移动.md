## 目标

定义世界地图的基本结构、移动城市的移动规则，以及地形如何迫使玩家改造或拆解移动城市。

## 范围

- **包含**：网格结构、城市占据方式、移动与停止状态、地形限制。
- **不包含**：具体移动消耗公式、路径寻优算法、渲染实现、格子多层内容与响应（见 [地图图层](./地图图层.md)）。

## 详细说明

### 术语

- **[移动城市](../03-图层与地点/建筑层/README.md)**即**玩家主城**；只有玩家的主城是移动城市，可占据多个格子并在地图上整体移动。
- 荒野上的固定外部据点由建筑层**城区**或外部城市占格构成，见 [势力系统](./势力系统.md)、[城市模块化](../03-图层与地点/建筑层/README.md)。
- 移动城市占据多个格子时，停泊状态下每占格对应建筑层上的一个**城区**实例；与核心区的连接拓扑记录在城廓结构中，**不改变**「城区即建筑层内容」的定义（见 [城市模块化 · 城区：定义与资产定位](../03-图层与地点/建筑层/城区总览.md#城区定义与资产定位)）。
- 特殊地形（如裂谷、狭窄山口、废墟迷宫）可能超出当前移动城市的通过能力，玩家必须改造或拆解移动城市——例如收窄城区宽度、拆卸体积过大的模块、牺牲部分城区以换取整体通过——之后才能继续前进。这类地形既是核心策略挑战，也是一种带来明显情感代价的牺牲。

### 程序设计

#### 地图结构

> **世界地图为一个六边形格组成的纵向卷轴。开发者可以在配置文件修改卷轴的横向宽度，而纵向方向会随着玩家视野的开拓而生成。**

| 属性 | 说明 | 配置方式 |
|------|------|----------|
| **格子形状** | 正六边形格子（Hex Grid） | 固定 |
| **横向宽度** | 地图横向格子数 | 配置文件可修改 |
| **纵向长度** | 无限/伪无限卷轴，随视野开拓动态生成 | 程序动态生成 |
| **生成时机** | 玩家视野开拓到新区域时 | 事件触发 |

- 采用六边形格子而非正方形格子，提供更自然的多方向移动（6 个方向 vs 4 个方向）。
- 纵向卷轴设计符合游戏「追逐太阳」的核心机制——第一、二章**向上**追日（与太阳**同向**），第三至五章**向下**（命名地区的纵向排列见 [地点与场景 · 地图地理关系](../../04-设定/03-地点与场景/README.md#地图地理关系)）。太阳沿**固定方向**移动，玩家追日亦沿该方向；动态压力来自速度与距离，而非轨迹难测（见 [太阳的真相](../../04-设定/05-隐秘真相/太阳的真相.md)）。

#### 格子数据

每个格子可包含：
- **地形层**：六种地形类型之一（平原、丘陵、山地、河流、沼泽、裂隙），见 [地图图层 · 地形类型清单](../03-图层与地点/地图图层.md#地形类型清单)。
- **资源点**：矿藏、果地、遗迹、村镇等（见 [单位类型与视野 - 资源点揭示等级](./单位类型与视野.md#资源点揭示等级)）。
- **城区**：建筑层内容；含**状态正常**与**状态废墟**（见 [城市模块化 · 状态：正常与废墟](../03-图层与地点/建筑层/城区总览.md#状态正常与废墟)）。
- **单位**：侦察队、勘探队、运输队等玩家单位，或 [铁壳](../../04-设定/02-角色与势力/铁壳.md) 等敌方单位。
- **外部据点**：村镇、城市等固定据点。

#### 视野与地图生成

- 格子**默认状态**：未生成/未探索（可能表现为迷雾）。
- 当任何单位（包括飞信）的视野覆盖到某区域时：
  1. 若格子未生成，触发动态生成。
  2. 生成地形、资源点、随机事件等内容。
  3. 格子进入已探索状态，记录在存档中。

#### 坐标系统

建议采用**轴向坐标（Axial Coordinates）**或**偏移坐标（Offset Coordinates）**管理六边形格子：
- 轴向坐标：使用 `q, r` 两个轴，第三个轴 `s = -q - r` 隐含计算。
- 适合六边形的六方向移动计算。

### 移动规则

- 移动城市占据多个连续格子，形成特定形状。
- 移动时，城市整体沿六边形格子方向移动一格或多格。
- **航行**移动时，各连接城区按类别 SO 累积 **[负载成本](../03-图层与地点/建筑层/城区总览.md#负载成本)**（每 **x** 格 → 相当于 **y** 资源修复的结构损伤）；**停泊**不触发此项。
- 整城移动的**速度**与**即时资源消耗**（若有）见 OPEN-007（**与负载成本分轨**；**不**预设为能源的主要用途；能源见 [四种核心资源](../04-资源与人口/四种核心资源.md)）。
- 遇到障碍物或狭窄地形时，可能需要调整城市形状或拆分城区。

#### 停泊与航行

移动城市在任意时刻处于 **停泊** 或 **航行** 之一。二者不仅是「能否移动」，更决定城市**是否座落世界地图**以及**与外界交互的物理边界**（见 [回合与行动表 · 移动城市与行动表](../07-玩法循环/回合与行动表.md#移动城市与行动表)）。

| 状态 | 世界地图 | 与外界交互 | 可指挥 | 移动能力 |
|------|----------|------------|--------|----------|
| **停泊** | **座落**在世界地图上，占据格子；与不可移动的普通外部城市类似，处于同一套地图图层与占格规则中 | **物理接触**交互：勘探、建设、道路、贸易、人口交换等须停泊或队伍抵近 | **是** | **禁用**（不可执行整城移动类指令） |
| **航行** | **不**占世界地图格；在**行进态**中作为**多格可移动单位**沿卷轴移动（ footprint 保留，不占底图格） | **仅远程**交互：通讯、飞信、约定补给等；**不能**形成道路、占格建设等需物理接触的交互 | **是** | **启用** |

#### 状态切换（核心区工作）

**停泊 ↔ 航行**的切换不是瞬时完成，而是 **核心区**上的一项 [工作](../07-玩法循环/工作.md)（见 [连接与多核心 · 停泊与航行切换](../03-图层与地点/建筑层/连接与多核心.md#停泊与航行切换核心区工作)）：

- 玩家在指挥阶段下达切换指令 → **核心区**进入 [工作中](../07-玩法循环/回合与行动表.md#工作中状态)。
- **被工作对象** = **核心区**城区实例；**工作对象**默认同为该核心区（由骄阳之心模块执行离港 / 入港）。
- **工作进度**每回合推进；达到该工作类型配置的**工作时长**后，整城才进入目标状态并启用/禁用相应能力。
- 切换工作进行中，移动城市仍占行动表位（若有该指令）；对照时长各 **2 回合**，实际时长受 [工作效率](../07-玩法循环/回合与行动表.md#工作效率与时长) 影响（默认不参与人数比修正）。
- 航行时的移动速度与消耗见 OPEN-007。

### 黄昏带与暗渊带（玩法）

玩法层使用 **黄昏带**、**暗渊带** 表示太阳照明前沿在地图上的位置；与设定层民众口中的 **[暗渊](../../04-设定/03-地点与场景/暗渊.md)**（一切不被太阳照射的区域）**区分**：

| 层面 | 含义 |
|------|------|
| **设定 · 暗渊** | 叙事与地理：无日照、危险、民众用语 |
| **玩法 · 黄昏带** | 太阳余晖前沿；格级环境压力过渡带 |
| **玩法 · 暗渊带** | 已脱离日照的格级环境带；随太阳移动在地图上**持续向上**推移 |

- 太阳在天上移动时，光照区域随之变化；在卷轴地图上体现为 **黄昏带、暗渊带** 沿卷轴**向上**推移（程序为**格修正工具**，见下节；带级效果与数值见 OPEN-006）。
- 第一、二章玩家**向上**追日，与光带推移同向；**第三章前期**仍可能经历「最后余晖」与光带相关的过渡；**第三章前期结束之后**，叙事与玩法上**全局无日照**（见 [章节生命周期与太阳移动停用](#章节生命周期与太阳移动停用)）。
- 环境行动阶段结算顺序见 [回合与行动表 · 环境结算顺序](../07-玩法循环/回合与行动表.md#环境结算顺序)；格级效果见 [胜利条件 · 动态难度](../01-核心体验/胜利条件.md#动态难度)。

#### 章节生命周期与太阳移动停用

太阳移动与光带推移**并非整局始终运行**。按 [章节划分与故事大纲](../../04-设定/05-隐秘真相/章节划分与故事大纲.md)：

| 阶段 | 日照与光带 | 太阳移动 / 格修正 |
|------|------------|-------------------|
| **第一、二章** | 卷轴上存在 **黄昏带 → 暗渊带** 梯度；追日途中有日照 | **启用**；**v(t)=v₀+a·t** 驱动 xy 推移 |
| **第三章前期** | 最后余晖消退；由有日照过渡到全局无日照 | **启用** → 在叙事节点**停用**（过渡细节 **待定**） |
| **第三章前期以后**（含第三至五章主体） | **全局均属玩法 · 暗渊带**；无 **黄昏带** 梯度、无格级日照差异 | **停用**；不再执行运动学与格修正，以**节省运行时与结算开销** |

**第三章前期以后**的玩法含义：

- 地图上**每一格**在环境结算时按 **暗渊带** 规则处理（与「落在暗渊带内过久」的 debuff 口径统一为**全局常驻**，具体清单仍 OPEN-006）。
- **不再**计算光带边界位置、**不再**每回合推移黄昏带/暗渊带前沿；Sun 模块可整体短路。
- 动态压力来源切换为：暗渊环境下的资源、叙事目标等（见各章系统设计），**不再**来自「被光带追赶」或「追日速度差」。

程序侧用 **`sun_motion_enabled`**（或等价章节标志）控制：为 `false` 时，[环境行动 · 太阳移动](../07-玩法循环/回合与行动表.md#环境结算顺序) 步骤**跳过**，格级光照层视为**全场暗渊带**。

#### 程序口径（OPEN-006 部分已定）

太阳移动与 **黄昏带 / 暗渊带** 在程序上实现为**格修正工具**（卷轴格线修正的执行层）：在 [环境行动](../07-玩法循环/回合与行动表.md#环境结算顺序) 阶段优先执行，将光带边界在卷轴纵向上**向上**推移 **`step_cells`（y）** 格；步进周期由 **`interval_turns`（x）** 与当前调速状态共同决定。

**运动学模型（已定）**

太阳（光带）在卷轴上的推进速率采用**恒定加速度**模型，且**有初速度**：

| 符号 | 含义 | 说明 |
|------|------|------|
| **v₀** | 初速度 | 太阳开始移动后、首次环境结算时的速率（格/回合，或等价口径） |
| **a** | 加速度 | 恒定、**a ≥ 0**；章节 / 剧情可改 **v₀、a**，不改算法结构 |
| **t** | 回合计数 | 自太阳开始移动起，累计的环境结算次数（或等价全局计数） |
| **v(t)** | 当前速率 | **v(t) = v₀ + a·t** |

- 叙事上的「太阳逐渐加速」「第二章速度差拉大」由 **a > 0** 体现；玩家追日压力来自 **v(t)** 随时间增大，而非轨迹难测。
- **v₀、a 的具体数值**、章节切换时是否重置 **t** 或改 **v₀/a**，仍 **待定**（OPEN-006）。

**与 xy 调速工具的衔接（已定）**

格修正工具是**执行层**；运动学模型是**调速层**，在每轮环境结算前计算本周期应推移多少格，再写入工具的 **x / y**：

```
每轮环境结算（Sun 模块）：
  1. 读取当前 t，计算 v(t) = v₀ + a·t
  2. 格残差 accum += v(t)          // 支持 v < 1 时隔回合才推一格
  3. step_cells = floor(accum)     // 本周期向上推移 y 格
  4. accum -= step_cells
  5. 调用格修正工具：interval_turns = x（通常 x = 1，即每环境回合结算一次）
  6. t += 1
```

| 概念 | 说明 |
|------|------|
| **格修正工具** | 执行「向上推移 **y** 格」；**x** 为两次推移之间的最小间隔（回合） |
| **xy 调速** | **y**、**x** 由当前 **v(t)** 与格残差 **accum** 导出，**非**整局固定常数 |
| **初速** | **v₀** 决定开局光带推移快慢；**a** 决定后期加速斜率 |
| **配置来源** | ScriptableObject 或关卡/章节配置：**v₀、a**、可选 **x** 上限/批量策略 |

当 **a = 0** 时退化为匀速（**v(t) = v₀**）；仍走同一套 xy 工具与残差逻辑。

| 概念 | 说明 |
|------|------|
| **推移方向** | 卷轴地图**纵向向上**（与玩法层「光带向上」一致；程序字段与格坐标映射见地图表，待补） |
| **待定逻辑** | 黄昏带与暗渊带是否同步步进、各带**宽度**、格级 debuff 与 [动态难度](../01-核心体验/胜利条件.md#动态难度) 的具体表现（OPEN-006） |

程序侧占位字段（命名 **待定**，仅表达结构）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `initial_velocity_v0` | float | 初速度 **v₀**（格/回合） |
| `acceleration_a` | float | 恒定加速度 **a**（格/回合²） |
| `sun_motion_turn_index` | int | 自太阳开始移动起的环境结算计数 **t** |
| `cell_accumulator` | float | 格残差 **accum**（小于 1 的部分留到下一回合） |
| `interval_turns` | int | 步进周期 **x**（运行时导出或配置；常为 1） |
| `step_cells` | int | 单次推移格数 **y**（由 **accum** 与 **v(t)** 导出） |
| `sun_motion_enabled` | bool | 是否执行太阳运动与格修正；**第三章前期以后**为 `false`（见 [章节生命周期与太阳移动停用](#章节生命周期与太阳移动停用)） |
| `scroll_axis` | enum | 卷轴推移轴（纵向为主，与地图表一致） |

实现归属 [Sun 模块](../../03-程序设计/01-架构总览/模块划分.md)；与环境阶段结算顺序见 [回合与行动数据结构 · 环境行动结算顺序](../../03-程序设计/03-数据字典/回合与行动数据结构.md#环境行动结算顺序)。
