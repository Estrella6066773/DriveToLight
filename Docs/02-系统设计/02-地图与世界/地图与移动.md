## 目标

定义世界地图的基本结构、移动城市的移动规则，以及地形如何迫使玩家改造或拆解移动城市。

## 范围

- **包含**：网格结构、城市占据方式、移动与停止状态、地形限制。
- **不包含**：具体移动消耗公式、路径寻优算法、渲染实现、格子多层内容与响应（见 [地图图层](./地图图层.md)）。

## 详细说明

### 术语

- **[移动城市](../03-图层与地点/建筑层/README.md)**即**玩家主城**；只有玩家的主城是移动城市，可占据多个格子并在地图上整体移动。
- 荒野上的固定外部据点由建筑层**城区**或外部城市占格构成，见 [势力系统](./势力系统.md)、[城市模块化](../03-图层与地点/建筑层/README.md)。
- 移动城市占据多个格子时，停泊状态下每占格对应建筑层上的一个**城区**实例；与核心区的连接拓扑记录在城廓结构中，**不改变**「城区即建筑层内容」的定义（见 [城市模块化 · 城区：定义与资产定位](../03-图层与地点/建筑层/城区总览.md#城区定义与资产定位)）。
- 特殊地形（如裂谷、狭窄山口、废墟迷宫）可能超出当前移动城市的通过能力，玩家必须改造或拆解移动城市——例如收窄城区宽度、拆卸体积过大的模块、牺牲部分城区以换取整体通过——之后才能继续前进。这类地形既是核心策略挑战，也是一种带来明显情感代价的牺牲。

### 程序设计

#### 地图结构

> **世界地图为一个六边形格组成的纵向卷轴。开发者可以在配置文件修改卷轴的横向宽度，而纵向方向会随着玩家视野的开拓而生成。**

| 属性 | 说明 | 配置方式 |
|------|------|----------|
| **格子形状** | 正六边形格子（Hex Grid） | 固定 |
| **横向宽度** | 地图横向格子数 | 配置文件可修改 |
| **纵向长度** | 无限/伪无限卷轴，随视野开拓动态生成 | 程序动态生成 |
| **生成时机** | 玩家视野开拓到新区域时 | 事件触发 |

- 采用六边形格子而非正方形格子，提供更自然的多方向移动（6 个方向 vs 4 个方向）。
- 纵向卷轴设计符合游戏「追逐太阳」的核心机制——第一、二章**向上**追日（与太阳**同向**），第三至五章**向下**（命名地区的纵向排列见 [地点与场景 · 地图地理关系](../../04-设定/03-地点与场景/README.md#地图地理关系)）。太阳沿**固定方向**移动，玩家追日亦沿该方向；动态压力来自速度与距离，而非轨迹难测（见 [太阳的真相](../../04-设定/05-隐秘真相/太阳的真相.md)）。

#### 格子数据

每个格子可包含：
- **地形类型**：荒地、暗渊、遗迹等。
- **资源点**：矿藏、果地、遗迹、村镇等（见 [单位类型与视野 - 资源点揭示等级](./单位类型与视野.md#资源点揭示等级)）。
- **城区**：建筑层内容；含**状态正常**与**状态废墟**（见 [城市模块化 · 状态：正常与废墟](../03-图层与地点/建筑层/城区总览.md#状态正常与废墟)）。
- **单位**：侦察队、勘探队、运输队等玩家单位，或 [铁壳](../../04-设定/02-角色与势力/铁壳.md) 等敌方单位。
- **外部据点**：村镇、城市等固定据点。

#### 视野与地图生成

- 格子**默认状态**：未生成/未探索（可能表现为迷雾）。
- 当任何单位（包括飞信）的视野覆盖到某区域时：
  1. 若格子未生成，触发动态生成。
  2. 生成地形、资源点、随机事件等内容。
  3. 格子进入已探索状态，记录在存档中。

#### 坐标系统

建议采用**轴向坐标（Axial Coordinates）**或**偏移坐标（Offset Coordinates）**管理六边形格子：
- 轴向坐标：使用 `q, r` 两个轴，第三个轴 `s = -q - r` 隐含计算。
- 适合六边形的六方向移动计算。

### 移动规则

- 移动城市占据多个连续格子，形成特定形状。
- 移动时，城市整体沿六边形格子方向移动一格或多格。
- 移动速度与消耗方式见 OPEN-007（**不**预设为能源的主要用途；能源见 [四种核心资源](../04-资源与人口/四种核心资源.md)）。
- 遇到障碍物或狭窄地形时，可能需要调整城市形状或拆分城区。

#### 停泊与航行

移动城市在任意时刻处于 **停泊** 或 **航行** 之一。二者不仅是「能否移动」，更决定城市**是否座落世界地图**以及**与外界交互的物理边界**（见 [回合与行动表 · 移动城市与行动表](../07-玩法循环/回合与行动表.md#移动城市与行动表)）。

| 状态 | 世界地图 | 与外界交互 | 可指挥 | 移动能力 |
|------|----------|------------|--------|----------|
| **停泊** | **座落**在世界地图上，占据格子；与不可移动的普通外部城市类似，处于同一套地图图层与占格规则中 | **物理接触**交互：勘探、建设、道路、贸易、人口交换等须停泊或队伍抵近 | **是** | **禁用**（不可执行整城移动类指令） |
| **航行** | **不**占世界地图格；逻辑上视为在独立的小地图 / 行进态中沿卷轴移动 | **仅远程**交互：通讯、飞信、约定补给等；**不能**形成道路、占格建设等需物理接触的交互 | **是** | **启用** |

- 航行时单位无法自由进出「城内」与「城外」的物理边界（与停泊时队伍出入规则相对）；具体容纳与派遣规则见 [队伍系统](./队伍系统.md)（细则 **待定**）。
- [交战系统](./交战系统.md) 中，停泊占格的移动城市可作为城市目标；航行中的交互范围 **待定**。

#### 状态切换（工作）

**停泊 ↔ 航行**的切换不是瞬时完成，而是一项**工作**：

- 玩家在指挥阶段下达切换指令 → 进入 [工作中](../07-玩法循环/回合与行动表.md#工作中状态) 状态。
- **工作进度**每回合推进；达到该工作类型配置的**工作时长**（回合数，SO 可配）后，才进入目标状态并启用/禁用相应能力。
- 切换工作进行中，移动城市仍占行动表位（若有该指令）；对照时长各 **2 回合**，实际时长受 [工作效率](../07-玩法循环/回合与行动表.md#工作效率与时长) 影响（移动城市切换默认不参与人数比修正）。
- 航行时的移动速度与消耗见 OPEN-007。

### 黄昏带与暗渊带（玩法）

玩法层使用 **黄昏带**、**暗渊带** 表示太阳照明前沿在地图上的位置；与设定层民众口中的 **[暗渊](../../04-设定/03-地点与场景/暗渊.md)**（一切不被太阳照射的区域）**区分**：

| 层面 | 含义 |
|------|------|
| **设定 · 暗渊** | 叙事与地理：无日照、危险、民众用语 |
| **玩法 · 黄昏带** | 太阳余晖前沿；格级环境压力过渡带 |
| **玩法 · 暗渊带** | 已脱离日照的格级环境带；随太阳移动在地图上**持续向上**推移 |

- 太阳在天上移动时，光照区域随之变化；在卷轴地图上体现为 **黄昏带、暗渊带** 沿卷轴**向上**推移（程序为**格修正工具**，见下节；带级效果与数值见 OPEN-006）。
- 第一、二章玩家**向上**追日，与光带推移同向；第三章起玩家**向下**，与光带推移**反向**，形成「光带在身后追赶」的压力——与叙事一致，**不矛盾**。
- 环境行动阶段结算顺序见 [回合与行动表 · 环境结算顺序](../07-玩法循环/回合与行动表.md#环境结算顺序)；格级效果见 [胜利条件 · 动态难度](../01-核心体验/胜利条件.md#动态难度)。

#### 程序口径（OPEN-006 部分已定）

太阳移动与 **黄昏带 / 暗渊带** 在程序上实现为**格修正工具**（可配置、与叙事「太阳移动」对齐的卷轴格线修正）：在 [环境行动](../07-玩法循环/回合与行动表.md#环境结算顺序) 阶段优先执行，**每 x 回合**将光带边界在卷轴纵向上**向上**推移 **y** 格；**x、y 的具体数值及带级逻辑仍待定**（OPEN-006）。

| 概念 | 说明 |
|------|------|
| **格修正工具** | 周期性修正卷轴上光带边界位置；触发条件为每 **`interval_turns`（x）** 回合一次，单次推移 **`step_cells`（y）** 格 |
| **配置来源** | ScriptableObject 或关卡/章节配置；**x、y 待定** |
| **推移方向** | 卷轴地图**纵向向上**（与玩法层「光带向上」一致；程序字段与格坐标映射见地图表，待补） |
| **待定逻辑** | 黄昏带与暗渊带是否同步步进、各带**宽度**、章节间是否改 x/y、格级 debuff 与 [动态难度](../01-核心体验/胜利条件.md#动态难度) 的具体表现（OPEN-006） |

程序侧占位字段（命名 **待定**，仅表达结构）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `interval_turns` | int | 步进周期 **x**（每 x 回合触发一次） |
| `step_cells` | int | 单次推移格数 **y** |
| `scroll_axis` | enum | 卷轴推移轴（纵向为主，与地图表一致） |

实现归属 [Sun 模块](../../03-程序设计/01-架构总览/模块划分.md) ；与环境阶段结算顺序见 [回合与行动数据结构 · 环境行动结算顺序](../../03-程序设计/03-数据字典/回合与行动数据结构.md#环境行动结算顺序)。
