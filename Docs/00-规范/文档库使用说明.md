# 文档库使用说明

## 1. 文档三分库

| 库 | 路径 | 回答的问题 |
|----|------|------------|
| 系统设计 | `Docs/02-系统设计/` | 玩家体验到什么、规则**理应**怎样 |
| 程序设计 | `Docs/03-程序设计/` | 用什么表、模块、数据**具体**实现 |
| 设定 | `Docs/04-设定/` | 世界与故事是什么 |

## 2. 文档状态

每篇文档顶部元数据里应标注 `状态`：

- **草稿**：想法未收敛，可大幅修改。
- **评审中**：基本写完，等待团队确认。
- **已锁定**：已通过评审，后续修改需走变更记录。

## 3. 命名规范

- 优先使用 `前缀-标题.md`，例如：
  - `核心循环.md`
  - `能力系统-设计.md`
  - `能力系统-实现.md`
  - `数据表-PlayerState.md`
- 避免使用 `新建文本文档.md`、`未命名.md` 等无意义名称。

## 4. 成对维护

同一主题的设计与实现，用**双向链接**互相引用。

示例：

```markdown
<!-- 在 02-系统设计/能力系统-设计.md 顶部 -->
> 实现文档：[03-程序设计/能力系统-实现.md](../03-程序设计/能力系统-实现.md)

<!-- 在 03-程序设计/能力系统-实现.md 顶部 -->
> 设计文档：[02-系统设计/能力系统-设计.md](../02-系统设计/能力系统-设计.md)
```

## 5. 白板流程

1. 灵感、参考图、情绪板先扔进 `01-白板/`。
2. 经过讨论收敛后，迁移到 `02-系统设计/` 或 `04-设定/`。
3. 进入开发阶段后，在 `03-程序设计/` 写实现文档。
4. 废弃草稿移动到 `99-备份与归档/`。
