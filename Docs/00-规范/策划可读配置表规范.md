## 目标

为系统策划提供**可直接阅读与填写**的配置表格体系：

- **列名**使用中文词组，对应玩法语义，而非程序字段缩写。
- **单元格取值**优先使用 [enums.csv](../03-程序设计/03-数据字典/tables/shared/enums.csv) 中的**显示名称**（如「全部允许」），而非英文枚举键（如 `all`）。
- **能力**用自然语言列描述「何时可用、做什么、消耗什么」；程序 ID 与 GE/GA 映射下沉到 [_program/](../03-程序设计/03-数据字典/tables/_program/)。
- 与 [SO配置与能力通道规范](./SO配置与能力通道规范.md) 并存：SO 仍是运行时权威载体；CSV 是策划维护入口，经导入管线生成或校验 SO。

## 适用范围

| 范围 | 说明 |
|------|------|
| **包含** | 地图图层 `L*_defs.csv`、队伍与能力说明表、后续数值框架中的策划主表 |
| **不包含** | `_program/` 导入映射表（程序只读）；运行时实例状态 |

## 双层结构

```mermaid
flowchart LR
  A["策划主表\n中文列名 + 中文枚举 + 能力自然语言"] --> B["导入 / 校验"]
  B --> C["_program/\n程序键、Tag、GE/GA"]
  C --> D["SO / 运行时"]
```

| 层级 | 谁维护 | 内容 |
|------|--------|------|
| **策划主表** | 系统策划 | 中文列名、中文枚举、自然语言能力说明、稳定英文 **类型编号** |
| **_program** | 程序 / 工具同步 | `column_registry` 列映射、枚举反查、`ge_catalog` 触发条件 |
| **数据字典** | 程序设计 | 列名中英对照、类型约束、导入规则 |

## 列名规则

1. **用完整中文词组**作列名，禁止单字缩写拼表头。
2. **稳定主键**保留英文 `id`（如 `plain`、`scout`），列名写作 **类型编号** 或 **能力编号**，单元格仍填英文 id，便于跨表引用与版本对比。
3. **布尔列**写作「可否……」「是否……」，单元格填 **是** / **否**（登记于 `enums.csv` 组 `yes_no`）。
4. **枚举列**单元格填 `display_name_zh`（如「可通过」「已定」），导入时经 `enums.csv` 反查 `enum_key`。
5. **列表列**用分号分隔；可填中文显示名或类型编号，导入统一解析为程序 id。

列名与程序键对照见 [column_registry.csv](../03-程序设计/03-数据字典/tables/shared/column_registry.csv)。

## 能力说明规则

能力不在策划主表中仅用 `GA_*` 编号表达；须满足至少其一：

| 方式 | 适用 |
|------|------|
| **专表** [能力说明.csv](../03-程序设计/03-数据字典/tables/shared/能力说明.csv) | 跨模块复用的 GA；队伍默认能力 |
| **行内列「改造能力说明」等** | 仅该行类型独有的简短能力（如丘陵平整） |

专表必备列：

| 列名 | 说明 |
|------|------|
| **能力编号** | 稳定英文 id，对应 `gameplay_ability_config.ability_id` |
| **能力名称** | 策划/UI 用短名 |
| **适用对象** | 地形 / 队伍 / 城区 / 设施 等 |
| **何时可用** | 自然语言前置条件 |
| **效果说明** | 自然语言效果 |
| **消耗说明** | 行动点、资源、工作量对照等 |
| **关联类型编号** | 可空；链到地形 `类型编号`、队伍模板 id 等 |
| **状态** | 已定 / 待定 |

程序侧 `executor_key`、`required_tags` 等写入 [_program/ge_catalog.csv](../03-程序设计/03-数据字典/tables/_program/ge_catalog.csv)，策划**不必阅读**。

## 策划工作流

1. 改对应层 **`L{n}_*_defs.csv`**（中文列名）。
2. 新增或调整能力时，同步 [能力说明.csv](../03-程序设计/03-数据字典/tables/shared/能力说明.csv)。
3. 枚举取值查 [enums.csv](../03-程序设计/03-数据字典/tables/shared/enums.csv) 的 **显示名称** 列。
4. 跨层交互改 [L0_cross_modifiers.csv](../03-程序设计/03-数据字典/tables/L0_cross_modifiers.csv)（后续按本规范逐步中文化）。
5. **待定** 行不进入正式构建；**不要**在 `_program/` 手填玩法数值。

字段详解与程序映射见 [03-程序设计/03-数据字典](../03-程序设计/03-数据字典/README.md)。

## 系统设计文档中的写法

在 [02-系统设计](../02-系统设计/) 正文中：

- 写清策划读哪张表、能力自然语言口径、与玩法的对应关系。
- **不写**完整字段表、`schema_version`、程序键与导入细节。
- 用链接指向本规范与程序设计数据字典。

示例：

> **配置入口**：地形参数见 [L1 地形策划主表](../03-程序设计/03-数据字典/tables/L1_terrain_defs.csv)；列名与取值均为中文。能力效果见 [能力说明表](../03-程序设计/03-数据字典/tables/shared/能力说明.csv)。
