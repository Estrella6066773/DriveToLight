## 待细化追踪

开放项按 **系统（sy）**、**程序（sf）**、**设定（st）**、**策划表（tb）** 分轨维护；每条仅登记在一个主轨道。

| 轨道 | 前缀 | 文件 | 收录范围 |
|------|------|------|----------|
| **系统** | `sy-NN` | [待细化追踪-系统.md](./待细化追踪-系统.md) | 玩法规则、流程规格、数值挂钩、界面交互逻辑 |
| **程序** | `sf-NN` | [待细化追踪-程序.md](./待细化追踪-程序.md) | SO/CSV、服务、测试、运行时结构、引擎映射 |
| **设定** | `st-NN` | [待细化追踪-设定.md](./待细化追踪-设定.md) | 叙事、世界观、角色势力、剧情文稿 |
| **策划表** | `tb-NN` | [待细化追踪-策划表.md](./待细化追踪-策划表.md) | CSV/配置表中待填的具体行与列值 |

**维护**：新增 → 判断主归属轨并编号 → 闭环时移入该轨「已解决记录」/「已关闭项」并在该轨修订记录留痕（系统轨见 [待细化追踪-系统 · 如何维护](./待细化追踪-系统.md#如何维护)：开放项**只写未解决**）。历史草稿中的 `OPEN-xxx` 对照见 git 历史中已废止的 `待细化追踪.md`（2026-07-10 起改用 sy/sf/st；2026-07-16 起增 tb 轨）。

**Agent 约束**：遇矛盾、缺口、待定规则或文档与实现不一致时，**不得当场编造结论**；须登记到上表对应轨（或 [设计缺口清单](../03-程序设计/设计缺口清单.md) / 单文「待确认事项」）后停手，见 [文档库使用说明 · 开放问题处理](./文档库使用说明.md#10-开放问题处理agent-与维护者) 与项目规则 `open-issue-backlog-first.mdc`。
