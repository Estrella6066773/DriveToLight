# 待细化追踪 · 策划表

记录**策划可读配置表**中待填、待补、待裁定的具体条目与列值。本轨专注「表里缺什么值」，不与系统轨（规则缺口）或程序轨（实现落地）叠床架屋。

← [00-规范](./README.md) · [系统轨](./待细化追踪-系统.md) · [程序轨](./待细化追踪-程序.md) · [设定轨](./待细化追踪-设定.md)

## 如何维护

| 事项 | 约定 |
|------|------|
| **何时新增** | 表结构已有（CSV 列名已定、`schema_version` 已写入），但某一行或某一列的值尚未裁定或仅占位时 |
| **ID 命名** | 本轨 **`tb-NN`**（两位序号，从 01 递增）；可附注对应表文件名与行关键标识 |
| **状态取值** | **部分符合**（有草案或示例值、正式值待定）· **待讨论**（列值或表行尚未动笔） |
| **闭环** | 值填入对应 CSV / SO / `tables/` 目录 → 移入「已关闭项」→ 修订记录留痕 |
| **交叉项** | 若待填值阻塞了系统规则（sy）或程序实现（sf），在本轨登记，并在系统/程序轨「未解决」列链回本轨 tb 号 |
