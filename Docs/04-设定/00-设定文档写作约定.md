# 设定文档写作约定

← [设定目录](./README.md)

本约定用于减少设定库内的重复与层级混乱。**新增或修订设定词条时须遵守。**

## 词条结构（精简）

**地点 / 势力 / 角色**（默认）：

1. **摘要**（一两句）
2. **玩家可见**（民众怎么说、玩家看到什么；神话与传说语境可用 **世人**）
3. **玩法关联**（与哪些系统/章节有关，用链接）
4. **关键关系**（小表，链到相关词条）
5. **待确认**（可选，仅列未决项）
6. **修订记录**

**禁止**在地点词条中增设：`设定母本（内部）`、`揭示层级` 整表、`章节关联` 整表（母本与章节已在专属文档中）。

**05-隐秘真相** 词条：摘要 + 母本正文 + 关键关系 + 待确认；**不写**与 [章节划分与故事大纲](./05-隐秘真相/章节划分与故事大纲.md) 重复的章节表（可一句链过去）。

**含参考图的词条**：须设 **视觉参考** 节；图片用 Markdown 嵌入（`![说明文字](../image/文件名.jpg)`），使文档内可直接预览，**不得**仅用文件链接代替预览。说明表可保留来源与可取方向；完整文件索引见 [image/README.md](./image/README.md)。

## 跨词条：只链不抄

- 专名定义、物理本质、层级归属等**只写在归属词条**的正文。
- 其它词条在摘要、关键关系、README 索引中：**只写与本词条相关的一句**，或**仅留链接**；禁止复制已在它处写明的句子（例如：不在 [铁壳](./02-角色与势力/铁壳.md) 写「铁巢废墟是巨大构造体」——见 [铁巢废墟](./03-地点与场景/铁巢废墟.md)）。
- README 索引：一句话定位即可，**不复制**正文定义。

## README 只做索引

- [04-设定/README.md](./README.md) 与各子目录 README：**只列目录与链接**，不复制子文档中的表格正文。
- 详细总览写在子目录 README（如 [03-地点与场景/README.md](./03-地点与场景/README.md)）。

交叉链接、返回导航、成对维护等通用规则见 [文档库使用说明](../00-规范/文档库使用说明.md#7-返回导航与目录索引)。

## 背景文档与正面表述

**世界观、概述类词条是背景说明，不是剧情正文**——只写世界是什么样的；章节与任务归 [章节划分与故事大纲](./05-隐秘真相/章节划分与故事大纲.md)，机制归 [02-系统设计/](../02-系统设计/)。文风简洁，不写成史诗散文。

**排除即删除**：用户、定案或信息归属划定「本文档不写 X」时，X 须从正文消失；禁止用否定、对比、括注、元叙述等把 X 换皮留档。自检：读者读完是否仍能还原 X 的实质信息？能则未删干净。细则见 Cursor 规则 `common/docs/docs-exclusion-is-deletion.mdc`；只保留本子文档范围内的**正面定义**，需它处信息时**只链不写**。

## 地图地理（地点）

- **纵向主线**顺序、**章节行进方向与日照**，以 [03-地点与场景/README.md · 地图地理关系](./03-地点与场景/README.md#地图地理关系) 为**单一来源**。
- **「卷轴」仅在该节描述地图结构**时使用；其它设定文档**不写**「卷轴」，用上下左右表述。
- **日照与地区名分离**：地图上的地区名（如日生之地）不表示当时一定有阳光；第三至五章太阳已离开循烬城路线，途经之地均无日照，属 [暗渊](./03-地点与场景/暗渊.md)。
- 命名地点词条在 **关键关系** 中标注 **主线上下方向**（或 **主线外**、**移动占格**）；**不写**完整章节表，章节节奏链至 [章节划分与故事大纲](./05-隐秘真相/章节划分与故事大纲.md)。

## 对人的称呼

- **不用「居民」**——不能涵盖城主、副官、外来者等，且易与城区类型 **居民区**（机制专名）混淆。
- **民众**：日常叙事、玩家可见层、指日生文明中的一般人怎么说、怎么想。
- **世人**：神话、传说、口耳相传、泛指天下人的说法（如暗渊深处的传闻、渊光）。
- **居民区**：城区模块类型，见 [城市模块化](../02-系统设计/03-图层与地点/建筑层/README.md)，**不**改作「民众区」。
