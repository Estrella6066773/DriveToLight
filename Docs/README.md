# 《循光之城》项目文档

本目录存放《循光之城》的**草稿脑暴、系统设计、程序设计、设定**，与 Unity 工程资源（`Assets/`）分离。

> 命名说明：项目代码仓库/本地目录名为 `延续`，游戏名为《循光之城》。本文档库同时服务于两者。

| 目录 | 用途 |
|------|------|
| [00-规范/](00-规范/) | 文档约定、待细化追踪、版本控制分工 |
| [游戏介绍.md](游戏介绍.md) | **对外展示入口**：游戏介绍与专题文档索引 |
| [01-草稿/](01-草稿/) | 脑暴、灵感碎片、情绪板、参考图、快速涂鸦 |
| [02-系统设计/](02-系统设计/) | 玩家体验、玩法规则、流程、数值**设计意图** |
| [03-程序设计/](03-程序设计/) | 运行时架构、逻辑规范、数据字典、引擎映射 |
| [04-设定/](04-设定/) | 世界观、角色、地点、时间线等叙事与背景 |

## 分工

| 库 | 回答的问题 | 典型读者 |
|----|------------|----------|
| **草稿** | 我们想要什么感觉、有哪些灵感碎片 | 策划、美术、全员对齐 |
| **系统设计** | 玩起来是什么、规则**理应**怎样 | 策划、美术、全员对齐 |
| **程序设计** | 用什么表、判定与架构**具体**实现 | 程序、技术策划 |
| **设定** | 故事与背景 | 叙事、美术 |

```
延续/
├── Assets/          ← Unity 资源（建议 UVC/Plastic 管理）
├── Docs/            ← 本目录（建议 Git 管理）
├── ProjectSettings/ ← Unity 配置
└── ...
```

> 规则见仓库根目录 `ignore.conf` 与 `.gitignore`。

## 文档约定

- 所有新增文档尽量从 [_templates/](_templates/) 复制模板开始。
- 文件名使用 `中文标题.md` 或 `前缀-标题.md`，避免空格以外的特殊字符。
- 文首须同时标注 **`状态`**（写作进度）与 **`校验状态`**（与关联文档的对照进度）。取值见 [文档库使用说明 §2](00-规范/文档库使用说明.md#2-文档元数据)。
- 同一主题的**设计意图**与**实现**成对维护：
  - 设计意图写在 `02-系统设计/`
  - 实现写在 `03-程序设计/`
  - 用双向链接互相引用。
- **02-系统设计**、**04-设定** 正文首次提及其它文档或已在库中单独立篇的专有名词时，须链到对应文件（同一文档内同一目标仅链接一次）。详见 [文档库使用说明 §6](00-规范/文档库使用说明.md#6-正文交叉链接对外展示文档)。
- 草稿里的内容一旦收敛，应整理后迁移到系统/设定/程序目录，原草稿可归档到 `99-备份与归档/`。
