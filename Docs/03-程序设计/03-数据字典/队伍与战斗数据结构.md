## 目标

建立“GAS-lite + SO 配置”的数据结构：

- 程序只提供 GA executor，不写死固定队伍逻辑。
- 队伍模板由初始 GE 与 GA 列表组合能力，参数由 GE/GA SO 细化配置。
- 开发者在引擎内手动配置模板与能力（可由 MCP 辅助执行）。
- 支持当前默认模板人数：侦察队 3、运输队 15、工程队 10；武装类民兵 15、步兵 10、弓手 10。

## 总体分层

| 层级 | 数据对象 | 作用 |
|------|----------|------|
| 模板层 | `team_template_config` | 定义队伍模板、默认人数、默认策略、挂载能力列表 |
| 效果层 | `gameplay_effect_config` | 定义 Buff、Debuff、瞬时修正、周期效果 |
| 技能层 | `gameplay_ability_config` | 定义可执行技能、激活条件、executor 与触发效果 |
| 标签 / 属性层 | `gameplay_tag_registry` + `attribute_registry` | 定义能力开关、状态、互斥条件与数值通道 |
| 实例层 | `team_instance_state` | 记录队伍运行时状态与派生值 |
| 行为层 | `team_command_queue` + `team_command_item` | 指令队列与执行结果 |
| 关系层 | `faction_relation_state` | **领袖关系**与敌我判定权威来源 |
| 结算层 | `combat_event_log` | 攻击/反击结算日志 |

## 表 8：`headcount_factor_curve`（人数比影响系数）

> **程序表名**保留英文标识 `headcount_factor_curve`；玩法描述用**人数比系数**，首版全线性，**不**用「曲线」指代本机制。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `curve_id` | string | PK | - | 配置 ID |
| `curve_type` | string | `linear` / `piecewise` | `linear` | 映射方式；首版仅用 `linear` |
| `points_json` | string(json) | 非空 | - | 分段定义（`piecewise` 时用；首版可忽略） |

### 统一人数比系数

- 输入：`headcount_ratio = current_headcount / default_headcount`（**满编** = 模板 `default_headcount`）。
- 输出（首版 `linear`）：**系数 = 人数比**，用于**视野、负载、攻击/战力**（**不含** `base_move_speed`）。
- **工作进度**：`turn_work_delta = participant_count × work_efficiency`；`apply_headcount_factor=true` 时 `participant_count = max(1, current_headcount)`，**不在** `work_efficiency` 内重复乘人数比（见 [回合与行动数据结构 · 工作效率与工作量](../03-数据字典/回合与行动数据结构.md#工作效率与工作量结算)）。

### 执行规则映射

- AI 按 `order_index` 逐步执行任务表；**如何执行**由 AI 自主决定，见 [回合与行动表 · 指令表与自主执行](../../02-系统设计/07-玩法循环/回合与行动表.md#指令表与自主执行)。
- 遭遇意外时 **不** 递增 `current_index`，当前条目保持 `running` / `suspended` 直至完成或 AI 接管规则判定失败/跳过；**不重置** `work_accumulated`（可中止类型）。
- 若 `required_ability_channel` 未启用，则 `fail_reason=ability_not_enabled`。
- 当前条目失败且可重试时，可在同目标内调整后重试；重试耗尽后写战报并跳至下一条。
- `form_team` / `reinforce_team` 受理时写入 `reserved_allocation_json` 并注册 `PopulationReservationState`；工作落实或条目清除时释放。
