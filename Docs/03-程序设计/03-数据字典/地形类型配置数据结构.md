## 表 1：`terrain_type_config`

与 `terrain_profile_config` **列对齐**，供关卡工具筛选；**权威数值以 profile 行为准**。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `terrain_type_id` | string | **PK** | 地形枚举 id；合法取值见 CSV |
| `display_name_zh` | string | 非空 | 显示名 |
| `profile_id` | string | FK | → `terrain_profile_config.profile_id` |
| `unit_path_cost_equiv` | int | ≥1 | 单位路径等效格数（与 profile 同步） |
| `facility_build_mode` | string | 枚举 | 可建造设施基础档位（与 profile 同步） |
| `facility_build_allow_type_ids` | string | 分号分隔 | 允许设施白名单（`facility_type_id` 列表） |
| `unit_enter_allowed` | bool | | 一般单位可否进入并停留 |
| `mobile_city_passability` | string | FK 枚举 | → `mobile_city_passability_enum.passability_id` |
| `config_status` | string | `defined` / `pending` | 该行定案状态 |
| `notes_zh` | string | 可空 | 备注 |

## 表 2：`terrain_profile_config`

每行一地形的**格基线**（无有效地形类设施实例时）。导入管线据此生成并 Apply 对应 GE/GA。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `profile_id` | string | **PK** | profile 唯一 id |
| `terrain_type_id` | string | FK | → `terrain_type_config.terrain_type_id` |
| `unit_path_cost_equiv` | int | ≥1 | 单位路径等效格数 |
| `unit_enter_allowed` | bool | | `false` 时导入 `GE_Terrain_BlockUnitEnter` |
| `facility_build_mode` | string | 枚举 | 可建造设施基础档位；见下节 |
| `facility_build_allow_type_ids` | string | 分号分隔 | 允许设施白名单；优先级 **高于** `facility_build_mode` |
| `mobile_city_passability` | string | FK 枚举 | 移动城市通行性；见 `mobile_city_passability_enum` |
| `mobile_city_path_cost_equiv` | int | 可空 | 移动城市路径等效；**空 = 与 `unit_path_cost_equiv` 相同** |
| `terrain_transform_ability_id` | string | 可空 | 非设施的地形改造 GA id（如丘陵平整）；空 = 无 |
| `granted_tags_json` | string | 分号分隔 | 格 ASC 初始化授予的 Tag |
| `notes_zh` | string | 可空 | 备注 |

**不在此表**：隧道、桥梁建成后的路径 / 通行 / 建造档位覆盖 → [facility_terrain_class_config.csv](./tables/facility_terrain_class_config.csv)。

### 枚举：`facility_build_mode`

| CSV 值 | 含义 | 导入映射 |
|--------|------|----------|
| `all` | 占格类、辅助类、地形类均可建（仍受 `facility_type` SO 的 `rule_guard` 约束） | `GE_Terrain_FacilityBuild_All` |
| `auxiliary_only` | 仅辅助类；阻断生产/仓储与地形类 | `GE_Terrain_FacilityBuild_AuxiliaryOnly` |
| `none` | 阻断一般建造；白名单中的 `facility_type_id` 除外 | `GE_Terrain_FacilityBuild_None` |

### 枚举：`mobile_city_passability`

定义见 [mobile_city_passability_enum.csv](./tables/mobile_city_passability_enum.csv)。`crossable` 由 `GA_TerrainRule_MobileCityCrossRift`（规划 executor：`EvaluateMobileCityCrossableFootprint`）校验城区 footprint。

## 表 2b：`facility_build_allow_catalog`

为指定 `facility_type_id` 追加 `Allow.Build.*` 建造权限；`rule_priority` 默认高于基础档位。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `allow_rule_id` | string | **PK** | 规则 id；profile 白名单列写的是 `facility_type_id`，导入时反查本表 |
| `facility_type_id` | string | FK | → `facility_type_config` |
| `display_name_zh` | string | 非空 | 显示名 |
| `rule_priority` | int | | 与基础档位冲突时的优先级 |
| `gameplay_effect_id` | string | FK | 授予 `Allow.Build.{facility_type_id}` 的 GE |
| `granted_tag` | string | | 运行时 Tag，形如 `Allow.Build.{facility_type_id}` |
| `source_zh` | string | 可空 | 挂载来源说明（地形 profile / 资源点等） |
| `notes_zh` | string | 可空 | 备注 |

资源点侧白名单字段见 L3 [resource_point_profile_config.csv](./tables/resource_point_profile_config.csv)；与地形 profile **叠加**于同一 hex ASC。

## 表 3：`terrain_capability_catalog`

程序实现目录：每条 `capability_id` 对应须实现的 GE/GA SO。策划**不**在此填玩法数值；`derived_from_profile_column` 标明由 profile 哪一列触发。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `capability_id` | string | **PK**；与 SO `effect_id` / `ability_id` 对齐 |
| `capability_kind` | string | `gameplay_effect` / `gameplay_ability` |
| `display_name_zh` | string | 可读名 |
| `executor_key` | string | GA 专用；程序入口 |
| `gameplay_effect_id` | string | GE 自引用或 SO id |
| `attribute_id` | string | GE 修正的 Attribute（如 `terrain_path_cost_equiv`） |
| `modifier_op` | string | `set` / `add` / `multiply` |
| `modifier_value` | number | 修正值（由 profile 列或导入规则填入） |
| `granted_tags_json` | string | 授予 Tag |
| `blocked_tags_json` | string | 阻断 Tag |
| `required_tags_json` | string | GA 激活所需 Tag |
| `rule_guard_zh` | string | 前置条件说明 |
| `impl_status` | string | `defined` / `pending` |
| `derived_from_profile_column` | string | 对应的 profile 列名或条件 |
| `notes_zh` | string | 备注 |

合法 `capability_id` 清单见 CSV，不在此维护副本。

### Tag 语义（地形相关）

| Tag 模式 | 说明 |
|----------|------|
| `Terrain.{Type}` | 地形种类标记 |
| `Capability.Build` | 建造总闸门 |
| `Capability.Build.ProductionStorage` | 生产/仓储占格类 |
| `Capability.Build.Terrain` | 地形类设施 |
| `Capability.Build.Auxiliary` | 辅助类设施 |
| `Allow.Build.{facility_type_id}` | 允许设施覆盖；优先于基础档位 |
| `Capability.Move.Enter` | 允许进入并停留 |
| `Actor.MobileCity` | 移动城市通行判定用 |

## 运行时合成

地图格有效参数按下列顺序合成（具体数值取自各 CSV 行）：

1. `terrain_profile_config` 格基线  
2. 资源点 / 建造白名单（`TryBuild` 时）  
3. 有效地形类设施覆盖（`FacilityTerrainClassService`）  
4. 环境层与其他图层规则（见 [layer_interaction_rule_config.csv](./tables/layer_interaction_rule_config.csv)）

| 步骤 | 行为 |
|------|------|
| 地图生成 / 读档 | 读 profile 列 + 白名单 → hex ASC |
| 建造 | `TryBuild`：先 `Allow.Build.*`，再 `facility_build_mode`，再 `build_rule_guard_json` |
| 移动结算 | 读合成后的 `terrain_path_cost_equiv` 等 |
| 地形改造 | 改造 GA 完成后替换 `terrain_type_id` 并重载 profile |
| 地形类设施 | 建成/击毁时 Apply / Revert 宿主格覆盖 |

| CSV | Unity SO（规划） |
|-----|------------------|
| `terrain_profile_config` | `TerrainProfileConfigSO` |
| `terrain_capability_catalog` · GE | `GameplayEffectConfigSO` |
| `terrain_capability_catalog` · GA | `GameplayAbilityConfigSO` |
