## 目标

定义回合推进、行动表与指令队列的分工数据结构。行动表决定**本回合内**谁先行动；指令队列决定**跨回合**任务规划（见 [队伍与战斗数据结构](./队伍与战斗数据结构.md)）。

| 值 | 说明 |

| `player_command` | 玩家指挥 |

### 插入与重排规则

与 [回合与行动表 · 行动表](../../02-系统设计/07-玩法循环/回合与行动表.md#行动表) 对齐：

**尾部追加**（默认）：

- 新条目 `order_index = max(existing) + 1`（空表时为 0）。

**顺序输入**：

- `sequential_input_only=true`：指挥阶段每次只接受一条行动表写入或重排操作；上一条提交完成后才接受下一条。

**插队后移**（`reorder_pushback=true`）：

- 将 `actor_ref` 从原位置 `from_index` 移到 `to_index`（`to_index < from_index` 表示提前）时：
  1. 对满足 `to_index <= order_index < from_index` 的条目，`order_index += 1`；
  2. 将该 actor 的条目设为 `order_index = to_index`；
  3. 重排后重新紧凑编号或保持稀疏索引（实现二选一，存档须稳定）。

**向后调整**（`to_index > from_index`）：对称处理，中间条目 `order_index -= 1`。

### 生成规则（运行时）

1. 读取 `turn_state.game_seed` 与当前存活外部城市 ID 列表。
2. 使用确定性伪随机算法（如 seeded shuffle）生成 `order_index` 排列；**同种子、同城市集合结果一致**。
3. AI 行动阶段按 `order_index` 升序依次驱动各外部城市行动表。
4. 城市集合变化时，下一回合重新生成；本回合内顺序不变。

### 工作中断

与 [工作中断与恢复](../../02-系统设计/07-玩法循环/回合与行动表.md#工作中断与恢复) 对齐：

| 程序入口 | 说明 |
|----------|------|
| `SuspendWork` | 读 `allow_suspend`；`true` 则 `suspended` 且保留进度；`false` 则进度归零 |
| `ResolveWorkInterrupt` | 读 `on_interrupt_disposition`：`retry` 挂起条目；`ended` 清除工作事件 |
| `ResumeWork` | 恢复；`status=running`；可更新 `work_actor_ref` |
| `AdvanceWorkProgress` | 有效回合推进进度；达阈值时按 `outcome_repeat_policy` 结算成果 |
| `TryCompleteWork` | 进度满且最终成果已结算 → `done` |

- 阶段性成果：`repeatable` 允许重启后再次结算；`once_per_subject` 写入 `granted_outcome_ids` 后不再发放。

### 工作效率与时长结算

与 [工作效率与时长](../../02-系统设计/07-玩法循环/回合与行动表.md#工作效率与时长) 对齐。

```text
work_efficiency = clamp( Σ(additive), efficiency_min, efficiency_max ) × Π(multiplicative)
work_duration_turn = max( 1, ceil( work_base_duration_turn / work_efficiency ) )
```

**人数乘法项**（`apply_headcount_factor=true` 时）：

- `headcount_ratio = current_headcount / default_headcount`（满编 = 模板 `default_headcount`）
- 写入数值通道：`source_id=headcount_ratio`，`modifier_op=multiply`，`modifier_value=headcount_ratio`

**其他模块**：通过 `stat_modifier_provider`（SO 或运行时注册）向 `work_efficiency` 通道追加条目；清单 OPEN-031。

结算流程（**仅在工作开始时**；进行中不重算）：

1. 读取 `work_base_duration_turn`。
2. 收集加法 / 乘法修正项（含**当时**人数比与各模块修正），写入 `stat_value_channel_entry`。
3. 合成 `work_efficiency_final`，计算 `work_duration_turn`。
4. 写入当前 `team_command_item`，设置 `work_duration_locked=true`。
5. 将明细挂到工作上下文，供 UI 读取。

**修正变化生效范围**：工作进行中人数或模块修正变化**不**改变已锁定的 `work_duration_turn`；**下一项**工作开始时按新状态重新执行本流程。

| `initial_velocity_v0` | float | 初速度 **v₀**（格/回合） |
| `acceleration_a` | float | 恒定加速度 **a**（格/回合²） |
| `sun_motion_turn_index` | int | 环境结算计数 **t**（存档） |
| `cell_accumulator` | float | 格残差 **accum**（存档） |
| `interval_turns` | int | 本周期步进间隔 **x**（运行时导出；常为 1） |
| `step_cells` | int | 本周期推移格数 **y**（运行时导出） |
| `sun_motion_enabled` | bool | 是否执行太阳运动；**第三章前期以后** false |
| `scroll_axis` | string / enum | 卷轴推移轴（与地图表一致，待地图表补全） |

### `sun_band_step_config`（兼容别名）

与 `sun_band_motion_config` 同义或为其子视图；旧字段 **`interval_turns` / `step_cells`** 保留为**运行时导出值**，不再表示整局固定常数。

程序从 SO 读取格序遍历规则，不在代码中写死地图方向常量（方向映射见地图表，待补）。
