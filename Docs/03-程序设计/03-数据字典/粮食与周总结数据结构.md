# 粮食与周总结数据结构

权威规则见 [粮食与周总结（草稿）](../../01-草稿/粮食与周总结/README.md)（**无饥饿版**）。

## 周总结编排（Yanxu.Turn）

| 标识 | 说明 |
|------|------|
| `IWeeklySummaryStep` | `StepId`、`Module`、`Order`、`Execute(WeeklySummaryContext)` |
| `WeeklySummaryContext` | `TurnId`、`WeeklySummaryIndex`、运行时服务引用 |
| `WeeklySummaryStepCatalog` | 注册与排序 |
| `WeeklySummaryOrchestrator` | 环境行动后 `Run`；维护 `weekly_summary_index` |

## 粮食分配（Yanxu.Resource）

| 标识 | 说明 |
|------|------|
| `FoodAllocationResult` | 本周期 `FedCount` / `UnfedCount` / `DeathCount` / `FoodConsumed` |
| `AllocationSortKey` | 离散分配与减员排序键（MVP：单归属单桶） |
| `FoodWarehouseDrawdown` | 「优先用于充饥」仓库扣减 |
| `IFoodAllocationRuleOverride` | 领袖/归属 Override 单管线 |
| `FoodAllocator` | 按 `mobileCityId` 运行分配链 |
| `FoodSufficiencyStatus` | `Sufficient` / `Insufficient`（常驻 UI 只读） |

## 人口与载荷

| 标识 | 说明 |
|------|------|
| `ApplyWorkHeadcountChange` | 粮食/战斗减员统一入口：`Apply(cityId, affiliationId, delta, reason)` |
| `TeamCargoState` | 队伍载荷粮食需求 |
| `EscortAccompanyingState` | 随行人员人口与粮食需求 |

**已定口径**（见 [粮食与周总结-已定案详述](../../01-草稿/粮食与周总结/粮食与周总结-已定案详述.md)）：

- 活跃人口每周期消耗 **1** 食物/人（周总结 = 1 粮食/人/周）。
- 队伍载荷设计目标约 **3** 周期粮食（具体数值待定）。
- 长期外派须 **运输队** 输送补给。

## 城市拓扑 SO（Yanxu.City）

| 标识 | 字段 |
|------|------|
| `DistrictRelocateConfigSO` | `minDurationTurns`（3）、`maxDurationTurns`（5） |
| `SailingSeparationConfigSO` | `damageFractionMin`（0.3）、`damageFractionMax`（0.5）、`ruinThreshold`（0.2） |
| `CityAbilityProfileSO` | `dockedEffects`、`sailingEffects`（双形态 GA/GE） |

## 地图（Yanxu.Map）

| 标识 | 字段 |
|------|------|
| `WorldMapLayoutConfigSO` | `horizontalVisibleCellsMin`（20）、`horizontalVisibleCellsMax`（30）、`prototypeRadius` |

## 废止（勿实现新逻辑）

| 废止 | 替代 |
|------|------|
| `IPopulationFoodUpkeepPolicy` | `PopulationFoodWeeklySummaryStep` + `FoodAllocator` |
| `PopulationFoodSettlementService` | 同上 |
| `PopulationFoodSettlementRequest` / `Result` / `Phase` | `FoodAllocationResult` + `WeeklySummaryContext` |
| `PopulationHungerCohort`、`State.Hunger` | 无饥饿；未分到直接减员 |
| `team_food_upkeep_enabled` | 周总结 |
