## 目标

支撑 **侦察塔触发式视野** 配置与 **资源点揭示等级** 写入玩家已知层。**视野即时同步**；**飞信**、**待同步队列**（`vision_sync_pending`）、**关系痕迹**（`relation_trace`）、**即时通讯覆盖 / 维持成本** 等旧机制 **已废止**（下文保留历史占位供迁移对照）。

## 枚举定义

### `resource_reveal_level`

与 [单位类型与视野 · 资源点揭示等级](../../02-系统设计/06-单位与交战/单位类型与视野.md#资源点揭示等级) 一致：

| 值 | 说明 |
|----|------|
| `hidden` | **隐藏**：仅有资源点标记 |
| `kind` | **揭示种类** |
| `amount` | **揭示储量** |

## 表 1：`watchtower_config`（侦察塔配置，SO）

关联 [运作与居民 · 侦察塔](../../02-系统设计/03-图层与地点/建筑层/运作与居民.md#城区能力被动--主动)（**势力级视野增益**；与飞信 / 信号覆盖 **无关**）。

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `station_id` | string | PK | - | 侦察塔配置 ID |
| `city_vision_bonus_hex` | int | >=0 | 2 | **被动**：每座就绪侦察塔为玩家**全部**移动城市视野半径加算格数 |
| `unit_vision_bonus_pct` | int | >=0 | 50 | **主动**：本回合玩家**全部**可指挥单位视野半径增加百分比 |
| `active_cost_json` | string(json) | 可空 | `{}` | **激活开销**（激活主动能力时扣减）；与 [L4 `activation_cost_json`](./地图图层配置数据结构.md#l4_district_defscsv) 对齐；留空视为无开销 |

## 运行时：地图情报（已定）

与 [单位类型与视野 · 合并规则](../../02-系统设计/06-单位与交战/单位类型与视野.md#资源点揭示等级) 一致：

1. 队伍发现资源点或提升揭示等级 → **即时** 写入玩家战略地图已知层。
2. 写入前计算 `new_level = max(当前 reveal_level, 本次来源等级)`（序数：`hidden`=0，`kind`=1，`amount`=2）；**不降级、不叠加**。
3. 多队重叠同一格：对各来源取 max 后**一次**落库；**与结算顺序、队伍数量无关**。
4. **不** 经 `vision_sync_pending` 排队或延迟回传。
