| 标识 | 说明 |
|------|------|
| `FacilitySiteState` | 被工作对象 `facility_site` |

| `CargoNodeState` | 城区 / 设施库存节点运行时（见 [§库存节点 `cargo_node`](#库存节点-cargo_node)） |

## 人口归属转化（与设施采集区分）

| 机制 | 触发 | 速率 |
|------|------|------|
| **归属转化**（领袖 / 人口独立功能） | **玩家主动计划**；**不**自动；**不是**城区能力 | 默认目标一般为任职领袖对应归属；**学院**开启时可扩至已解锁（持有对应领袖）的任意归属；速率由领袖与词条等决定 |
| **储量采集**（矿区 / 果园等） | 玩家下达 `extract_reserve` 等工作，进度 100% 后结算 | 设施 SO + 工作效率（**待定**数值） |
| **征兵办** | 村镇 **无归属**储量 → 可招募人口：**待定**；**不**走 `FacilityProductionService` 自动路径；程序规格 [recruitment-office-data](./recruitment-office-data.md) |

## 地形类设施基本能力

**占格类 · 地形类**（桥梁、隧道等）对宿主格的修改是**设施类型自带的固有行为**，不是地形 profile 上的特殊 capability，也不走通用 Buff 管线挂 `GE_Terrain_PostTunnel` 之类条目。

| 分层 | 职责 |
|------|------|
| **L1 terrain_defs** | 无有效地形类设施时的格基线 |
| **建造白名单** | `build_allow_facility_ids` → `_program/build_grant_tags` |
| **地形类行** | `facility_defs` 中 `subtype=terrain` 的 `on_valid_*` 列 |

`subtype=terrain` 时使用的列（详见 [地图图层配置 v1 §L5](./地图图层配置数据结构.md#5-各层-defs-字段摘要)）：

| 字段名 | 说明 |
|--------|------|
| `host_terrain_ids` | 可建宿主地形 |
| `on_valid_path_cost` | 有效时覆盖路径等效 |
| `on_valid_build_mode` | 有效时覆盖建造档位 |
| `on_valid_unit_enter` | 有效时覆盖单位可否进入 |
| `on_valid_span_connect` | 有效时跨阻断连通 |
| `revert_on_destroy` | 击毁后回退至 terrain_defs 基线 |

**程序**：`FacilityLifecycleService` 建成 / 击毁时 `FacilityTerrainClassService.Apply` / `Revert`；移动与建造读合成后格参数。
