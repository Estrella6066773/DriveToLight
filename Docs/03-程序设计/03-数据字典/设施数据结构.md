| `FacilityCapacityContributor` | 屋舍承载 / 仓储加成 |

## 人口归属转化（与设施采集区分）

| 机制 | 触发 | 速率 |
|------|------|------|
| **归属转化**（领袖 / 学院等） | **玩家主动计划**（开启学院、领袖操作等）；**不**自动 | 由**任职领袖**配置，不同领袖不同 |
| **储量采集**（矿区 / 果园等） | 玩家下达 `extract_reserve` 等工作，进度 100% 后结算 | 设施 SO + 工作效率（**待定**数值） |
| **征兵办** | 村镇储量 → 可招募人口：**待定**；**不**走 `FacilityProductionService` 自动路径；归属转化另见 [领袖与势力](../../02-系统设计/05-城市与领袖/领袖与势力.md) |

## 地形类设施基本能力

**占格类 · 地形类**（桥梁、隧道等）对宿主格的修改是**设施类型自带的固有行为**，不是地形 profile 上的特殊 capability，也不走通用 Buff 管线挂 `GE_Terrain_PostTunnel` 之类条目。

| 分层 | 职责 |
|------|------|
| **地形 profile** | 描述**无有效地形类设施时**的格基线（路径、建造档位、单位可否进入） |
| **地形白名单** | 仅解决「能否在此格**开工建造**」；`Allow.Build.bridge` / `Allow.Build.tunnel` |
| **地形类设施 SO** | 描述实例**有效时**对宿主格（及桥梁的跨格连通）的覆盖；击毁按 `revert_on_destroy` 回退 |

权威 CSV：[facility_terrain_class_config.csv](./tables/L5_facility/facility_terrain_class_config.csv)

| 字段名 | 说明 |
|--------|------|
| `valid_on_host_terrain_type_ids` | 可建宿主地形 |
| `when_valid_override_unit_path_cost_equiv` | 有效时覆盖路径等效（隧道 **1**） |
| `when_valid_override_facility_build_mode` | 有效时覆盖建造档位（隧道 **auxiliary_only**） |
| `when_valid_override_unit_enter_allowed` | 有效时覆盖单位可否进入（桥梁 **true**） |
| `when_valid_enable_span_connectivity` | 有效时启用跨阻断连通（桥梁 **true**） |
| `revert_on_destroy` | 击毁后回退宿主格至 profile 基线 |

### 首版地形类设施

| `facility_type_id` | 宿主地形 | 有效时效果 |
|--------------------|----------|------------|
| `tunnel` | 山地 | 路径 **1**；建造档位升为 **仅辅助类** |
| `bridge` | 河流、裂谷 | 恢复单位进入与两端连通；路径仍按宿主地形 |

**程序**：`FacilityLifecycleService` 建成 / 运维失效 / 击毁时调用 `FacilityTerrainClassService.Apply` / `Revert`；移动与建造结算读**合成后**的格参数（profile + 有效地形类设施覆盖）。
