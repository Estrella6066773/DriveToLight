## 目标

定义 `Yanxu.Relation` 程序集的模块结构、运行时数据模型、服务接口与依赖关系。关系系统在现有各模块（City、Team、Turn、Resource）之上提供**集中的关系存储与演算**，不替代各模块已有的数据持有。

## 模块定位

### 程序集

`Yanxu.Relation` 是新增程序集：

| 维度 | 说明 |
|------|------|
| **依赖** | `Yanxu.Core`、`Yanxu.Turn`（回合阶段·关系行动钩子）、`Yanxu.Resource`（人口数据查询） |
| **被依赖** | `Yanxu.City`、`Yanxu.Team`、`Yanxu.UI` |
| **命名空间** | `Yanxu.Relation` |

> **设计原则**：关系系统是**演算层**，从 City / Team / Resource 读取人口数据与归属信息，计算关系变化后写回自身存储。各模块不需感知关系计算的内部细节，只通过关系系统的公开接口查询关系状态。

### 依赖图更新

```
Yanxu.Core
  ├── Yanxu.Map
  ├── Yanxu.Turn (依赖 Core + Map)
  ├── Yanxu.Resource (依赖 Core + Turn)
  ├── Yanxu.Relation (依赖 Core + Turn + Resource)  ← 新增
  └── ...
```

> 注：`Yanxu.Relation` 通过接口/事件与 `Yanxu.City`、`Yanxu.Team` 通信（关系行动后的脱离/效忠状态回写），但不直接依赖 City/Team 程序集，避免循环依赖。

## 运行时数据模型

### LeaderRelationRecord

| 字段 | 类型 | 说明 |
|------|------|------|
| `targetLeaderId` | string | 对方城市领袖 ID |
| `rValue` | int | 直接领袖关系 R |
| `previousTier` | RelationTier | 上回合末档位（跨越检测用） |

### LeaderRelationProfile

| 字段 | 类型 | 说明 |
|------|------|------|
| `selfLeaderId` | string | 本档案所属城市领袖 ID |
| `organizationId` | string? | 所属组织 ID |
| `records` | Dictionary<string, LeaderRelationRecord> | key = targetLeaderId |
| `cityId` | string | 所属城市 ID（用于路由） |

### RelationTier 枚举

```csharp
Hostile  = -1   // R ≤ −50
Neutral  = 0    // −50 < R < 50
Friendly = 1    // R ≥ 50
```

### PopulationLossEntry

| 字段 | 类型 | 说明 |
|------|------|------|
| `channel` | PopulationLossChannel | Unit / District |
| `count` | int | 本次损失人数 |
| `victimHomeCityRef` | string | 受害方归属城市 ID（路由到对应城市领袖） |
| `actorLeaderId` | string | 行为方城市领袖 ID |
| `turnIndex` | int | 损失发生回合 |

### PopulationLossChannel 枚举

```csharp
Unit     = 0   // 队伍/守军编制减员
District = 1   // 城区居民减员
```

### LeaderDelegation

| 字段 | 类型 | 说明 |
|------|------|------|
| `delegationId` | string | 委托唯一标识 |
| `issuerLeaderId` | string | 委托发布方城市领袖 ID |
| `delegationType` | string | 委托类型（≥2 种） |
| `status` | DelegationStatus | Pending / Accepted / Completed / Failed / Expired |
| `rewardRelation` | int | 关系奖励量（+10～+20） |
| `isRecruitDelegation` | bool | 是否为招募委托 |

### DelegationStatus 枚举

```csharp
Pending    = 0   // 待接取
Accepted   = 1   // 已接取

```

### PopulationLossTracker

```csharp
namespace Yanxu.Relation.Loss
{
    public class PopulationLossTracker
    {
        // 记录人口损失（交战/灾害等触发）
        void RecordLoss(PopulationLossChannel channel, int count,
                        string victimHomeCityRef, string actorLeaderId);

        // 结算所有待结算损失→折算关系下降（当场结算）
        void SettleAllLosses(RelationStore store, OrganizationConduction conduction);

        // 记录己方归属人口损失（招募后，4 人降 1）
        void RecordOwnedLoss(string orgId, int count, string leaderId);

        // 清空待结算队列
        void ClearPending();
    }
}
```

各轨内部维护 `pendingLossDict`（key = `(victimHomeCityRef, actorLeaderId)` → 累计值），`SettleAllLosses()` 中：

```
foreach (key, accum) in pendingLossDict：
  decrement = floor(accum / 2)
  if decrement > 0：
    store.ModifyRelation(victimLeader, actorLeader, -decrement)
    accum -= decrement * 2
```

### RelationActionService（已落地 2026-07-17，接口注入）

与 City 模块通过 `IRelationActionHandler` 接口注入通信（已定 2026-07-17）：接口定义在 `Yanxu.Relation.Actions`，City 程序集实现，Bootstrap 接线时注入，避免循环依赖。

```csharp
namespace Yanxu.Relation.Actions
{
    public interface IRelationActionHandler
    {
        // 该领袖是否处于「招募未效忠」（city_control_state = RecruitedUnloyal）
        bool IsRecruitedUnloyal(string leaderId);

        // 该领袖是否处于任一「已招募」状态（RecruitedUnloyal 或 RecruitedLoyal）
        bool IsRecruited(string leaderId);

        // 执行脱离：城区强制脱离、队伍转敌对、city_control_state → Detached
        DetachResult PerformDetach(string leaderId, string cityId);

        // 执行效忠：外部城消解、资源划归、city_control_state → RecruitedLoyal
        FealtyResult PerformFealty(string leaderId, string cityId);
    }

    public class RelationActionService
    {
        // 注入 City 侧处理器
        void BindHandler(IRelationActionHandler handler);

        // 关系行动阶段调用：批量判定并执行脱离（效忠须玩家确认，不自动执行）
        List<ActionLogEntry> ExecuteRelationActions(string playerLeaderId, int turnId);

        // 检测是否需要脱离（招募未效忠 + 领袖对玩家 R ≤ −50）
        bool CheckDetach(string playerLeaderId, string leaderId);

        // 检测是否可效忠（已招募 + 领袖对玩家 R ≥ 100）
        bool CheckFealty(string playerLeaderId, string leaderId);

        // 执行效忠（UI 确认后调用；内部再次校验条件）
        FealtyResult PerformFealty(string playerLeaderId, string leaderId);
    }
}
```

## 与现有模块的衔接

### 与 Yanxu.Turn 的关系

- `RelationOrchestrator.RunRelationPhase()` 由 `TurnPhaseMachine` 在**关系行动**阶段调用
- 现有 `TurnPhase.cs` 须新增 `RelationAction = 4`（当前枚举 `EnvironmentAction = 3` 为最大值）

### 与 Yanxu.Resource 的关系

- 人口损失数据由交战/灾害等外部事件写入 `PopulationLossTracker`
- `PopulationLossTracker` 不直接持有人口数据，通过接口从 `Yanxu.Resource` 查询归属关系
- 己方归属人口损失的索引用 `organizationId` 而非 `cityId`（4 人降 1 为组织级统一累计）

### 与 Yanxu.City / Yanxu.Team 的关系

- 脱离时通过**事件/回调**通知 City 模块（城区强制脱离、队伍转敌对）
- 效忠时通过**事件/回调**通知 City 模块（城市消解、资源划归）
- 不直接依赖 City/Team 程序集，通过 C# 事件或接口注入避免循环依赖

### 回合阶段枚举更新

`TurnPhase.cs` 须新增一位：

```csharp
public enum TurnPhase
{
    PlayerCommand     = 0,
    PlayerAction      = 1,
    AiAction          = 2,
    EnvironmentAction = 3,
    RelationAction    = 4   // 新增：关系结算
}
```
