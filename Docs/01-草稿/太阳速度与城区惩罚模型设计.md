## 目标

定义玩家移动城市的游玩节奏模型，确定太阳速度参数，并评估城区数量对城市移动的惩罚机制。

## 范围

- **包含**：太阳速度计算公式、两种玩法节奏方案、城区惩罚设计方案对比与推荐
- **不包含**：具体数值实现、程序代码、界面交互设计

## 一、核心定义

### 1.1 时间单位

| 单位 | 定义 | 说明 |
|------|------|------|
| **回合** | 游戏最小时间单位 | 每回合包含玩家指挥、玩家行动、AI行动、环境行动、关系行动五个阶段 |
| **周期（周）** | 7回合 | 粮食周总结结算周期；`turn_number % 7 == 0` 触发 |
| **格** | 六边形地图单位 | 城市移动的最小距离单位 |

### 1.2 移动参数

| 参数 | 当前值 | 状态 | 说明 |
|------|--------|------|------|
| **城市基础移动速度** | 1格/回合 | **已定** | 航行状态下，无额外惩罚时的移动速度 |
| **停泊↔航行切换时间** | 1回合 | 已定 | 状态切换作为核心区工作，需1回合完成 |
| **太阳速度** | v(t) = v₀ + a·t | 模型已定 | 恒定加速度；n₀=1 格/周（B宽松，日生之地起点）→ n=3 格/周（A宽松，铁门关）；a 为 n 值增量；每 7 回合推进，向下取整 |
| **黄昏带/暗渊带宽** | 横跨整个地图 | **已定** | 类似日落时整个地平线变色，非局部区域带 |

## 二、太阳速度计算模型

### 2.1 核心假设

1. 玩家需要在移动阶段覆盖太阳在整个周期内移动的距离，才能保持相对位置不变
2. 城市速度 = 1格/回合（当前设计，待定）
3. 太阳速度 = n格/周期 = n/7格/回合

> **说明**：当前讨论仅针对匀速阶段。恒定加速度模型 v(t) = v₀ + a·t 在后续章节中引入，a 的具体数值待定。

### 2.2 方案A：移动2周期，探索2周期（2:2节奏）

**玩法节奏**：
- 移动阶段：2周期 = 14回合，推进14格
- 探索阶段：2周期 = 14回合，城市停泊不动
- 总周期：4周期 = 28回合

**计算公式**：
```
城市移动距离 = 太阳总移动距离
14格 = (n/7) × 28回合
n = 14 × 7 / 28 = 3.5
```

**推荐太阳速度**：

| 方案 | n值（格/周期） | 实际速度（格/回合） | 效果 |
|------|---------------|-------------------|------|
| 基准线 | 3.5 | 0.5 | 精确平衡，无压力 |
| 推荐值 | 3 | ~0.43 | 略宽松，适合前期 |
| 挑战性 | 4 | ~0.57 | 有压力，适合后期 |

**适用场景**：
- 强调平衡的玩法体验
- 玩家有充足时间探索和发展
- 适合第一章"初速度"阶段

### 2.3 方案B：移动1周期，探索3周期（1:3节奏）

**玩法节奏**：
- 移动阶段：1周期 = 7回合，推进7格
- 探索阶段：3周期 = 21回合，城市停泊不动
- 总周期：4周期 = 28回合

**计算公式**：
```
城市移动距离 = 太阳总移动距离
7格 = (n/7) × 28回合
n = 7 × 7 / 28 = 1.75
```

**推荐太阳速度**：

| 方案 | n值（格/周期） | 实际速度（格/回合） | 效果 |
|------|---------------|-------------------|------|
| 基准线 | 1.75 | ~0.25 | 精确平衡，无压力 |
| 推荐值 | 1 | ~0.14 | 宽松，适合探索导向 |
| 挑战性 | 2 | ~0.29 | 有压力，需要高效移动 |

**适用场景**：
- 强调探索和资源收集的玩法体验
- 玩家有大量时间进行城市建设和任务
- 适合需要深度经营的章节

### 2.4 方案对比

| 维度 | 方案A（2:2节奏） | 方案B（1:3节奏） |
|------|------------------|------------------|
| **移动压力** | 中等 | 较低 |
| **探索时间** | 50% | 75% |
| **太阳速度基准** | n=3.5 | n=1.75 |
| **推荐太阳速度** | n=3～4 | n=1～2 |
| **游戏节奏** | 紧凑平衡 | 悠闲探索 |
| **适用章节** | 第一章、第二章 | 第一章前期、第三章 |

## 三、城区惩罚设计方案

本文中「策略余裕」指玩家通过提前规划路线、控制城区规模、投入资源（科技/设施/领袖技能）来规避惩罚的操作空间。

### 3.1 方案A：切换状态惩罚

**设计机制**：
城市在停泊/航行状态切换时，每加装n个城区，切换时间增加1回合。

**公式**：
```
切换时间 = 1回合 + floor(城区数量 / n) × 1回合
```

**参数建议**：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| n | 3～4 | 每3～4个城区增加1回合切换时间 |
| 最大额外时间 | 2～3回合 | 设置上限，避免极端情况 |

**优点**：
- 决策集中，玩家有明确的决策点
- 战略权衡清晰：城区收益 vs 切换成本
- 节奏控制：强制玩家在切换时停顿
- 计算简单，玩家容易理解

**缺点**：
- 惩罚一次性，航行中移动不受影响
- 可能导致玩家长时间保持同一状态
- 与移动距离无关，无论移动多远惩罚相同

### 3.2 方案B：移动惩罚

**设计机制**：
城市每移动1格需要1回合（当前设计）。每加装 m 个城区，移动1格额外增加1回合。

**公式**：
```
移动1格所需回合 = 1回合 + floor(城区数量 / m) × 1回合
```

**参数建议**：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| m | 3～4 | 每3～4个城区增加1回合/格 |
| 最大额外时间 | 2回合/格 | 设置上限，避免极端情况 |

**优点**：
- 持续压力：每次移动都会感受到城区负担
- 与距离相关：移动越远，惩罚累积越大
- 动态平衡：城区越多，追日越难，形成自然难度曲线
- 与现有负载成本设计逻辑一致

**缺点**：
- 计算相对复杂，玩家需要实时计算移动成本
- 惩罚频率高，可能导致挫败感
- 城区较多时，移动成本可能过高

### 3.3 方案对比

| 维度 | 方案A（切换惩罚） | 方案B（移动惩罚） |
|------|------------------|------------------|
| **惩罚时机** | 状态切换时 | 每次移动时 |
| **惩罚频率** | 低 | 高 |
| **策略深度** | 中等 | 高 |
| **玩家感知** | 明确、集中 | 持续、分散 |
| **计算复杂度** | 低 | 中高 |
| **与现有系统兼容性** | 好（切换已作为核心区工作） | 好（与负载成本呼应） |
| **策略余裕空间** | 充足 | 有限 |
| **适合游戏节奏** | 慢节奏策略 | 快节奏追日 |
| **叙事契合度** | 一般 | 好（大城更笨重） |

### 3.4 推荐方案

**推荐选择：方案A（切换惩罚）**

**理由**：

1. **策略余裕空间更大**：
   - 玩家可以通过减少切换次数、提前规划路线来规避惩罚
   - 一次性惩罚比持续性惩罚更容易通过策略管理
   - 即使城区很多，也可以通过优化策略维持城市速度

2. **缓解机制更直观且丰富**：

| 缓解机制 | 效果 | 策略选择 |
|----------|------|----------|
| 特定城区能力 | "引擎核心"城区减少切换时间 | 优先加装/保留该城区 |
| 领袖能力 | 赫利奥技能：立即完成切换 | 在关键时刻使用技能 |
| 科技/设施 | 建造"快速坞站"减少切换惩罚 | 投资基础设施 |
| 工作效率修正 | 高工作效率加速切换 | 配置高效团队 |

3. **与现有系统兼容**：
   - 切换时间已作为核心区工作，有成熟的工作效率修正机制
   - 工作效率通道已支持加法/乘法修正项

4. **玩家体验更友好**：
   - 惩罚集中在切换时刻，玩家有明确的决策点
   - 不会在每次移动时都感受到压力，减少挫败感

**推荐参数**：
- n = 4（每4个城区增加1回合切换时间）
- 最大额外时间 = 2回合（设置上限，避免极端情况）

**策略示例**：
- 玩家拥有10个城区，基础切换时间 = 1 + min(floor(10/4), 2) = 3回合
- 通过"引擎核心"城区能力减少1回合 → 2回合
- 通过领袖技能立即完成 → 0回合额外时间

## 四、综合模型

### 4.1 完整移动成本公式（推荐方案）

```
移动成本（回合）= 切换时间 + 移动格数 × 1回合

其中：
- 切换时间 = 1回合 + min(floor(城区数量 / 4), 2)回合（停泊↔航行）
- 移动每格成本 = 1回合（航行状态下无额外惩罚）
```

### 4.2 示例计算

**场景**：玩家拥有5个城区，需要移动3格

```
切换时间 = 1 + min(floor(5/4), 2) = 1 + 1 = 2回合
移动成本 = 3 × 1 = 3回合
总移动成本 = 2 + 3 = 5回合
```

**场景**：玩家拥有10个城区，需要移动5格

```
切换时间 = 1 + min(floor(10/4), 2) = 1 + 2 = 3回合
移动成本 = 5 × 1 = 5回合
总移动成本 = 3 + 5 = 8回合
```

**场景**：玩家拥有10个城区，使用"引擎核心"能力减少1回合切换时间

```
切换时间 = 3 - 1 = 2回合（引擎核心减1回合）
移动成本 = 5 × 1 = 5回合
总移动成本 = 2 + 5 = 7回合
```

**场景**：玩家拥有10个城区，使用领袖技能立即完成切换

```
切换时间 = 0回合（领袖技能立即完成）
移动成本 = 5 × 1 = 5回合
总移动成本 = 0 + 5 = 5回合
```
