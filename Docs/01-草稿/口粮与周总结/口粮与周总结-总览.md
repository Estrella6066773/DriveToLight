## 1. 已定案速查


| 主题           | 定案                                                                                       |
| ------------ | ---------------------------------------------------------------------------------------- |
| **周总结期**     | 每 **7 回合**；buff、地图等作为 **周总结子步骤**（按模块排序，§2.1b）                                            |
| **触发时机**     | **第 7 回合环境行动结束后** → **周总结** → **第 8 回合玩家指挥**；之后第 14、21… 回合同理                             |
| **回合编号**     | 从 **1** 起计；`turn_number % 7 == 0` 且环境行动完成后触发                                             |
| **挂钩点**      | **不**增第五阶段；环境行动 **先** 完成，**再** `WeeklySummaryOrchestrator.Run()`，**再** 进入下一回合指挥          |
| **子步骤顺序**    | **按模块排序**；在 `WeeklySummaryStepCatalog` **一处配置**，代码可调、可测                                  |
| **结算单元**     | `**mobile_city_id`**（相连分区）；每单元 **独立** 口粮池 + `FoodAllocator` + cohort                     |
| **需求**       | 分区内 **活跃人口 × 1 口粮/人/周**（城区含废墟 + 队伍编制与载荷人口；**不含** 村镇储量）                                   |
| **库存**       | 分区内 **城市口粮池 + 绑定队伍载荷口粮**；周总结汇总后分配                                                        |
| **分配策略**     | 口粮 **充足** 时人人 1:1；**不足** 时按 `**FoodShortagePriorityMode`** 二选一（**默认 `WorkFirst` 工作者优先**） |
| **工作者优先**    | 模式 `WorkFirst`：**先** 保障全部在岗人口，**再** 余粮给非在岗（见 §2.4a）                                      |
| **归属优先**     | 模式 `AffiliationFirst`：**先** 按归属优先级切份额，**再** 各归属内工作者优先（见 §2.4b）                           |
| **策略切换**     | 两种算法 **分别实现**、**可配置切换**；配置经 `**FoodAllocationConfig`**，后续其它模块可 **Modifier** 改写           |
| **饥饿**       | 未分到口粮 → **饥饿** → **无法工作**；下次分到 → **自动解除**                                                |
| **连续饥饿致死**   | 上周期已饥饿且本周期仍未分到 → **死亡 ⌊饥饿×50%⌋**；余下 **仍饥饿**（默认 **向下取整**）                                 |
| **追踪**       | **仅周总结** 批量判定；**不**做回合内逐时刻追踪                                                             |
| **废墟**       | **计** 口粮需求与均分；**永远不能工作**（与饥饿无关）                                                          |
| **村镇**       | **可提取储量**；占领/提取前 **不计** 势力口粮                                                             |
| **队伍**       | **所有队伍有载荷**（运输队经资产 **载荷更大**）；周总结与城区 **同规则**；**废止** OPEN-042 每回合另扣                        |
| **玩家范围**     | **仅** 城市领袖 **已加入玩家阵营** 的城区（及相连网络）；**占领但领袖未入阵营 ≠ 玩家**                                     |
| **相邻 vs 相连** | **相邻** = 地图邻格；**相连** = 连接拓扑连到核心区网络（文档「连接」= 本方案「相连」）                                      |
| **迁移（同分区）**  | **相连** 同一 `mobile_city_id` 的 **玩家经营** 城区间可迁；跨分区须分离/再连接或运输队载荷                             |
| **无未安置池**    | **不存在** `UnassignedPool` / 城市虚池；人口 **只** 在活跃容器或村镇储量                                      |
| **归属优先范围**   | 归属优先级策略 **仅** 在 **同一 `mobile_city_id` + 玩家经营网络** 内生效                                     |
| **废止旧口径**    | **废止**：OPEN-042 每回合另扣；**城市需求 − 外出编制** 双轨计费                                               |
| **工作统一入口**   | 饥饿/减员/恢复/特殊城区维持 → `**ApplyWorkHeadcountChange`**（队伍同思路）                                  |
| **GAS**      | 饥饿 **人数** = cohort **权威**；ASC 仅 **Tag/GE 投影**（如 `State.Hunger`）                          |

## 6. 设计 rationale（简要）

- **7 回合一批**：降微操，与 buff/地图同频；第 6 回合可预告下周期风险。
- **半数致死**：比全灭可读、可补救；连续断粮仍危险（指数减员）。
- **mobile_city_id 独立结算**：对齐多核心分离玩法；带粮远征可规划。
- **领袖入阵营**：占领与经营解耦；与势力主城区定义一致。
- **Delta 统一入口**：特殊城区无分支；饥饿/减员/恢复同路径；**禁止**城区模块双写状态。
