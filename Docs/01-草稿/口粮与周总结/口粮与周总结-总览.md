## 1. 已定案速查

| 主题 | 定案 |
| ------------ | ---------------------------------------------------------------------------------------- |
| **周总结期** | 每 **7 回合**；buff、地图等作为 **周总结子步骤**（按模块排序，§2.1b） |
| **触发时机** | **第 7 回合为正常行动回合**；**环境行动结束后** → 周总结 → **第 8 回合**指挥；之后第 14、21… 同 |
| **回合编号** | 从 **1** 起计；`turn_number % 7 == 0` 且环境行动完成后触发 |
| **挂钩点** | **不**增第五阶段；环境行动 **先** 完成，**再** `WeeklySummaryOrchestrator.Run()` |
| **子步骤顺序** | **按模块排序**；`WeeklySummaryStepCatalog` **一处配置** |
| **结算单元** | **`mobile_city_id`**（相连分区）；**城市仓库扣减** + **队伍载荷** **分池** |
| **城市分配** | **L1** 城区驻留人口占比 → **L2** 城区内归属人口占比 → **L3** 归属内同比例 |
| **需求** | 活跃人口每周期 × **1** 食物/人（周总结口径 = **1** 口粮/人/周；城区含废墟 + 队伍编制 + **随行人员**；**不含** 村镇储量） |
| **队伍载荷** | 设计目标：默认载荷可维持约 **3** 周期口粮；长期外派须 **运输队** 输送（数值待定） |
| **不足减员** | 未分到 → **⌊n/2⌋ 死亡、⌈n/2⌉ 存活**；**无饥饿**中间态 |
| **仓库扣减** | 「**优先用于充饥**」优先仓均分 → 不足再其它仓（§2.8） |
| **Override** | 领袖/归属能力 **先一轮**；一般规则只吃 **剩余** stock/人口（§2.4.3） |
| **工作者优先** | **仅 Override 能力**（§2.4.4）；默认在岗 **不** 优于其他归属 |
| **废墟** | **计** 口粮需求；**永远不能工作** |
| **村镇** | **可提取储量**；提取前 **不计** 势力口粮 |
| **队伍** | **编制** + **随行人员**；**载荷口粮** 独立容器；**废止** OPEN-042 每回合另扣 |
| **随行人员** | **非** 编制；占载荷、增口粮需求；接纳时已属玩家（§2.10） |
| **玩家范围** | **仅** 领袖 **已入玩家阵营** 的城区（及相连网络）；**占领 ≠ 玩家** |
| **相邻 vs 相连** | **相邻** = 地图邻格；**相连** = 连接拓扑连通 |
| **迁移（同分区）** | **相连** 同一 `mobile_city_id` 的 **玩家经营** 城区间可迁 |
| **无未安置池** | **不存在** `UnassignedPool` |
| **UI** | **常驻** 每回合口粮充足性；多分城标注不足（§2.9） |
| **工作统一入口** | 口粮/战斗减员 → **`ApplyWorkHeadcountChange`** |
| **废止** | 饥饿 cohort、`WorkFirst`/`AffiliationFirst` 全局、载荷迁徙人口、`State.Hunger` |

## 6. 设计 rationale（简要）

- **7 回合一批**：降微操，与 buff/地图同频。
- **无饥饿**：当周未分到 → 半数减员；幸存者本周期可工作；规则更简单。
- **比例公平**：L1→L2→L3 按人口占比；领袖能力 **Override** 破例。
- **分池**：城市仓库与队伍载荷独立；带粮远征可规划。
- **随行人员**：废止迁徙人口；与运输载具类队伍分轨。
- **Delta 统一入口**：减员/恢复同路径；无 cohort 跨周状态。
