## 1. 核心乐趣

### 1.1 一句话体验

驾驶一座可拼接、可拆解的模块化移动城市，在太阳逐渐的永昼世界里追逐太阳。体验亲手建造设施，又被迫将其放弃的求生之旅。

### 1.2 三条核心乐趣

| 乐趣轴 | 玩家感受 | 设计落点 |
|--------|----------|----------|
| **夹缝求生的经营感** | 暗渊逼近、资源枯竭、势力纠缠——永远在压力下做出选择 | 四种核心资源循环、周总结粮食、动态难度 |
| **建设与自毁的取舍** | 亲手拆掉自己建的城区以换取速度与通过性；城市形态记录你的每一次抉择 | 城区拼接/分离/拆解/航行放弃；完整度与负载系统 |
| **沉浸式剧情参与** | 旅途中遭遇势力与事件，逐步揭开世界真相，认同城主身份 | 章节里程碑、势力初遇、城主身份揭示 |

### 1.3 差异化张力

| 张力 | 说明 |
|------|------|
| **太阳朋克 × 后启示录** | 艳阳、巨城、金色光带 vs 正在变暗的废土——视觉与基调的强烈冲突 |
| **亲手拆掉自己的成果** | 对标《冰汽时代》式道德压力，但载体是**城市拓扑与完整度**，不是温度条 |
| **长线规划满足感** | 回合策略下的区域扩张、据点开发、跨章目标，对标《文明》式时间尺度 |

> 参考作品：《文明六》《IXION》《冰汽时代》《无光之海》。  
> 详见 [核心幻想](../02-系统设计/01-核心体验/核心幻想.md)、[循光之城-策划案 §3 & §6](./循光之城-策划案.md)。

## 2. 核心循环

### 2.1 三级时间尺度

| 尺度 | 跨度 | 玩家在做什么 |
|------|------|--------------|
| **短期** | 数回合内 | 本回合下达指令；查看资源与人口；调整队伍编制；处理眼前事件 |
| **中期** | 十数回合 | 组织勘探与开发；规划城区拼接与拆解；周旋势力关系 |
| **长期** | 整局 | 追日或转入暗渊；扩张城市能力；推进五章主线与终局 |

### 2.2 一轮活动循环

```
规划路线 → 移动城市 → 停下交互 → 勘探与开发 → 资源与人口管理 → 循环
```

| 环节 | 玩家在权衡什么 |
|------|----------------|
| 停下开发 | 是否建采集设施（果园/矿区/能源站/征兵办），值不值得付建造金属与回合 |
| 资源管理 | 共用仓库囤什么；周粮够不够；负载损伤怎么修 |
| 确认生存 | 仓还能装多少；完整度与修复金属够不够撑到下一站 |
| 改造拆解 | 仅主动拆解回收金属（−10% → +10金属）；航行放弃 −40% 不回收 |

### 2.3 每回合五阶段

1. **玩家指挥**：编辑指令表与行动表
2. **玩家行动**：主城必然最先执行
3. **外部城市 AI**：势力城市按种子顺序行动
4. **环境结算**：暗渊推进、第 7 回合周总结粮食
5. **关系结算**：脱离、效忠等阈值跨越在此阶段一并结算；下回合起生效

> 权威：[势力系统 · 关系行动](../02-系统设计/05-城市与领袖/势力系统.md#关系行动已定)

### 2.4 Demo 范围：核心系统裁剪

前两章覆盖完整的核心循环——追日→停泊→建设→勘探→势力交互→继续追日——因此**全部 8 个模块的基础能力都需要**。

| 模块 | 当前工程状态 | Demo 需要 | 缺口 |
|------|:---:|-----------|------|
| 01 核心体验 | — | 追日阶段胜利条件、动态难度 | 需配置 |
| 02 地图与世界 | 85% | 六边形地图、停泊/航行、环境层（日照/暗渊） | 环境层运行时（日照带/黄昏带/暗渊带随时间上移） |
| 03 图层与地点 | 75% | 城区拼接/分离/拆解、负载修复 | **已就绪**（多核心可后续） |
| 04 资源与人口 | 75% | 金属/食物/能源/人口四轨、共用仓库、荒野采集 | 粮食周总结已完成；金属/能源数值框架 |
| 05 城市与领袖 | 0% | 铁壳势力 + 领袖关系 + 委托/贸易 | **从零开发**（友善八折贸易、恒3条委托、招募/效忠链等见 [势力系统](../02-系统设计/05-城市与领袖/势力系统.md)） |
| 06 单位与交战 | 85% | 7 种队伍类型、视野、基础交战 | 侦察/勘探/搬运 executor；视野运行时；交战管线（攻击系数×人数系数→防御减免→结构承伤→人口损失，见 [交战系统](../02-系统设计/06-单位与交战/交战系统.md)） |
| 07 玩法循环 | 90% | 回合五阶段、跨回合工作、探索 | **已就绪**（五阶段已定；关系结算待落地） |
| 08 关卡与叙事 | 0% | 第一章 + 第二章关卡数据与事件触发 | **待配表**（事件按章节配置，见 [08-关卡与叙事](../02-系统设计/08-关卡与叙事/)） |

**Demo 终点**：铁巢终局，转向暗渊（第二章节拍结束）。

> 工程详情见 [03-程序设计](../03-程序设计/)；设计见 [02-系统设计](../02-系统设计/)。

## 3. 制作人员需求

### 3.1 团队编制（4 人）

| 角色 | 人员 | 职责 |
|------|:---:|------|
| **制作人 / 主策划** | 你 | 设计方向、数值框架、关卡配置、进度管理、程序兜底 |
| **程序** | 1 人 | 全部工程实现（六边形地图、回合系统、城区系统、资源循环、AI、UI 逻辑） |
| **地图 / 场景美术** | 1 人 | 六边形地块模型、城区模块、荒野资源点、日照/暗渊视觉 |
| **人物 / UI 美术** | 1 人 | 角色模型（城主、势力领袖、队伍单位）、UI 界面与图标 |

| 职能 | 方案 |
|------|------|
| 音频 | Demo 阶段暂缺或使用临时素材 |
| QA | 全员自测 |

### 3.2 关键约束

- **程序只有一人**，你是兜底。架构要简单、可快速迭代；优先用 Unity 内置功能和现有插件，避免造轮子。
- **美术资源量**需严格控制：Demo 仅需主城与城区变体、荒野资源点模型、铁巢势力场景、少量 UI 面板。
- **设计文档已有基础**（详见 [02-系统设计](../02-系统设计/)），一个月内以实现为主，不作新的大设计。

## 5. 风险与假设

| 风险 | 等级 | 缓解措施 |
|------|:---:|----------|
| 前两章需要全部基础系统，W1-W2 程序压力极大 | 高 | 已有 80% 骨架兜底；W1 优先环境层+资源四轨+executor，W2 视野+交战+事件管线；你程序兜底分担 |
| 势力系统从零开发 + 运输/贸易交互 | 中 | Demo 仅 1 个势力（铁壳）；委托/贸易按 [势力系统](../02-系统设计/05-城市与领袖/势力系统.md) 首版口径（恒3条委托、友善八折） |
| 基础交战可能影响 W2 进度 | 中 | 按 [交战系统](../02-系统设计/06-单位与交战/交战系统.md) 已定管线：攻击系数×人数系数→结构减免→承伤→人口损失，先实现最小闭环 |
| 日照/暗渊（环境层子系统）视觉效果达不到 pitch 要求 | 中 | W1 先用材质区分；动效挪到 W4 打磨 |
| 美术资源量超出一月产能 | 低 | Demo 仅需主城、铁门关、铁巢、设施、荒野、队伍、人物共十几组模型，量可控 |
| 关卡数据配表返工频繁 | 中 | SO 配置表可热修改，不阻塞程序；你用 W2-W3 集中配表即可 |

**假设**：已有 Unity 工程继续使用；Demo 面向内部演示，不处理本地化/Steam/音频。
